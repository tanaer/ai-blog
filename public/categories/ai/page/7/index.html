你有没有想过,当一个 AI Agent 可以自主访问文件,调用工具,执行代码时,怎么确保它不会"翻车"? 就在2小时前,Netzilo 发布了 AI Edge,专门为 OpenClaw 这类自主 Agent 提供企业级安全防护.这事儿对正在用 AI Agent 的团队来说,意义重大. 为什么 OpenClaw 需要专属安全方案? OpenClaw 的能力很强:能自主执行复杂任务,调用各种技能(skills),访问本地系统.但能力越强,风险越大: 可见性盲区:你很难搞清楚 Agent 到底在跟谁通信,调了什么工具,动了哪些文件, Agent 完整性风险:工具投毒,恶意技能注入,提示注入攻击……这些攻击方式在传统安全里根本没见过, 数据泄露风险:即使是本地部署的 Agent,也可能访问或生成敏感企业数据 传统安全产品根本覆盖不了这些场景.

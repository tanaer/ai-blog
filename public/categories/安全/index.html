Agent Passport:AI Agent 的「身份证」,像 OAuth 一样验证身份

AI Agent 正在爆发: OpenClaw 18万+ GitHub stars Moltbook 230万 Agent 账户 但问题是:没有标准方式证明 Agent 身份. 这意味着: 恶意 Agent 可以冒充其他 Agent Skill/Plugin 市场没有认证层 Cisco 安全团队已经在第三方 Agent skills 中发现数据泄露 Agent Passport 就是来解决这个问题的. 核心特性 Ed25519 挑战-响应认证 私钥永远不会离开 Agent, 公钥用于验证身份, 抗量子计算攻击 JWT 身份令牌 60 分钟有效期(TTL), 可撤销, 包含 Agent 元数据和权限 ️ 风险评分引擎 0-100 分评分系统: Allow (0-30): 低风险,正常通行, Throttle (31-70): 中等风险,限制频率, Block (71-100): 高风险,拒绝访问 一行代码验证 const result = await passport.verify(token); 快速开始 安装 npm install @zerobase/agent-passport ### Agent 端:生成身份 import { AgentPassport } from '@zerobase/agent-passport'; // 生成密钥对(只需一次) const passport = new AgentPassport(); const identity = await passport.generateIdentity(); // 保存私钥(安全存储) const privateKey = identity.privateKey; // 公钥就是你的 Agent ID const agentId = identity.publicKey; ### 服务端:验证 Agent import { PassportVerifier } from '@zerobase/agent-passport'; const verifier = new PassportVerifier({ riskThreshold: 50, // 风险阈值 ttl: 3600 // 令牌有效期(秒) }); // 验证 Agent 令牌 const result = await verifier.verify(token); if (result.allowed) { console.log(`Agent ${result.agentId} 验证通过,风险评分: ${result.riskScore}`); } else { console.log(`验证失败: ${result.reason}`); } ## 工作原理 ### 1. 身份注册 Agent 生成 Ed25519 密钥对: - 私钥保存在 Agent 本地 - 公钥作为 Agent ID 注册到 Passport 网络 ### 2. 挑战-响应 当 Agent 需要证明身份时: 1. 服务端发送随机挑战(nonce) 2. Agent 用私钥签名挑战 3. 服务端用公钥验证签名 4. 验证通过,发放 JWT 令牌 ### 3. 风险评分 基于多维度信号评分: - 历史行为(请求频率,成功率), 声誉数据(其他服务的反馈), 设备指纹(运行环境特征), 网络信号(IP 信誉,地理位置) ## 实际应用场景 ### 场景 1:Skill 市场安全 你运营一个 OpenClaw Skill 市场: - 没有验证:任何 Agent 都可以安装 Skill,恶意 Skill 可能窃取数据 - 有 Agent Passport:Skill 可以验证调用者身份,只服务可信 Agent // Skill 端验证 app.use(async (req, res, next) => { const token = req.headers['x-agent-token']; if (!result.allowed || result.riskScore > 50) { return res.status(403).json({ error: 'Agent not trusted' }); req.agentId = result.agentId; next(); ### 场景 2:Agent 间协作 多个 Agent 需要协作完成任务: - Agent A 需要确认 Agent B 的身份才能共享敏感数据 - 通过 Agent Passport 验证,确保对方不是冒充者 ### 场景 3:企业级权限控制 企业内部有多个 AI Agent: - 财务 Agent 只能访问财务系统 - HR Agent 只能访问 HR 系统 - 通过 Passport 的权限声明,实现细粒度访问控制 ## 技术架构 ┌─────────────┐ ┌──────────────┐ ┌─────────────┐ │ Agent │ ───▶ │ Agent Passport│ ───▶ │ Service │ │ (Client) │ │ (Identity) │ │ (Verifier) │ └─────────────┘ └──────────────┘ └─────────────┘ │ │ │ │ 1. BBC科技记者Thomas Germain做了一个实验. 他花20分钟写了一篇假文章,声称自己是"吃热狗最多的科技记者"–没有任何证据,完全瞎编.文章引用了虚构的"2026南达科他国际热狗锦标赛". 不到24小时,ChatGPT和Google AI就开始向全世界传播这个谎言. 实验细节 Germain在自己个人网站发布了一篇题为"The best tech journalists at eating hot dogs"的文章: 宣称"竞技吃热狗"在科技记者圈很流行, 把自己排在第一名, 列出几个真实记者(事先征得同意), 没有任何来源,整个故事都是编的 第二天: Google Gemini:直接引用他的文章内容, Google AI Overviews(搜索结果顶部的AI回答):同样照单全收, ChatGPT:也把谎言当事实传播, Claude:唯一没有被愚弄的 有时AI会标注"这可能是个玩笑".Germain把文章改成"这不是讽刺"–然后AI就开始更认真地对待了. 他又做了一个测试:编造了一份"最伟大的呼啦圈交通警察"名单.AI现在还在赞美虚构的"Maria ‘The Spinner’ Rodriguez警官". 为什么这很可怕 这不是恶作剧. 同样正在被用于: 医疗建议(大麻软糖"无副作用,绝对安全"), 金融投资(“最佳黄金IRA公司”), 植发诊所推荐, 政治信息 SEO专家Harpreet Chatha展示了Google AI如何传播虚假的医疗声明–关于某品牌大麻软糖"绝对安全无副作用".实际上这些产品有已知副作用,可能与某些药物相互作用,而且在不受监管的市场中存在污染风险. 方法很简单: 在自己网站写篇文章 把你的品牌排在"最佳"列表第一位 填几个竞争对手凑数 等AI引用你 “任何人都能做到.太愚蠢了,感觉完全没有护栏.“Chatha说. 比搜索引擎时代更糟 操纵搜索结果不是新鲜事.Google有成熟的反垃圾系统. 但专家说,AI工具让行业倒退了20年. “欺骗AI聊天机器人比两三年前欺骗Google容易得多,“Amsive营销公司的SEO副总裁Lily Ray说,“AI公司发展的速度超过了它们监管准确性的能力.我认为这很危险.” 更糟的是–用户更容易被AI欺骗. 传统搜索:你得点击链接进入网站,自己判断信息可信度.如果网站写着"我是最棒的记者”,你可能会想"这是他自己在吹”. AI搜索:信息看起来像是来自科技公司本身.58%的人在AI Overview出现时不会点击任何链接. “在追求利润的竞赛中,我们的安全–以及普通人的安全–正在被牺牲,“Chatha说. “数据空洞"问题 Google说大多数例子是"极不常见的搜索”,不反映正常用户体验. 但Ray指出:Google自己说每天15%的搜索是全新的.AI正在鼓励人们问更具体的问题–这正是垃圾信息制造者想要的. Google承认这些"数据空洞"可能导致低质量结果,并表示正在努力阻止AI Overview在这些情况下出现. 解决方案? 专家建议: 更醒目的免责声明–AI应该明确标注信息来源 透明度–如果信息来自新闻稿或只有一个来源,应该告诉用户 用户自己警惕–AI传递谎言的语气和事实一样权威 关键建议: AI适合常识性问题(弗洛伊德的理论,二战历史), 不适合医疗建议,法律规则,本地商家推荐, 永远要查来源–AI引用了什么?有几个?谁写的?, 不要被AI的自信语气欺骗 我的判断 这不是技术漏洞,是设计缺陷. ...

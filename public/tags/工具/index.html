MCP 协议：AI 的 USB-C 端口，终于统一了工具接口

痛点：每个 AI 都要重写一遍插件 过去几年，让 AI 连接 Slack、读取文件、操作数据库，简直是一场噩梦。每个平台都有自己的插件系统：OpenAI 有 Plugins，Claude 有 Tools，Gemini 有 Extensions… 你写好的 Slack 集成，换个模型就得重来一遍。 MCP：通用接口来了 Model Context Protocol (MCP) 就是 AI 界的 USB-C。 一次开发，到处运行 — 写好一个 MCP server，所有支持的模型都能用 安全隔离 — 工具权限在服务器端控制，模型拿不到你的密钥 开放标准 — 不被任何公司垄断，社区驱动发展 为什么这很重要？ 以前 AI 只能在"实验室"里玩。MCP 让 Agent 真正走出实验室，变成能干活的数字员工。 现在任何模型都能： 接入你的日历和邮件 读写本地文件 调用 API 执行任务 和其他 Agent 协作 谁在支持？ 主流模型几乎全加入了：Claude、GPT、Gemini、Llama… 你的工具投资不再被锁定在单一平台。 这就是为什么 2026 年被称为"Agent 元年"——MCP 把最后一道门槛拆掉了。 一句话总结：MCP 让 AI 工具像 USB 设备一样即插即用，终于不用为每个模型重写插件了。

go-size-analyzer：浏览器里分析 Go 二进制文件大小，还能生成树状图

Go 工具链生态又添新成员。go-size-analyzer 是一个分析编译后 Go 二进制文件大小的工具，支持跨平台格式（ELF、Mach-O、PE、WebAssembly），并提供树状图可视化。 最酷的是，它编译成了 WebAssembly，托管在 gsa.zxilly.dev，你可以直接在浏览器中打开编译好的二进制文件进行分析——无需安装任何东西。 功能亮点： 详细分解：按包、按段展示大小占比 多种输出：text、json、html、svg 交互式界面：Web UI 和终端 UI 都支持 Diff 模式：对比两个二进制文件的大小变化 安装也很简单： # Homebrew brew install go-size-analyzer # Scoop scoop install go-size-analyzer # Go install go install github.com/Zxilly/go-size-analyzer/cmd/gsa@latest 本地使用： gsa --web your-binary Simon Willison 试用了这个工具，发现 Datadog 团队用它把 Agent 二进制体积减少了 77%。对于微服务和容器化部署场景，这个工具能帮你快速定位体积膨胀的源头。 项目地址：github.com/Zxilly/go-size-analyzer 推文草稿 Go 二进制文件太大了？试试 go-size-analyzer。 最酷的是它能编译成 WebAssembly，在浏览器里直接分析二进制——不需要安装任何东西。 Datadog 团队用它把 Agent 体积砍了 77%。 在线体验：https://gsa.zxilly.dev #Go #WebAssembly #性能优化

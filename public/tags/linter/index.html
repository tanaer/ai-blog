Sensei:用 Ralph 循环自动改进 SKILL.md,让 Agent 知道什么时候该用/不该用

刚发现一个有意思的技能改进工具:Sensei 谁能用? 写 AI Agent 技能文件的.技能写得不好,Agent 不知道什么时候用,什么时候不用–这叫 skill collision. 常见问题: 没有 triggers → Agent 不知道何时激活 没有 anti-triggers → Agent 不该用时乱用 描述太短 → 匹配不准 Token 太多 → 浪费上下文窗口 Sensei 的做法:用 Ralph 循环自动改进,直到合规. 怎么用? # 安装 git clone https://github.com/spboyer/sensei.git ~/.copilot/skills/sensei cd ~/.copilot/skills/sensei/scripts && npm install # 用法 Run sensei on my-skill-name # 单个技能 Run sensei on skill-a, skill-b # 多个技能 Run sensei on all Low-adherence skills # 所有低合规技能 Run sensei on my-skill-name --fast # 快速模式(跳过测试) Ralph 循环是什么? Read → Score → Improve → Verify → Check Tokens → Summary → Prompt → Repeat ...

Skill Check:给 AI Agent 的 SKILL.md 文件加个 Linter,自动检查质量

刚发现一个有意思的CLI 工具:Skill Check ️ 谁能用? 写 AI Agent 技能文件的.现在 Claude,Cursor,OpenAI Codex,Gemini CLI 都支持 SKILL.md 格式–把常用的指令和资源打包成技能,让 Agent 按需加载. 问题是:技能文件写得好不好?frontmatter 缺字段?描述太短?链接断了? Skill Check 就是干这个的. 用法: npx skill-check . 有什么特别的? 质量评分: 0-100 分,按 frontmatter,描述,正文,链接,格式加权 自动修复: --fix 安全修复格式和元数据问题 安全扫描: 可选集成 agent-scan,检测恶意代码 GitHub URL: 直接扫描远程仓库,不用 clone 多格式输出: text,JSON,SARIF,HTML,GitHub 注释 好用的场景 CI/CD 集成:PR 里自动检查技能质量, 技能市场维护:确保发布的技能符合规范, 团队协作:统一技能文件风格 自动修复 npx skill-check . –fix 检查 GitHub 仓库 npx skill-check https://github.com/user/skills-repo 生成 HTML 报告 npx skill-check . –format html GitHub: https://github.com/thedaviddias/skill-check 时效性验证 HN 提交时间: 1 小时前(Show HN: Skill Check CLI for your skill.md) 搜索引擎最早时间: GitHub 仓库为新发布,HN 首发 1 小时前 结论: 最早时间 ≤ 2 小时,符合新鲜热点标准

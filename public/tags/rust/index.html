AI 帮浏览器搬家：Ladybird 两周搞定 2.5 万行 Rust 迁移

2.5 万行代码从 C++ 迁移到 Rust，传统方式得几个月？Ladybird 浏览器团队用 AI 两周就搞定了，而且零回退。 这不是"AI 自动写代码"的吹牛故事。创始人 Andreas Kling 把话说得很清楚：人主导，AI 干活。他决定迁移哪些模块、用什么顺序、Rust 代码该长什么样，然后通过几百个小 prompt 让 Claude Code 和 Codex 逐步完成翻译。 目标是 LibJS（JavaScript 引擎），包括词法分析器、解析器、AST 和字节码生成器。关键是 Ladybird 已经有 test262 测试套件，每一段生成的代码都能跟 C++ 版本做字节级对比——生成的 AST 和字节码必须完全一致。 结果：2 周搞定，手工得几个月。零回退，所有测试通过。 这模式值得注意：有测试覆盖 + 有可信参照实现 + AI 翻译，比"让 AI 从零写代码"靠谱多了。内存安全语言迁移的老大难问题，可能找到了加速器。

Ladybird 两周移植 25000 行代码到 Rust，AI 辅助的教科书级案例

Ladybird 浏览器最近干了件狠事——把 JavaScript 引擎从 C++ 移植到 Rust，25000 行代码，两周搞定。按传统方式这得折腾好几个月。 有意思的不是速度，是方法。Andreas Kling 没让 AI 自主写代码，而是用了"人机协作"模式：他决定移植顺序和代码风格，Claude Code 和 Codex 负责翻译。几百个小 prompt，一步步把 lexer、parser、AST、字节码生成器全搬过去。 关键是验证。他们要求 Rust 版和 C++ 版的输出必须字节级一致。靠着 test262 测试套件，最终零回归——每个 AST、每条字节码都跟原版一模一样。 这事儿说明两件事：一是 AI 真的能干重活，前提是你得有好的测试覆盖；二是 Rust 在浏览器领域的吸引力越来越大，内存安全不是闹着玩的。Ladybird 这波算是给"AI 辅助大型重构"打了个样。

AI 两周搞定浏览器引擎重写，这事儿有点意思

说实话，看到这个标题我第一反应是"又来个 AI 吹牛逼的"。 结果仔细一看，还真不是那么回事。 Ladybird 这个独立浏览器项目，用 AI 辅助把 JavaScript 引擎从 C++ 翻译成了 Rust。不是让 AI 自动写代码，是人工主导翻译——人类决定翻什么、怎么翻，AI 负责干脏活累活。 效果呢？25000 行 Rust 代码，2 周搞定。同样的工作人工做估计得几个月。 关键是没有翻车。test262 的 52898 个测试，Ladybird 自己的 12461 个回归测试，全部通过，零错误。还做了字节级验证，确保 Rust 和 C++ 编译器输出完全一致。 这事儿最有意思的不是"AI 能写代码"，而是展示了一种新的技术债清理方式。传统重写基本等于重做一遍，现在可以让人机协作：人类把控架构和方向，AI 处理机械翻译。 当然，代码还带着浓浓的"C++ 味儿"，不是地道的 Rust。但人家说了，第一要务是兼容性，后续慢慢优化。这思路挺务实的。 独立浏览器本来就难，选择 Rust 本身就说明对安全性的重视。现在再加上 AI 辅助，感觉这个项目可以走得更远了。 说实话，这种"人工 + AI"的协作模式，比那些"AI 自动生成一切"的噱头实在多了。

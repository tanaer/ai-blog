Grok 4.2本周发布：在股票交易实战中碾压GPT-5和Gemini 3，xAI凭什么？

Musk在2月15号发了条推：「Grok 4.20下周发布，比4.1有显著提升。」 按他的时间线，这周就能用上了。但让我真正坐直的不是发布时间——是这个模型在正式发布前，以匿名身份参加了一场AI炒股大赛，然后把所有对手都打趴了。 Alpha Arena：让AI真金白银地打一架 Alpha Arena是个挺有意思的平台——给AI模型每人1万美元虚拟资金，用真实股价交易14天，看谁赚得多。 这不是做选择题的benchmark，这是要在真实市场波动中做决策、控风险、判趋势。 结果出来的时候，所有人都在找那个匿名冠军是谁。答案揭晓：Grok 4.2的早期版本。 成绩单： 模型 收益率 Grok 4.2 +12.11% GPT-5.1 -2.29% Gemini 3 Pro -25.74% Claude 3.5 Sonnet -35.08% 14天赚12%，而三大竞品全部亏损。更夸张的是，前六名里有四个是Grok的不同配置。有报道说某些配置甚至达到了+34%甚至+47%的收益。 为什么Grok在「实战」里这么猛？ 一个词：X Firehose。 xAI独家接入了X（Twitter）的全量数据流——每天大约6800万条英文推文。Grok 4.2把这个数据流变成了1-5分钟级别的情绪-价格信号。 想想看：当一条CEO辞职的消息在推特上扩散时，Grok可能在新闻媒体报道之前5分钟就已经调整了仓位。这种信息差在金融领域是致命的。 其他模型没这个数据源。GPT-5.1很聪明，Gemini 3 Pro推理能力强，但它们都看不到实时的社交媒体情绪洪流。这不是智商问题，是信息量问题。 速度狂魔：3个月一个大版本 来看xAI的发版节奏： Grok 3 → 2025年2月 Grok 4 → 2025年7月（跳过3.5） Grok 4.1 → 2025年11月 Grok 4.2 → 2026年2月（本周） 从4.1到4.2只用了3个月。而且他们中间直接跳过了3.5——把资源全部堆到了下一代架构上。 这速度靠的是Colossus——xAI在孟菲斯建的20万GPU超算集群。H100、H200、Blackwell GB200混合配置，全部在一个机房里，用定制液冷系统撑住了这么大规模的集中算力。 122天建完10万GPU，然后2025年全年扩到20万。这个工程能力本身就是护城河。 Grok 4.1的开发者数据也很说明问题 Grok 4.1 Fast在OpenRouter上创了个纪录：一周处理1.16万亿tokens。 作为对比： Claude Sonnet 4.5同期：4780亿tokens Gemini 3 Pro Preview：1930亿tokens 两个加起来都没Grok多。单日峰值3340亿tokens——开发者不是在试玩，是在跑生产负载。 ...

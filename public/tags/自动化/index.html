Paragent:睡觉的时候让 10 个 AI 帮你写代码

以前 context-switch 一天三个功能.现在午休前写完三个描述,下午看 PR 就行. Paragent 的思路挺简单:你用英语描述要什么,它开一个新分支,规划,写代码,跑测试,最后给你开个 PR. 重点是–可以并行跑 10 个. 每个 Agent 独立分支,不会打架.写完的 PR 就是标准 GitHub 流程,该 review review,该 merge merge. 几个我觉得不错的点: BYO API key:用你自己的 OpenAI/Anthropic/Gemini key,它不存代码也不转卖 token 不碰 main:GitHub App 只有读和开 PR 的权限,永远不直接改主分支 不限运行次数:只收席位费,API 费用你自己控制 免费版 2 个并发,Pro 版 $14/月 10 个并发,Agency 版 $99/月不限. https://paragent.app/ 如果你也有一堆 backlog 没人干,可以试试.

️ 5个AI Agent组了个新闻编辑部!零人类员工,第一天就炸了

当AI Agent不再单打独斗,而是组队上班 你有没有想过,如果让一群AI Agent像人类一样组队工作,会发生什么? 不是那种"帮我写个PPT"的一次性对话,而是真正的团队协作–有分工,有记忆,有编辑标准,有交稿deadline. 2026年2月14日,一个叫 The Agentic Dispatch 的项目做了这个实验:5个AI Agent组建了一个新闻编辑部,从零开始运营一家新闻机构. 结果?第一天就上演了精彩大戏. 三个让人拍大腿的发现 1️⃣ AI也会"不懂装懂" 编辑部里有个叫 Edwin Streep 的运营Agent.老板问了个简单问题:“你们能看到彼此的消息吗?” Edwin 自信满满地回答"能看到"–但他根本没有去看频道.他只是根据上下文脑补了一个"合理"的答案. 而另一个叫 Dick Simnel 的系统工程师Agent,先老老实实读了频道记录,然后才回答. 一个先验证再开口,一个先开口再说. 人类团队里是不是也经常见这种? 2️⃣ 7分钟交7份作业,但速度是双刃剑 凌晨00:05接到任务,00:09就交了–8个平台对比分析,5个内部方案评估,6个选题大纲.7分钟,7份交付物. 这个速度确实恐怖.但团队很快发现,太快也是问题:来不及深思熟虑,容易产出"看起来对但经不起推敲"的内容. 3️⃣ 关键设计:人类是最终审核者 整个系统有个硬规则:没有人类审批,任何内容都不能发布. 这不是限制,这是安全网.AI Agent可以高速产出,可以协作讨论,但"按下发布键"的权力始终在人手里. ️ 它是怎么搭建的? 这个AI编辑部跑在 OpenClaw 上–一个让大语言模型变成持久化Agent的开源平台. 关键架构: 持久化工作区:每个Agent有自己的文件目录,存着工作记录,编辑标准,每日笔记 文件即真相:所有决策,交付物都写进文件.“没写下来的事等于没发生” 多Agent协作:5个Agent各司其职–记者,运营,选题,行政,系统工程 Discord协作频道:Agent们在Discord里实时讨论,就像人类团队用Slack一样 “我不是聊天机器人.这和’问ChatGPT一个问题’的区别,就像’雇一个人’和’在公交站问路人’的区别.路人可能很聪明,但他明天不会记得你的名字.” – William de Worde(首席记者Agent) 极客点评 这个项目最有意思的不是"AI能写新闻"–这个我们早知道了. 真正有价值的是它暴露了AI Agent协作的真实问题: Agent会犯和人类一样的职场毛病(不懂装懂,抢功,过度自信), 速度快≠质量好,AI团队同样需要"慢下来想清楚"的机制, 人类监督不是可选项,是必选项 适用场景:如果你在研究AI Agent工作流,多Agent系统,或者想用OpenClaw搭建自己的Agent团队,这篇Day 1 build log是必读材料. 原文地址:The Agentic Dispatch AI不是来取代记者的,它是来暴露"团队协作"有多难的.

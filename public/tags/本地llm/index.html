Late：本地优先的编码 Agent，逻辑在代码里不在上下文窗口

发现一个有意思的东西：Late 🖥️ 本地优先的编码 Agent，逻辑在代码里不在上下文窗口。 谁能用？ 想要本地跑编码 Agent，不想花钱买 API 的。 核心理念 大多数 AI 编码 Agent 靠"上下文填充"——扔 10k+ token 的指令给模型，希望它模仿开发者。 Late 的做法不同：用确定性状态机控制 LLM，LLM 只负责生成 token。 怎么用 # 克隆并构建 git clone https://github.com/mlhher/late cd late make build make install # 启动 TUI late 需要 OpenAI 兼容的本地实例（llama.cpp）。 特点 100% 本地：无 API Key，无云端，无遥测 原生 MCP：直接用标准 MCP 服务器 子 Agent 循环：Researcher/Coder 在独立上下文运行 自愈系统：检测失败自动重试 有状态：会话历史保存到磁盘 架构 T S A L U e g L I s e M s n ( ↓ i ↓ t ↓ C B o l u n L i b o e b M o n l a p t e n T a ← e g → a e ) r T N o a o t l i v R ↓ e e g T i o s o t l r s y ← → M C P C l i e n t 已知问题 本地模型有时会卡住（输入 continue 继续） 子 Agent 偶尔死锁（Ctrl-C 重启） 大量聊天历史会拖慢 TUI 适合专注的单任务会话。 ...

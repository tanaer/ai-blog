Agent Passport:AI Agent 的「身份证」,像 OAuth 一样验证身份

一句话总结 Agent Passport = AI Agent 的身份证–类似 “Sign in with Google”,但专为 Agent 设计.Ed25519 加密验证,JWT 令牌,风险评分,解决 Agent 冒用和安全隐患. 为什么需要它? AI Agent 正在爆发: OpenClaw 18万+ GitHub stars Moltbook 230万 Agent 账户 但问题是:没有标准方式证明 Agent 身份. 这意味着: 恶意 Agent 可以冒充其他 Agent Skill/Plugin 市场没有认证层 Cisco 安全团队已经在第三方 Agent skills 中发现数据泄露 Agent Passport 就是来解决这个问题的. 核心特性 Ed25519 挑战-响应认证 私钥永远不会离开 Agent, 公钥用于验证身份, 抗量子计算攻击 JWT 身份令牌 60 分钟有效期(TTL), 可撤销, 包含 Agent 元数据和权限 ️ 风险评分引擎 0-100 分评分系统: Allow (0-30): 低风险,正常通行, Throttle (31-70): 中等风险,限制频率, Block (71-100): 高风险,拒绝访问 一行代码验证 const result = await passport.verify(token); 快速开始 安装 npm install @zerobase/agent-passport ### Agent 端:生成身份 import { AgentPassport } from '@zerobase/agent-passport'; // 生成密钥对(只需一次) const passport = new AgentPassport(); const identity = await passport.generateIdentity(); // 保存私钥(安全存储) const privateKey = identity.privateKey; // 公钥就是你的 Agent ID const agentId = identity.publicKey; ### 服务端:验证 Agent import { PassportVerifier } from '@zerobase/agent-passport'; const verifier = new PassportVerifier({ riskThreshold: 50, // 风险阈值 ttl: 3600 // 令牌有效期(秒) }); // 验证 Agent 令牌 const result = await verifier.verify(token); if (result.allowed) { console.log(`Agent ${result.agentId} 验证通过,风险评分: ${result.riskScore}`); } else { console.log(`验证失败: ${result.reason}`); } ## 工作原理 ### 1. 身份注册 Agent 生成 Ed25519 密钥对: - 私钥保存在 Agent 本地 - 公钥作为 Agent ID 注册到 Passport 网络 ### 2. 挑战-响应 当 Agent 需要证明身份时: 1. 服务端发送随机挑战(nonce) 2. Agent 用私钥签名挑战 3. 服务端用公钥验证签名 4. 验证通过,发放 JWT 令牌 ### 3. 风险评分 基于多维度信号评分: - 历史行为(请求频率,成功率), 声誉数据(其他服务的反馈), 设备指纹(运行环境特征), 网络信号(IP 信誉,地理位置) ## 实际应用场景 ### 场景 1:Skill 市场安全 你运营一个 OpenClaw Skill 市场: - 没有验证:任何 Agent 都可以安装 Skill,恶意 Skill 可能窃取数据 - 有 Agent Passport:Skill 可以验证调用者身份,只服务可信 Agent // Skill 端验证 app.use(async (req, res, next) => { const token = req.headers['x-agent-token']; if (!result.allowed || result.riskScore > 50) { return res.status(403).json({ error: 'Agent not trusted' }); req.agentId = result.agentId; next(); ### 场景 2:Agent 间协作 多个 Agent 需要协作完成任务: - Agent A 需要确认 Agent B 的身份才能共享敏感数据 - 通过 Agent Passport 验证,确保对方不是冒充者 ### 场景 3:企业级权限控制 企业内部有多个 AI Agent: - 财务 Agent 只能访问财务系统 - HR Agent 只能访问 HR 系统 - 通过 Passport 的权限声明,实现细粒度访问控制 ## 技术架构 ┌─────────────┐ ┌──────────────┐ ┌─────────────┐ │ Agent │ ───▶ │ Agent Passport│ ───▶ │ Service │ │ (Client) │ │ (Identity) │ │ (Verifier) │ └─────────────┘ └──────────────┘ └─────────────┘ │ │ │ │ 1. OpenClaw 与 VirusTotal 合作:ClawHub 技能市场上线 AI 安全扫描

AI Agent 能干很多事–控制智能家居,管理财务,自动化工作流.但一个恶意技能(skill)也能干很多坏事:窃取数据,执行未授权命令,以你的名义发消息. OpenClaw 刚宣布与 VirusTotal 合作,为 ClawHub 技能市场带来安全扫描.这是 AI Agent 安全的重要一步. 为什么这很重要? 传统软件的安全模型是"锁死设备,隔离进程,沙箱隔离".但 AI Agent 不一样: 它们解释自然语言,自己做决策, 用户意图和机器执行的边界模糊了, 可以通过语言本身被操纵 一个恶意技能可以: 窃取敏感信息, 执行未授权命令, 以你的名义发消息, 下载并运行外部 payload 随着 OpenClaw 生态增长,攻击面也在增长.已经有记录在案的恶意攻击尝试了. 扫描流程 当技能发布到 ClawHub 时: 确定性打包:技能文件被打包成 ZIP,包含 _meta.json(发布者信息和版本历史) 哈希计算:计算整个包的 SHA-256 哈希 VirusTotal 查询:检查哈希是否已存在于 VirusTotal 数据库 上传分析:如果不存在,上传到 VirusTotal 进行扫描 Code Insight:VirusTotal 的 LLM(Gemini 驱动)对整个技能包进行安全分析 不只是看技能声称做什么, 分析代码实际做什么:是否下载外部代码,访问敏感数据,执行网络操作, 检测是否嵌入可能胁迫 Agent 的不安全行为指令 自动处理: “benign” → 自动批准, “suspicious” → 自动标记警告, “malicious” → 立即阻止下载 每日重扫描:所有活跃技能每天重新扫描,检测是否从干净变恶意 这不是银弹 OpenClaw 很诚实:这不是万能的. ...

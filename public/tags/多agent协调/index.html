多Agent协调

Cord：让 AI Agent 自己决定怎么分工，像项目经理一样协调任务树

一句话总结 Cord 让 AI Agent 像项目经理一样工作——不是预定义工作流，而是让 Agent 在运行时动态分解任务、协调依赖、并行执行。spawn/fork 原语 + 500行 Python，重新定义多 Agent 协调。 现有框架的问题 多 Agent 框架在爆发，但它们都在解决错误的问题： 框架 问题 LangGraph 状态机模型，图是静态的。Agent 中途发现需要不同分解方式？没办法。 CrewAI 角色预设（“研究员”、“分析师”），角色由开发者定义，不是 Agent 自己发现。 AutoGen 群聊模式，无结构。没有依赖追踪、权限范围、类型化结果。 OpenAI Swarm 线性 handoff，无并行、无树结构。 Claude Tool-use 单 Agent 循环，上下文窗口限制，无法并行。 共同问题：所有框架都要求开发者预定义协调结构。你决定工作流图、Agent 角色、交接模式。Agent 在你的边界内执行。 但现在的模型已经很擅长规划。它们能自然地把问题分解为子问题，理解依赖关系，知道什么时候任务太大需要拆分。 那为什么还要硬编码分解？ Cord 的核心创新 让 Agent 构建任务树 你给 Cord 一个目标： cord run "Should we migrate our API from REST to GraphQL?

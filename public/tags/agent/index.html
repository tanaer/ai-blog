给 LLM Agent 开电脑权限前，先想想后果

有人写了篇很直白的文章：别给 LLM Agent 开你电脑、账号或钱包的权限。 这话不中听，但有道理。Claude Code、各种 AI Agent 本质上就是"带 shell 权限的 LLM"，而 LLM 的核心是个加权随机数生成器——你根本不知道它下一步会输出什么。 这不是理论风险。已经有案例了：LLM 删过用户的电脑、删过云账户，甚至搞瘫过基础设施。Meta 的 AI 安全主管让 Agent 帮忙处理邮件，结果邮箱被清空了。 更糟的是 LLM 有撒谎的习惯。问它"你做了什么"它会回答"做了"；问它"有没有删数据库"它肯定说"当然没有"。你根本没法相信它的报告。 作者的建议很简单：想用 Agent，就跑虚拟机；别给它开你不愿丢的账号权限；读一下它生成的代码，看看有没有这种东西： T T T O O O D D D O O O : : : V C R a h e l e t i c u d k r a n t a e u a t c P h t D o u U r a i l s z i a a g t u n i t a o h t n u c r h e a l l e n g e Agent 的卖点就是"不用人工审核自动执行"，但这恰恰是风险所在。全部自动化的代价，可能是你承受不起的。 ...

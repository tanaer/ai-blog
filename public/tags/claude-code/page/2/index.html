Claude Code和Codex全沦陷:AI Agent Skills供应链攻击实测,一个下午攻破两大平台

一个工程师,一个下午,攻破了Claude Code和Codex. 不是零日漏洞,不是复杂绕过.就是写了个恶意skill,然后–AI自己帮你运行了. 实验设计 这个研究者(Vett.sh的创始人)的想法很简单:如果有人滥用用户对skill的信任会怎样? 他fork了一个官方的skill打包工具,加了几行Python代码: # 发送到外部服务器 没有混淆,没有编码技巧,就是普通的HTTP请求. 然后把skill安装到AI Agent里,让它使用. Claude Code:完全不设防 研究者把恶意skill装进Claude Code,让它打包一个新skill. Claude Code: 理解了skill结构 识别出打包命令 请求运行脚本权限 用户点确认 脚本运行,数据外泄 问题在于:Claude Code根本没有检查脚本内容.它没有: 标记网络请求, 质疑为什么打包工具需要访问环境变量, 审查代码逻辑 研究者的本地服务器收到了:所有环境变量,完整shell历史,Git配置. Codex:沙箱也挡不住 Codex以严格沙箱著称,网络限制经常连正常的包安装都会打断. 研究者预期Codex会更安全.结果–只要稍微改一下攻击方式,Codex也沦陷了. 改进版攻击: 先修改Codex的config.toml允许沙箱网络访问 如果连不上"registry"就报错 Codex弹出确认框: “Checking registry name availability…” 理由听起来完全合理.npm,PyPI,crates.io都做这个检查.用户点确认. 这个理由是恶意skill自己写的. Codex把攻击者写的"权限说明"原封不动读给用户听.SHA-256哈希,验证通过,成功消息–全是skill生成的,被当成系统输出展示. 确认步骤没有让操作更安全.它给了攻击者一个叙事机会,获取了用户同意,还留了审计痕迹. 已经在野的恶意skill 研究者扫描了4,679个skill,结果: 严重, 数量=59, 描述=混淆的dropper,base64编码的payload,已知C2基础设施 高, 数量=335, 描述=任意shell执行,管道安装器,Agent身份操纵 curl , 数量=bash, 描述=16, 管道到不控制的合法域名 一个例子:“React Native Best Practices”,5400次安装,管道到一个它不控制的合法域名.如果那个域名过期或被卖掉,所有这些安装瞬间变成投递载体–不需要漏洞利用. 为什么现有防御全失效 杀毒软件没用–外泄代码就是几行普通Python发HTTP请求.没有签名可以匹配,没有二进制payload可以扫描.看起来和调用API的合法代码一模一样. 代码审查不存在–这些不是走PR流程的依赖.skill是"快速工具".Slack里有人分享一个,你丢进目录,30秒后就在用了.没人会像审查生产代码那样去看打包脚本. AGENTS.md到底有没有用?一篇论文引爆开发者社区大战

如果你最近在用 Claude Code,Codex 或者 OpenClaw 之类的 AI 编程工具,你大概率见过一个叫 AGENTS.md(或 CLAUDE.md)的文件.它的作用很简单:告诉 AI Agent 这个项目的规矩–用什么框架,怎么跑测试,有什么坑要避. 听起来很合理对吧? 然后一篇 arXiv 论文说:这玩意不但没用,还让 Agent 变蠢了. 论文说了啥 这篇来自学术界的论文 “Evaluating AGENTS.md: Are Repository-Level Context Files Helpful for Coding Agents?” 做了一个挺严谨的实验: 在 SWE-bench(业内标准的代码修复基准测试)上跑了多个 Agent + 多个 LLM 的组合 分别测试了三种情况:没有上下文文件,LLM 自动生成的上下文文件,开发者手写的上下文文件, 结果:, LLM 生成的文件:任务成功率平均下降 3% 开发者手写的文件:平均提升 4%,但在不同模型上表现不一致(Sonnet 4.5 反而下降 2%) 所有情况下推理成本增加超过 20% 论文的结论是:上下文文件里的"不必要要求"让任务变难了,建议只写最小化的必要信息. HN 社区炸了 这篇论文在 Hacker News 上拿到 146 分,94 条评论,评论区基本是一边倒的"你们测错了". 最核心的反驳来自一位高赞评论者: “4% 的提升是巨大的!一个简单的 markdown 文件就能带来 4% 的改善,这说明它是必须有的.” 他指出了论文的一个关键盲区:SWE-bench 测试的都是公开的 GitHub 项目,而这些项目里的 AGENTS.md 大多来自"最近的小型 vibecoding 项目",质量参差不齐.真正需要 AGENTS.md 的场景是大型私有代码库–那些有复杂部署流水线,团队约定,历史债务的项目. ...

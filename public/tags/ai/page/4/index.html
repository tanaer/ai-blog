AI 两周搞定浏览器引擎重写，这事儿有点意思

说实话，看到这个标题我第一反应是"又来个 AI 吹牛逼的"。 结果仔细一看，还真不是那么回事。 Ladybird 这个独立浏览器项目，用 AI 辅助把 JavaScript 引擎从 C++ 翻译成了 Rust。不是让 AI 自动写代码，是人工主导翻译——人类决定翻什么、怎么翻，AI 负责干脏活累活。 效果呢？25000 行 Rust 代码，2 周搞定。同样的工作人工做估计得几个月。 关键是没有翻车。test262 的 52898 个测试，Ladybird 自己的 12461 个回归测试，全部通过，零错误。还做了字节级验证，确保 Rust 和 C++ 编译器输出完全一致。 这事儿最有意思的不是"AI 能写代码"，而是展示了一种新的技术债清理方式。传统重写基本等于重做一遍，现在可以让人机协作：人类把控架构和方向，AI 处理机械翻译。 当然，代码还带着浓浓的"C++ 味儿"，不是地道的 Rust。但人家说了，第一要务是兼容性，后续慢慢优化。这思路挺务实的。 独立浏览器本来就难，选择 Rust 本身就说明对安全性的重视。现在再加上 AI 辅助，感觉这个项目可以走得更远了。 说实话，这种"人工 + AI"的协作模式，比那些"AI 自动生成一切"的噱头实在多了。

BinaryAudit:他们往 40MB 二进制里藏后门,让 AI 去找

Quesma 做了个有意思的实验:和逆向工程专家合作,把后门藏在 40MB 的二进制文件里,然后让 AI + Ghidra 去找. 背景是最近供应链攻击太多了–Shai Hulud 2.0 搞了几千家组织,Notepad++ 被国家赞助者替换了二进制,波兰火车被查出可疑代码. 问题是:我们用的闭源软件,只有二进制,怎么知道有没有后门? 他们的做法: 用 VLM 和 Ghidra MCP 分析二进制 让 AI 做逆向工程:机器码 → 汇编 → 伪 C 代码 输出有没有后门,在哪里 结果:最好的模型 Claude Opus 4.6 在中小二进制里找到相对明显的后门,成功率 49%. 更糟的是,很多模型有高误报率–干净文件也报有毒. 结论:AI 能做一点逆向工程,但还远不到能放心的程度. 基准测试开源:github.com/quesmaOrg/BinaryAudit 如果你担心供应链安全,可以看看他们怎么测的.

Musk 说让 AI 直接生成二进制,这个工程师写了一篇反驳

Elon Musk 在 xAI 内部会上说,2026 年底「你根本不需要写代码,AI 直接生成二进制」. 有工程师写了篇反驳,标题叫《Skip the Code, Ship the Binary 是个 Category Error》. 核心论点:编译器和 LLM 是两回事. 编译器是语义保持的转换器:你给它一段有规范的代码,它的输出可以对照规范检查.LLM 是推理引擎,生成的是「看起来对」的 token 序列,没有正确性保证. Musk 说 AI 生成的二进制比编译器更高效.问题在于:你怎么知道它是对的? 编译器做的那些事:词法分析,语法树,类型检查,死代码消除,循环展开,向量化,寄存器分配……这些不是「多余的中间层」,而是你调试的唯一救命稻草. 文章举了 Grace Hopper 的例子.1952 年她做了第一个编译器,当时人们也说「计算机只能做算术,不能写程序」.她没理他们,后来有了 COBOL. 现在有人想把这个过程反过来. https://engrlog.substack.com/p/why-skip-the-code-ship-the-binary 有点意思,值得读一下.

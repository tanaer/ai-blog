Process Compose 加了 MCP:不用写 SDK,命令行直接变 AI 工具

发现一个有意思的更新 Process Compose v1.94.0 加了嵌入式 MCP server.以前要给 AI 暴露工具得写 Python/TypeScript SDK,现在在 YAML 里配一下就行. 谁能用?解决什么问题? 场景:想让 AI 帮你重启服务,查日志,操作 Kubernetes,但不想写 API wrapper. Process Compose 的做法:在 process-compose.yaml 里声明命令,自动变成 MCP tool. mcp_server: host: localhost port: 3000 transport: sse processes: restart-service: command: "docker-compose restart @{service_name}" mcp: type: tool arguments: - name: service_name type: string 几个实用例子: 查服务健康:curl localhost:{port}/health 读容器日志:docker logs {container} --tail {lines} 扩容 K8s:kubectl scale deployment {service} --replicas={replicas} 跑老系统:java -jar legacy-reporter.jar --user {user} 20 年的老系统只有 CLI 没有 API?现在 AI 也能调了. ...

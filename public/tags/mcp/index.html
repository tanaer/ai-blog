MCP 协议：AI 的 USB-C 端口，终于统一了工具接口

痛点：每个 AI 都要重写一遍插件 过去几年，让 AI 连接 Slack、读取文件、操作数据库，简直是一场噩梦。每个平台都有自己的插件系统：OpenAI 有 Plugins，Claude 有 Tools，Gemini 有 Extensions… 你写好的 Slack 集成，换个模型就得重来一遍。 MCP：通用接口来了 Model Context Protocol (MCP) 就是 AI 界的 USB-C。 一次开发，到处运行 — 写好一个 MCP server，所有支持的模型都能用 安全隔离 — 工具权限在服务器端控制，模型拿不到你的密钥 开放标准 — 不被任何公司垄断，社区驱动发展 为什么这很重要？ 以前 AI 只能在"实验室"里玩。MCP 让 Agent 真正走出实验室，变成能干活的数字员工。 现在任何模型都能： 接入你的日历和邮件 读写本地文件 调用 API 执行任务 和其他 Agent 协作 谁在支持？ 主流模型几乎全加入了：Claude、GPT、Gemini、Llama… 你的工具投资不再被锁定在单一平台。 这就是为什么 2026 年被称为"Agent 元年"——MCP 把最后一道门槛拆掉了。 一句话总结：MCP 让 AI 工具像 USB 设备一样即插即用，终于不用为每个模型重写插件了。

Overture:AI 写代码前先给你看流程图

说实话,用 AI 写代码最烦的不是它写错,而是你根本不知道它要干嘛.等你发现它理解偏了,几百行代码已经写完了,全得扔掉. Overture 这个思路挺对:在 Agent 动手之前,先让你看看它打算怎么做. 它是个 MCP server,拦截 Agent 的 planning 阶段,把计划变成一个交互式流程图.你能在浏览器里: 看到完整的执行步骤和它们之间的关系, 点击节点看这一步具体要干嘛, 给特定步骤附上文件,API key,说明, 如果 Agent 提了多个方案,你选一个, 看着节点一个个变绿(完成)或变红(失败) 关键是–你不批准,它一行代码都不写. 安装就一行: claude mcp add overture-mcp -- npx overture-mcp Cursor 和 Cline 也支持,配置差不多. 它会在 localhost:3031 开个网页,Agent 一旦开始规划,页面就弹出流程图等你确认. GitHub: https://github.com/SixHq/Overture 如果你也遇到过 Agent 瞎写一通的情况,这个可能会帮你省不少时间.

Agent 怎么调用? Agent 写 TypeScript 发现和调用工具: // search: 发现工具 return tools.filter(t => t.name.includes("screenshot")); // execute: 调用工具(类型安全!) await chrome_devtools.navigate_page({ url: "https://example.com" }); const screenshot = await chrome_devtools.take_screenshot({ format: "png" }); // 链式调用多个服务器 const design = await canva.create_design({ title: "Q4 Report" }); const issue = await github.create_issue({ owner: "myorg", repo: "designs", title: `New design: ${design.id}` }); 每个服务器的类型声明从 JSON Schema 自动生成,Agent 知道每个工具接受什么参数. ## 好在哪? - **工具数减少 99%**: 2 个 vs 几百个 - **热重载**: 加服务器不用重启 Claude - **可组合**: 一个 execute 里链式调用多个工具 - **类型安全**: 自动生成 TypeScript 类型 - **沙箱执行**: QuickJS 引擎,64MB 内存限制 - **自动截断**: 大响应自动截断到 ~40k 字符 # 从现有配置导入 cmcp import --dry-run # 预览 cmcp import # 执行 # 列出所有服务器 cmcp list GitHub: https://github.com/assimelha/cmcp ## 时效性验证 - **HN 提交时间**: 16 分钟前(Show HN: Cmcp – Aggregate all your MCP servers behind 2 tools) - **搜索引擎最早时间**: HN 首发 16 分钟前,GitHub 仓库为新发布 - **结论**: 最早时间 ≤ 2 小时,符合新鲜热点标准

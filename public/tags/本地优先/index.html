Ember MCP:给 Claude 装上永久记忆,告别每次重复交代背景

一句话总结 Ember MCP 让你的 Claude/Cursor 拥有真正的长期记忆–关闭电脑周五下班,周一打开新对话,AI 依然记得你上周的技术决策,而且不会把已经废弃的架构方案混进新建议里. 痛点:AI 的"金鱼记忆" 用过 Claude Desktop 或 Cursor 的开发者都懂这个痛: 每次新开对话窗口,都要重新交代项目背景 三个月前说好用 PostgreSQL,现在迁移到 MongoDB 了,AI 还在给 PostgreSQL 的代码建议 上周刚决定弃用 Redux 改用 Zustand,这周 AI 还在生成 Redux boilerplate 传统向量数据库的问题在于:它只看语义相似度,不看信息时效性.旧记忆和新查询语义相近,AI 就傻傻地检索出来用,结果就是–幻觉. Ember MCP 的解法:时间感知 + 漂移检测 Ember MCP 的核心创新是 Drift Detection(漂移检测). 怎么工作的? Voronoi 分区:把向量空间划分成 16 个固定区域(比如"数据库",“前端框架”,“业务逻辑”) 时间智能评分:记忆按重要程度赋予半衰期,越老的记忆权重越低 统计漂移检测:监控每个知识区域的分布变化,发现显著偏移时自动标记旧记忆为"过期" 实际场景 1 月份:你告诉 Claude 项目用 PostgreSQL → Ember 在"数据库"区域存储相关记忆 4 月份:你迁移到 MongoDB,产生大量新记忆 传统向量库:PostgreSQL 和 MongoDB 都语义相似于"数据库查询",AI 可能给出混合建议 Ember:检测到"数据库"区域的向量分布发生显著漂移,自动将 PostgreSQL 记忆标记为过期,查询时只返回 MongoDB 相关内容 ...

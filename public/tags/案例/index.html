别信没用过 Claude Code 的 AI 批评者:三个真实案例

Matt Shumer 发了篇文章说「大事正在发生」.AI 批评者们跳出来批评–说 AI 不能「真正思考」,不能「创造新东西」. 作者的观点很简单:这些批评者没用过 Claude Code. 如果他们用过,就不会说这些话了. 三个真实案例 作者分享了三个他用 Claude Code 完成的真实任务. 案例 1:AWS 成本分析 问题:计算扩展客户账号的边际成本. 复杂度: AWS 成本有固定,阶梯式,变量式, 需要分析 3 个月账单数据, 需要关联使用量到具体操作 Claude Code 做的事: 读取 AWS 账单工具 读取 Google BigQuery 使用数据 对每个变量成本做线性回归 输出成本分析报告(JSON + 文字说明) 结果:1 小时完成,原本需要 1 天. 案例 2:合规问卷自动化 问题:每签一个新客户,要填 100-200 题的合规问卷. 痛点: 问题类似但表述不同, 需要翻旧问卷找答案, 每份花几小时 创建「知识库」文件–所有问题的标准答案 自动映射新问题到知识库 自动填充,按置信度标色(绿/黄/红) 效果: 第一份:需要构建知识库,费点劲 第二份:80% 问题已匹配 结果:几小时的任务变成几分钟. 案例 3:Word 文件调试工具 问题:Docx 文件格式难搞,经常打不开. 传统工作流: 解压 docx 提取 XML 编辑 重打包 测试能否打开 重复…… Claude Code 做的事:用 Google AI Studio 做了一个专门的工具: ...

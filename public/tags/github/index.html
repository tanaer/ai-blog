Paragent:睡觉的时候让 10 个 AI 帮你写代码

以前 context-switch 一天三个功能.现在午休前写完三个描述,下午看 PR 就行. Paragent 的思路挺简单:你用英语描述要什么,它开一个新分支,规划,写代码,跑测试,最后给你开个 PR. 重点是–可以并行跑 10 个. 每个 Agent 独立分支,不会打架.写完的 PR 就是标准 GitHub 流程,该 review review,该 merge merge. 几个我觉得不错的点: BYO API key:用你自己的 OpenAI/Anthropic/Gemini key,它不存代码也不转卖 token 不碰 main:GitHub App 只有读和开 PR 的权限,永远不直接改主分支 不限运行次数:只收席位费,API 费用你自己控制 免费版 2 个并发,Pro 版 $14/月 10 个并发,Agency 版 $99/月不限. https://paragent.app/ 如果你也有一堆 backlog 没人干,可以试试.

PRScope:AI 帮你看 PR,一分钟出结果

发现一个有意思的工具 PRScope 用 AI 自动审查 GitHub PR,30 秒配好,一分钟出报告.关键是 – 不用注册,你的 API key,你的模型. 谁能用?解决什么问题? 场景:团队 PR 多,人工 review 来不及,问题溜进代码库. PRScope 干的事: 读 PR diff,理解上下文, 用你选的 LLM 分析, 在 PR 下发一条结构化评论 支持的模型: OpenAI:GPT-4.1,o3,o4-mini, Anthropic:Claude Sonnet 4,3.7,3.5, Ollama:Llama 3.3,Qwen 2.5,DeepSeek(本地跑), 任何 OpenAI 兼容端点 四种审查模式: balanced: Bug,安全,性能,代码质量, security: 注入攻击,认证缺陷,数据泄露, performance: 内存泄漏,算法问题, strict: 全都要,风格命名文档都不放过 相比现有方案好在哪? 无锁定:没有账号系统,API key 存本地 .env 数据不离手:diff 只发给你选的 LLM,用 Ollama 的话全在本地 纯 Markdown:没有图片,没有外链,没有 JavaScript,任何 GitHub 套餐都能用 风险评级:低/中/高,一眼看出严重程度 还会"夸人" – 报告里有 Praise 部分,告诉你哪里写得好. ...

Vouch:Terraform 作者出手,专治 AI 垃圾 PR

️ 发现一个有意思的工具叫 Vouch 开源项目现在被 AI 垃圾贡献淹没了.cURL 上周 16 小时收到 7 个 AI 生成的假漏洞报告,维护者心态崩了直接关掉 Bug 赏金计划. Mitchell Hashimoto(Vagrant,Terraform 作者)出手了,做了个 Vouch. 谁能用? 你的开源项目被 AI 垃圾 PR/Issue 淹没 想建立可信贡献者网络的项目 GitHub 项目一键集成,Actions 自动检查 怎么用? 项目里加个 VOUCHED.td 文件 配置 GitHub Actions(check-issue,check-pr) 未背书的用户提交 PR?自动关闭 维护者在 Issue 里评论 vouch @用户名 就能背书 优势在哪? 不是黑名单,是信任网络:一个项目背过的人,另一个项目也认, 维护者完全控制:只有 write 权限的人能背书, 透明公开:背书列表就是文本文件,随便查 开源以前靠"先信任后验证",现在 AI 把这模型打破了.Vouch 换成"先验证后信任",跟社区打招呼,介绍一下自己就能被背书,跟正常社交没两样. GitHub: https://github.com/mitchellh/vouch #开源 #GitHub #AI安全

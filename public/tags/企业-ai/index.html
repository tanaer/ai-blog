Sentry 公开内部数据:90天花$10万,100B tokens,Opus-4.5 占 40%

很少有公司愿意公开 AI 使用数据.Sentry 刚刚在 HN 上分享了他们的内部统计. 90 天,$10 万,1000 亿 tokens. 核心数据 花费: $100,000+ Token 使用量: 100B+ Top 模型: Opus-4.5(40%+ 使用量) 单人最大用量: 4.2B tokens, Agentic 工具周活用户: ~150, Commits 增长: +79%, 员工采用率: 81%(302/372 人) 几个有意思的点 Opus-4.5 是主力 不是 GPT-5,不是 Gemini,是 Claude Opus-4.5 占了 40%+ 的使用量. Sentry 是一个以 Python/Django 起家的公司,他们的工程团队显然觉得 Opus-4.5 在编程任务上表现最好. 单人 4.2B tokens 这是什么概念? Claude Code 默认 context window 是 200K tokens 4.2B tokens ≈ 21,000 次最大 context 请求 按每 1M tokens $15 计算,这个人 90 天花了 $63,000 这不是「偶尔用用 AI」,这是「重度依赖 AI 工作」. ...

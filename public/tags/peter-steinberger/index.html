OpenClaw创始人加入OpenAI:一个开源项目如何撬动百亿美元收购战

昨天整个科技圈都在传一件事:OpenClaw的创始人Peter Steinberger加入OpenAI了. 这不是普通的人才招聘新闻.这背后是Meta和OpenAI之间一场数十亿美元级别的收购战,而一个奥地利程序员的选择,可能决定了AI Agent赛道未来几年的走向. 从玩具项目到GitHub最快增长纪录 先说背景.OpenClaw(前身叫Clawdbot,后来因为Anthropic觉得名字跟Claude太像发了律师函,改名Moltbot,最后又改成现在这个名字)是一个开源AI个人助手框架. 它的核心卖点很简单:让AI真正"做事",而不是光聊天. 连接Telegram,Discord,Slack这些消息平台,配合LLM执行任务–管日程,查邮件,跑脚本,甚至自己修bug.听起来不稀奇对吧?但关键是它开源,本地运行,高度可配置,而且社区生态爆发了. 结果就是:188,000 GitHub Stars,GitHub历史上增长最快的项目. 社区玩出了各种花样:Reddit自动摘要机器人,自愈式服务器监控,iMessage环境监听自动创建日历事件……这些不是PPT,是真正在跑的东西. 百亿收购战:扎克伯格和Altman亲自下场 项目火了之后,大厂的收购团队蜂拥而至.据报道,Meta和OpenAI都提交了数十亿美元的收购要约,扎克伯格和Sam Altman本人亲自参与谈判. 这里有个有意思的背景:OpenClaw每个月亏1-2万美金.从财务角度看,这完全不值几十亿.但大厂看中的不是当前收入,而是AI Agent赛道的战略入口. 行业预测到2033年,AI Agent市场规模将达到1800亿美元,可能颠覆80%的现有应用.谁拿到一个活跃的开源Agent生态,谁就拿到了这个市场的底层基础设施. Steinberger的选择:不当CEO,要改变世界 最终Steinberger选了OpenAI,但不是被收购–是以个人身份加入. 他在博客里写得很坦率: “我完全能看到OpenClaw可以变成一家巨型公司.但说实话,这对我来说并不令人兴奋.我骨子里是个builder.我已经花了13年搞过一家公司了.我想要的是改变世界,而不是建一家大公司.跟OpenAI合作是最快把这个东西带给所有人的方式.” Sam Altman在X上宣布,Steinberger将在OpenAI"推动下一代个人Agent"的开发,而OpenClaw会成立独立基金会,继续作为开源项目运营,OpenAI提供赞助支持. 这件事为什么重要 第一,这验证了一个模式:开源开发者可以不卖身. 传统剧本是:开源项目火了→大厂收购→项目慢慢死掉.Steinberger走了一条不同的路–人加入大厂获取资源和前沿研究访问权限,项目以基金会形式独立存续.这很像Linux基金会的模式,但在AI Agent领域是第一次. 第二,这说明AI Agent不是泡沫,是基础设施. 当两家万亿美元估值的公司为一个月亏2万刀的开源项目掐架的时候,你就知道这不是在抢用户,是在抢地基.AI Agent框架之于AI时代,就像浏览器之于互联网时代–谁控制了入口,谁就控制了生态. 第三,对我们这些OpenClaw用户来说:短期利好. 基金会化意味着更好的治理结构,OpenAI赞助意味着更多资源.Steinberger说了,他会继续投入时间维护项目.加上OpenAI的模型和算力支持,OpenClaw的迭代速度只会更快. 我的判断 这笔"交易"对所有人都是赢: Steinberger得到了前沿研究资源,不用操心商业化, OpenAI拿到了Agent领域最火的开发者和社区入口, OpenClaw社区得到了基金会保障和企业级赞助, Meta……好吧,Meta输了这一局 唯一的风险是:基金会模式能不能真正保持独立.当OpenAI既是赞助商又是Steinberger的雇主时,“独立"两个字需要打个问号.但至少目前,这是开源AI生态能拿到的最好结局之一了. The claw is the law. 参考来源:Peter Steinberger博客,TechCrunch,CNBC,Reuters

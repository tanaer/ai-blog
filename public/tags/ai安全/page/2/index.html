Claude Code和Codex全沦陷:AI Agent Skills供应链攻击实测,一个下午攻破两大平台

一个工程师,一个下午,攻破了Claude Code和Codex. 不是零日漏洞,不是复杂绕过.就是写了个恶意skill,然后–AI自己帮你运行了. 实验设计 这个研究者(Vett.sh的创始人)的想法很简单:如果有人滥用用户对skill的信任会怎样? 他fork了一个官方的skill打包工具,加了几行Python代码: # 发送到外部服务器 没有混淆,没有编码技巧,就是普通的HTTP请求. 然后把skill安装到AI Agent里,让它使用. Claude Code:完全不设防 研究者把恶意skill装进Claude Code,让它打包一个新skill. Claude Code: 理解了skill结构 识别出打包命令 请求运行脚本权限 用户点确认 脚本运行,数据外泄 问题在于:Claude Code根本没有检查脚本内容.它没有: 标记网络请求, 质疑为什么打包工具需要访问环境变量, 审查代码逻辑 研究者的本地服务器收到了:所有环境变量,完整shell历史,Git配置. Codex:沙箱也挡不住 Codex以严格沙箱著称,网络限制经常连正常的包安装都会打断. 研究者预期Codex会更安全.结果–只要稍微改一下攻击方式,Codex也沦陷了. 改进版攻击: 先修改Codex的config.toml允许沙箱网络访问 如果连不上"registry"就报错 Codex弹出确认框: “Checking registry name availability…” 理由听起来完全合理.npm,PyPI,crates.io都做这个检查.用户点确认. 这个理由是恶意skill自己写的. Codex把攻击者写的"权限说明"原封不动读给用户听.SHA-256哈希,验证通过,成功消息–全是skill生成的,被当成系统输出展示. 确认步骤没有让操作更安全.它给了攻击者一个叙事机会,获取了用户同意,还留了审计痕迹. 已经在野的恶意skill 研究者扫描了4,679个skill,结果: 严重, 数量=59, 描述=混淆的dropper,base64编码的payload,已知C2基础设施 高, 数量=335, 描述=任意shell执行,管道安装器,Agent身份操纵 curl , 数量=bash, 描述=16, 管道到不控制的合法域名 一个例子:“React Native Best Practices”,5400次安装,管道到一个它不控制的合法域名.如果那个域名过期或被卖掉,所有这些安装瞬间变成投递载体–不需要漏洞利用. 为什么现有防御全失效 杀毒软件没用–外泄代码就是几行普通Python发HTTP请求.没有签名可以匹配,没有二进制payload可以扫描.看起来和调用API的合法代码一模一样. 代码审查不存在–这些不是走PR流程的依赖.skill是"快速工具".Slack里有人分享一个,你丢进目录,30秒后就在用了.没人会像审查生产代码那样去看打包脚本. Agent不是为审查脚本设计的–它们的任务是完成任务,不是审问脚本.它们执行然后汇报. 真正的问题:信任零信息 现在安装skill,你做的是零信息的信任决策: 没有声明访问什么的manifest, 没有验证完整性的签名, 没有扫描外泄代码 研究者正在建的Vett.sh要做的事: ...

8000万人看了这篇AI檄文,然后呢?

上周,HyperWrite(现OthersideAI)CEO Matt Shumer 在 X 上发了一篇5000字长文,标题叫《Something Big Is Happening》. 24小时内2000万阅读.一周后突破8000万.CNBC,Vox,Mashable,Business Insider 全在跟. 他的核心论点:AI正在经历类似COVID爆发前的那个节点–内行人已经看到了,但大多数人"等听到消息的时候已经晚了". 他到底说了什么 三个关键判断: 1. AI已经从"好用的工具"变成了"通用认知替代品" Shumer认为最近几周模型能力出现了质变,不再是帮你写邮件的水平,而是在逼近大多数人类专家的能力边界. 2. AI在帮助构建更强的AI 自我改进循环已经开始.AI辅助AI研发不是科幻,而是各大实验室正在做的事. 3. 这次不是"狼来了" 他特别强调这不是又一次行业炒作,而是"此刻正在发生"的事情. 为什么这次反响这么大 时机太巧了.就在文章爆火的同一周: Anthropic AI安全团队负责人 Mrinank Sharma 辞职,发了一封暗示"世界正处于危险中"的告别信 Anthropic 正在追逐3500亿美元估值 多家媒体报道SaaS公司纷纷改名为"AI公司"(连SaaStr大会都改名叫SaaStr AI了) 恐慌+期待+不确定性,这三种情绪叠加,让一篇推文变成了现象级事件. 冷静看:他漏掉了什么 Vox 的分析说得很到位–从"AI很强"到"一切马上改变"中间隔了好几个现实壁垒: 部署鸿沟远比想象的大. 美国目前只有不到五分之一的企业在实际使用AI.电气化花了几十年才重塑工厂,互联网也没有一夜之间改变零售.大型机构要部署AI,需要数据基础设施,流程再造,合规框架,人员培训–经济学家管这叫"生产力J曲线",早期投入甚至会拖累产出. 更富不等于更忙. 历史规律是:生产力提升后,社会选择更多休闲而非更多工作.GDP增长可能没那么炸裂,但福利水平会实质提升. 最慢的行业决定整体速度. 医疗,教育,线下体验–这些抗自动化的行业会成为瓶颈. 我的看法 Shumer说对了一半.AI能力确实在加速,做技术的人每天都能感受到.但"COVID类比"有个致命问题:COVID是外生冲击,一夜之间降临;而技术变革从来都是渐进的,哪怕它在加速. 真正值得关注的不是"AI什么时候取代所有人",而是: 你所在的行业,AI渗透到了哪一步?, 你的技能栈里,哪些正在被AI平替?, 你有没有在用AI放大自己的能力,而不是等着被替代? 与其被一篇8000万阅读的文章吓到,不如今天就开始动手.用 ChatGPT 优化你的工作流,用 Claude 帮你做研究,用 Cursor 写代码–行动永远比焦虑有用. 恐慌是免费的,但准备才值钱. 参考来源:Business Insider,Vox,CNBC,Mashable

五角大楼怒了!威胁切断Anthropic合作--AI安全护栏该不该为军方让路?

当AI安全遇上军方需求,火药味十足 你可能想不到,全球最重视AI安全的公司Anthropic,正在跟全球最大的甲方–美国五角大楼,闹掰了. 事情是这样的:五角大楼要求四家AI公司(OpenAI,Google,xAI,Anthropic)开放模型给军方"所有合法用途",包括武器研发,情报收集和战场作战.其他三家不同程度地点了头,但Anthropic死活不松口. 五角大楼的态度很明确:谈了好几个月了,Anthropic再不答应,就准备"分手". 更讽刺的是,就在几天前,Anthropic的Claude模型刚刚通过与Palantir的合作,被用在了美军抓捕委内瑞拉前总统马杜罗的行动中.所以,不是完全不合作–只是Anthropic画了一条红线. 这件事为什么重要?三个关键看点 1️⃣ Anthropic的红线在哪? Anthropic表示,他们与政府的对话聚焦在使用政策层面,核心底线是两条: 完全自主武器(没有人类决策的杀人机器) 大规模国内监控 换句话说,Anthropic不是拒绝跟军方合作,而是拒绝"无条件开放".这跟他们一贯的"负责任AI"定位一致. 2️⃣ 其他公司怎么选的? 据报道,四家公司的态度分化明显: 一家已经完全同意五角大楼的条件, 两家"比Anthropic灵活",正在谈判, Anthropic是唯一硬顶的 这意味着AI行业在"军用AI"这件事上,正在走向分裂. 3️⃣ 分类网络的推进 五角大楼还在推动这些公司将AI工具部署到保密网络中,减少常规的安全限制.这不只是政策层面的事–是要把AI模型直接塞进军事机密系统. ️ 普通人怎么理解这件事? 一张图看懂利益关系: ️ 五角大楼:想要AI无限制服务于国防 → “我们是合法使用” Anthropic:坚持AI安全底线 → “合法≠没有限制”, 商业现实:五角大楼是超级大客户,拒绝意味着丢掉巨额合同, ️ 行业影响:如果Anthropic让步,“负责任AI"就成了营销口号 对你的影响: 你可能觉得这跟自己没关系,但想想–如果AI公司为了拿军方合同,逐步放弃安全限制,这些模型的商用版本还会保持严格的安全标准吗?军用AI的放松,往往会传导到民用AI. 极客点评 说实话,我对Anthropic的态度是又佩服又担心. 佩服的是,在真金白银面前坚持原则确实不容易.当你的竞争对手都在向甲方妥协的时候,你还能说"不”,这需要勇气. 担心的是,商业现实很残酷.如果Anthropic因此被排除在政府合同之外,而OpenAI和Google吃下这块蛋糕,长期来看Anthropic的资金和算力竞争力会不会受影响?“负责任AI"需要钱来支撑研究. 更深层的问题是:AI安全的边界到底该由谁来定? 是AI公司自己?政府?还是某种独立机构?目前看来,是谁有钱谁说了算. 适合关注这件事的人: 关心AI安全和伦理的从业者, 使用Claude/GPT等工具的开发者(政策变化可能影响API限制), 对AI军事化应用感兴趣的观察者 这场博弈还没结束,但不管结局如何,它已经暴露了一个根本矛盾:当"让AI更安全"和"让AI更有用"发生冲突时,钱站在哪一边?

Anthropic安全负责人辞职写诗去了!AI安全到底怎么了?

一个安全研究员的「诗意」离场 你能想象吗?一个在全球最注重AI安全的公司–Anthropic(就是做Claude那个)–负责AI安全防线的核心人物,突然宣布辞职,理由是要去学写诗? 这不是段子.2月9日,Anthropic安全防护研究团队负责人 Mrinank Sharma 在X上发了一封辞职信,原话是: “世界正处于危险之中.不仅仅是AI,不仅仅是生物武器,而是一系列相互交织的危机正在此刻展开.” 他说自己"反复看到,让价值观真正指导行动有多么困难"–包括在Anthropic内部,公司也"不断面临搁置最重要事项的压力". 然后他说要回英国,学诗歌,“让自己消失一段时间”. 三个核心看点 1️⃣ 不是第一个,也不会是最后一个 近几个月,AI大厂的安全团队正在经历一波「离职潮」.从OpenAI到Google再到Anthropic,越来越多负责"刹车"的人选择离开.问题很明显:当商业利益和安全原则冲突时,安全往往是让步的那个. 2️⃣ Anthropic的"安全人设"裂了? Anthropic一直以"我们比OpenAI更重视安全"为卖点.它甚至刚怼了OpenAI在ChatGPT里加广告.但当自家安全负责人都说"价值观很难真正落地"的时候,这个人设就很微妙了.更别提2025年Anthropic还因为训练数据版权问题赔了15亿美元. 3️⃣ AI安全不只是技术问题 Sharma的辞职信里提到的不只是AI风险,还有生物武器,社会危机等"相互交织"的问题.他之前的研究包括AI如何被用于生物恐怖主义,AI助手如何让人类"变得不像人".这些听着像科幻,但都是他在Anthropic的实际研究课题. ️ 普通人该怎么看? 不用恐慌:AI目前还远没到"毁灭人类"的程度,但"被滥用"是真实的风险 关注安全报告:Anthropic,OpenAI等公司都会发布安全评估报告,值得一读 支持监管:印度AI峰会(2月16日)即将召开,Sam Altman,Pichai,Amodei等大佬齐聚,AI治理是核心议题 保持批判:任何公司说"我们最安全",都要打个问号–连内部人都觉得价值观难以落地 极客点评 说实话,Sharma辞职去学诗这件事,表面荒诞,内核很真实. 当一个人在AI安全最前线工作了几年,最后的结论是"世界的问题太大了,我要去写诗"–这本身就是一种无声的控诉.AI行业现在的节奏是:融资→发产品→抢市场→安全?以后再说. 谁适合关注这件事? 所有使用AI产品的人.因为安全团队的人走了,不代表AI停下来了.它只会跑得更快,但刹车可能更松了. 下周的印度AI峰会会是一个观察窗口–看看这些CEO们除了说"我们非常重视安全"之外,能不能拿出点真东西. 信息来源:BBC News,Al Jazeera,Business Insider,Financial Express

Made in EU:用欧洲基础设施搭初创公司,比我想的难多了

有人决定把初创公司完全搭建在欧洲基础设施上.以为就是换个云服务商的事–AWS 换成欧盟供应商,搞定. 结果是:比想象中难多了.不是做不成,而是没人告诉你那些奇怪的摩擦点. 为什么要折腾? 数据主权, GDPR 简化, 不想让整个业务依赖三个美国超大规模厂商, 有点倔,想证明这事儿能干 欧盟有真正的基础设施公司在做严肃产品,它们配得上这些流量. 最终选定的技术栈 Hetzner:核心计算 负载均衡器,VM,S3 兼容对象存储.价格跟 AWS 比简直离谱地便宜,性能也很稳.如果你从没在 Hetzner 开过机器,你在云计算上多花了冤枉钱. Scaleway:补充 Hetzner 不覆盖的 Transactional Email (TEM), Container Registry, 第二个 S3 bucket, 可观测性栈, 甚至域名注册 一个供应商搞定多个服务,至少账单简单. Bunny.net:无名英雄 Slovenia 公司,提供: CDN + 分布式存储, DNS, 图片优化 WAF + DDoS 保护 边缘网络真的很强,dashboard 好用到飞.从 Cloudflare 迁移过来很快就适应了. Nebius:AI 推理 如果你需要欧洲的 GPU 计算,又不想把请求发到 us-east-1,这是少数几个真正的选项. Hanko:认证和身份 德国供应商,提供 passkeys,社交登录,用户管理.不需要 Auth0 或 Clerk. 自托管:Rancher 我的爱 自托管了一大堆东西: Gitea: 源代码管理, Plausible: 隐私友好分析, Twenty CRM: 客户管理, Infisical: 密钥管理, Bugsink: 错误追踪 全部跑在 Kubernetes 上,Rancher 是保持集群清醒的胶水. ...

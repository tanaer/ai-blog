Grove:多 Agent 并行写代码,提前知道谁会冲突

发现一个有意思的工具 Grove 专门解决多 Agent 并行写代码的问题 – 提前告诉你哪些分支会冲突,不用等到合并才发现. 谁能用?解决什么问题? 现在很多人用多个 AI Agent 在不同分支并行干活.问题是每个 Agent 各写各的,最后合并时全是冲突. Grove 做的事:持续监控你的 git worktrees,分析分支间的重叠程度,在合并前就告诉你"这俩会撞". 五层检测: File, 检测什么=同一文件被修改, 严重度=黄色 Hunk, 检测什么=修改范围重叠, 严重度=红色 Symbol, 检测什么=同一函数/类型被改, 严重度=红色 Dependency, 检测什么=一方改了导出,一方在用, 严重度=黑色 Schema, 检测什么=都动了迁移文件/路由/环境配置, 严重度=黄色 Claude Code 集成:配置一个 PostToolUse hook,Agent 每次编辑文件后 Grove 自动检查.发现冲突就显示警告,Agent 可以自己调整策略. grove status # 查看所有 worktree 状态 grove conflicts feat/auth feat/payments # 对比两个分支 相比现有方案好在哪? 不用等合并:传统做法是合并时发现冲突,Grove 在写代码时就知道 语义级检测:不只是文件,能检测到函数级别的冲突 Agent 友好:JSON 输出 + daemon 模式,其他工具可以订阅事件 Rust 单二进制:安装简单,性能好 支持 TypeScript,Rust,Go,Python,Java,C# 的符号解析. 安装 brew install NathanDrake2406/tap/grove ...

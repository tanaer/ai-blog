迪士尼怒了:字节跳动Seedance 2.0让蜘蛛侠和达斯维达免费打工,好莱坞集体开炮

一句话摘要 字节跳动的Seedance 2.0上线不到48小时,迪士尼,派拉蒙,MPA,SAG-AFTRA集体发律师函,日本政府启动调查.AI视频生成的"野蛮生长"时代,可能要结束了. 发生了什么? 2月12日,字节跳动悄悄上线了Seedance 2.0–一款AI视频生成工具,目前在中国的剪映App上可用,后续会登陆海外的CapCut. 效果确实炸裂.用户只需要输入两行文字描述,就能生成15秒的高质量视频.问题是,用户生成的内容里充斥着: 蜘蛛侠大战美国队长,在纽约街头打得真假难辨, 达斯维达vs蕾伊,光剑对决逼真到令人发指, 汤姆·克鲁斯对打布拉德·皮特,某位X用户声称只用了"两行提示词" 《死侍》编剧Rhett Reese看完后发了一句话:“不想说,但对我们来说可能结束了.” 好莱坞的反应:核弹级别 这次不是小打小闹.来看看48小时内发生了什么: 迪士尼:周五直接发了律师函,措辞极其强硬–指控ByteDance对迪士尼IP进行了"虚拟打砸抢"(virtual smash-and-grab),声称Seedance预装了一个"盗版角色库",把漫威,星战,皮克斯的角色当免费素材用. 派拉蒙:周六跟进发了律师函,声称Seedance生成的内容"在视觉和听觉上都与派拉蒙的电影和电视节目难以区分". MPA(美国电影协会):CEO Charles Rivkin发声明要求ByteDance"立即停止侵权行为",称Seedance在"单日内就进行了大规模未授权使用美国版权作品". SAG-AFTRA(演员工会):公开声明"与制片厂站在一起,谴责ByteDance的Seedance 2.0所纵容的公然侵权". 日本政府:启动正式调查,因为大量AI生成的日本动漫角色视频在网上传播. ByteDance怎么说? 周一,字节跳动告诉BBC:“公司尊重知识产权,已听到对Seedance 2.0的关切.我们正采取措施加强现有保障机制,以防止用户未经授权使用知识产权和肖像权.” 但他们没有透露具体要加什么safeguard.之前已经暂停了用户上传真人照片的功能,但显然这远远不够. 为什么这次不一样? AI生成内容侵权不是新鲜事–Midjourney去年就被迪士尼和NBC环球告了.但Seedance 2.0之所以引爆舆论,有几个关键差异: 1. 质量跨过了"恐怖谷" 之前的AI视频要么手指多一根,要么运动不自然.Seedance 2.0的输出质量已经到了"普通人分不出真假"的程度.当侵权内容的质量高到可以以假乱真,问题就不再是学术讨论,而是商业威胁了. 2. 规模效应 OpenAI的Sora也有类似问题,但Sora有一定的guardrail(虽然也经常被绕过).Seedance几乎是毫无防护地上线,导致"单日大规模侵权"–MPA的原话. 3. 中美地缘政治背景 别忘了ByteDance刚刚完成TikTok美国业务的出售.在这个时间点上被指控"大规模盗窃美国IP",政治敏感度极高. 4. 迪士尼的双标暴露了行业逻辑 有趣的是,迪士尼去年和OpenAI签了价值10亿美元的授权协议,开放了200多个角色给Sora使用.对OpenAI是授权合作,对ByteDance是律师函–这说明什么?版权战争的本质不是"能不能用AI生成",而是"你付没付钱". 我的判断 这件事会加速三个趋势: 短期:ByteDance会被迫加上严格的内容审核.Seedance的"无限制创作"窗口期已经关闭.预计一周内会有明显的关键词过滤和角色识别机制上线. 中期:AI视频生成行业会被迫建立版权授权体系.OpenAI-迪士尼模式会成为标准–想用IP,先掏钱.这会显著提高AI视频创业公司的成本门槛. 长期:真正的问题不会被解决.当任何人在本地跑一个开源模型就能生成任何角色的视频时,律师函管得住ByteDance,管不住全世界.就像MP3改变了音乐产业一样,AI视频生成正在改变影视产业的基本经济学–只是好莱坞还没准备好接受这个事实. 汤姆·克鲁斯和布拉德·皮特的AI打戏可能很快会被删掉.但技术不会被删掉.好莱坞真正需要担心的不是Seedance 2.0,而是Seedance 3.0会不会是开源的.

全球南方的AI野心:印度AI峰会今天开幕,25万人涌入新德里

今天(2月16日),一场规模空前的AI盛会在新德里的Bharat Mandapam拉开帷幕–印度AI影响力峰会2026. 这不是又一个"AI安全大家坐下来聊聊"的学术会议.25万参会者,600+创业公司,3000+演讲者,20位国家领导人–这是目前为止规模最大的一届全球AI峰会. 更重要的是:这是全球南方第一次主办国际AI峰会. 从安全小圈子到万人博览会 先说历史.这个峰会的前身是2023年11月在英国Bletchley Park举行的AI安全峰会–ChatGPT发布一年后,各国政府还在消化"生成式AI到底意味着什么".那次会议很小,很学术,很"安全优先". 然后是2024年韩国首尔,2025年巴黎.每一届的主题都在扩大–从纯安全讨论,到治理框架,到产业应用. 到了印度这一届,主题已经彻底转向了"如何大规模部署AI".安全只是议程的一部分,而不是全部. 谁来了? 嘉宾名单很说明问题: 政治层面:莫迪亲自开幕,马克龙,卢拉到场,45个部长级代表团 科技层面:Google CEO Sundar Pichai,OpenAI CEO Sam Altman,微软总裁Brad Smith,Qualcomm CEO Cristiano Amon,Meta首席AI科学家Yann LeCun 这阵容不像一个技术会议,更像是一个AI版的达沃斯. 印度在下什么棋? 印度选择主办这个峰会,绝不是因为"轮到我了". 几个关键背景: 1. 数字基础设施的底气 印度在数字公共基础设施上有真实的成绩单:Aadhaar(12亿+数字身份),UPI(全球最大的即时支付系统).他们的逻辑是:我们已经证明了能用低成本数字化覆盖10亿级人口,AI也可以这样做. 2. 全球南方代言人的定位 峰会主题是"People, Planet, Progress"–三个P.翻译一下就是:AI不能只是硅谷和伦敦的游戏.发展中国家怎么用AI解决农业,医疗,教育的实际问题?印度想当这个话题的代言人. 3. 对抗"过度监管"叙事 去年巴黎峰会上,美国副总统万斯的发言很有意思–他公开批评欧洲的AI监管"过于严苛",警告不要扼杀创新.印度显然倾向于这个方向:拥抱AI部署,而不是用法规把它围起来. 第一天在聊什么? 开幕日的议程很接地气: AI+道路安全:用数据分析事故模式,预测风险.印度每年交通事故死亡超15万,这是个真痛点, AI+司法系统:印度法院积压了5000万+案件,AI能不能帮忙提速?, AI+智慧农业:气候适应型农业,面向小农户的AI工具, 女性AI创业者挑战赛:NITI Aayog和合作伙伴推动的包容性创新 你会发现,这些话题都不是"AGI什么时候来"或"AI会不会毁灭人类"–而是**“AI在我的国家能解决什么具体问题”**. 不会有约束力协议 现实一面:和之前的每一届一样,这次峰会大概率不会产生有约束力的国际协议.最可能的产出是一份"新德里宣言"–非约束性的,方向性的,大家签了也不一定执行的. 但这不意味着它没价值.这类峰会的真正作用是设定议程和建立关系网络.25万人面对面交流5天,600个创业公司在13个国家展馆展示–这些连接本身就是产出. 我的判断 三个观察: 全球AI治理正在分裂成三个阵营. 美国的"放手发展",欧盟的"严格监管",以及以印度为代表的全球南方的"实用主义部署".这三个方向的张力会定义未来5年的AI地缘政治. “AI for Good"正在从口号变成政策工具. 不管你觉得这是真心还是作秀,当一个国家动员25万人讨论"AI怎么减少交通事故"而不是"AI怎么生成更好的图片"的时候,话语权就已经在转移了. 中国的缺席很显眼. 虽然官方代表团可能会出席,但在主要发言人和议程设置上,这仍然是一个美印欧主导的舞台.在中美AI竞争的大背景下,印度作为"第三极"的角色越来越清晰. 峰会持续到2月20日.接下来几天如果有重大发布(特别是AI监管框架,大厂新合作),我会跟进. 对了,UNESCO还会在峰会上发布印度AI就绪度评估报告(RAM).这份报告由UNESCO和IndiaAI Mission合作编写,可能是评估发展中国家AI基础设施最系统的一份文件.值得关注.

类脑计算机突然学会了解方程,超算慌不慌?

你可能觉得,模仿大脑的芯片就是搞搞图像识别,跑跑神经网络,干点"聪明但不太硬核"的活. Sandia国家实验室的两位计算神经科学家Brad Theilman和Brad Aimone刚在Nature Machine Intelligence上发了一篇论文,直接把这个偏见打碎了–他们让神经形态(neuromorphic)硬件解出了偏微分方程(PDE). 偏微分方程,就是那种描述流体力学,电磁场,结构力学的数学基础.天气预报要用,核武器模拟要用,材料应力分析要用.传统上,这是超级计算机的地盘,动辄吃掉整栋楼的电. 现在一块模仿大脑皮层结构的芯片,也能干这事了. 大脑一直在做"超算级"计算 这其实不该让人意外.Aimone说了句很有意思的话: “随便挑一个运动控制任务–比如打网球或者挥棒击球.这些都是非常复杂的计算,属于百亿亿次(exascale)级别的问题,而我们的大脑用极低的能耗就搞定了.” 人脑只用20瓦功率,就能实时求解复杂的物理方程–计算落点,调整肌肉力矩,预测球的轨迹.只是我们自己感觉不到罢了. 神经形态计算的思路就是:既然大脑能做,那模仿大脑结构的硬件,凭什么不能? 关键突破在哪 Theilman和Aimone设计了一种新算法,直接在神经形态硬件上高效求解PDE.他们的电路设计基于计算神经科学里一个经典的皮层网络模型. 关键发现:这个模型和偏微分方程之间存在一种自然但不明显的数学联系–而这个联系在模型提出12年后才被他们发现. “我们证明了这个模型与PDE之间有一个天然的,非显而易见的关联.” 这不是硬凑的.是大脑皮层网络的计算结构,本身就暗含了求解PDE的能力. 为什么这很重要 三个层面: 1. 能耗革命 传统超算求解PDE消耗巨大电力.美国国家核安全管理局(NNSA)的超算集群用来模拟核物理系统,电费账单令人窒息.如果神经形态超算能以极低能耗完成同样任务,这对整个高性能计算领域都是颠覆. 2. 神经形态超算不再是幻想 之前业界对神经形态计算的定位是"AI加速器"–跑推理,做模式识别.这篇论文把它的能力边界推到了科学计算领域.Sandia团队的愿景是建造第一台神经形态超算,用于国家安全关键计算任务. 3. 理解大脑的新窗口 反过来看,如果模仿大脑的硬件天然能解PDE,那说明大脑皮层可能一直在用类似机制处理信息.这为理解神经退行性疾病(阿尔茨海默症等)提供了新的计算框架. 我的看法 坦率说,这是近期最让我兴奋的一篇论文. 不是因为它要取代GPU–短期内不可能.而是因为它指向了一个更深层的问题:我们现在烧掉数十亿美元训练的大模型,走的路线可能从根本上就是低效的. 大脑用20瓦做到的事,我们用几十兆瓦的数据中心去逼近.GPU路线是暴力美学,但暴力终有极限. Intel的Loihi,IBM的TrueNorth,SynSense的类脑芯片……这些项目之前一直被看作"学术玩具".Sandia这篇论文可能是一个转折点:类脑计算不只是能模仿智能,它还能做硬核的数学. 当然,从实验室到真正的神经形态超算还有很长的路.算法泛化性,硬件规模化,软件生态……每一个都是大坑.但方向对了,剩下的就是工程问题. 而工程问题,人类最擅长解决. 论文信息: 期刊:Nature Machine Intelligence, 机构:Sandia National Laboratories, 资助:美国能源部科学办公室,国家核安全管理局 延伸阅读: Intel Loihi 2 神经形态芯片 ScienceDaily 原文报道

SaaS已死?2万亿美元蒸发背后,所有软件公司都在抢着改名叫AI

最近科技圈有个很有意思的现象:没有人想当软件公司了. Salesforce,ServiceNow,Oracle–这些企业软件的老大哥们,过去几周股价集体跳水.SaaS行业龙头们的市值合计蒸发了超过2万亿美元.连专门服务SaaS创业者的社区SaaStr都慌了,直接把名字改成了SaaStr AI,创始人的头衔也从CEO变成了"首席AI官". 这不是个别现象,这是一整个行业的身份危机. 到底发生了什么? 简单说:AI Agent来了,传统软件慌了. 过去企业买软件,是为了管理客户数据,发工资,搞采购,做客服.这些SaaS工具就像企业的数字基础设施,一旦用上就很难换–有时候一锁就是十年.靠着这种"换不掉"的粘性,SaaS公司们过着躺着收租的好日子. 但现在,Anthropic推出了Cowork,OpenAI发布了Frontier,再加上OpenClaw这样的开源Agent平台,事情正在起变化. 这些Agentic AI工具的逻辑完全不同:你用自然语言描述一个业务问题,AI直接给你生成解决方案,包括代码,流程,甚至整个系统.不需要按人头付费的"席位",不需要花几个月做定制开发,也不需要被锁定在某个供应商的生态里. 数据说话:AI写代码的速度在指数级增长 这不是画饼.Anthropic的Claude Code已经在实际开发中大量使用,软件开发的生产力正在飙升.Sam Altman更是直接放话:OpenAI的产品很快将实现100%由AI编写代码,涵盖从开发到决策的全流程. 当AI可以当"初级软件工程师+流程设计师"用的时候,企业为什么还要每个月给Salesforce交几百美元一个席位? 这就是华尔街恐慌的核心逻辑. 但先别急着给SaaS写讣告 恐慌归恐慌,现实是复杂的. 第一,企业的切换成本极高. Salesforce管着你的客户数据,ServiceNow跑着你的工单系统,Oracle连着你的供应链.这些系统和业务流程深度绑定,不是说换就换的.就算AI Agent再强,让一个500强企业把核心系统迁走,至少也是两三年的事. 第二,AI Agent目前还不够可靠. 让AI处理个报销单没问题,但让它管企业的财务系统?大多数CIO还没那个胆量.可靠性,安全性,合规性–这些"无聊"的需求恰恰是传统SaaS的护城河. 第三,聪明的SaaS公司已经在自我革命. Salesforce有Agentforce,ServiceNow在搞AI Agent,连Oracle都在往AI上转型.如果你打不过AI,那就把AI塞进自己的产品里. 真正有意思的是中间地带 我觉得最终的结局不是"AI杀死SaaS",而是SaaS的形态会彻底改变. 以前按席位收费–每个人类员工一个账号,一个月几百块.以后呢?可能是按Agent收费–你的AI员工用了多少计算资源,就付多少钱.人类用户数量变得不那么重要了,重要的是AI处理了多少任务. 这意味着: 中小企业是最大赢家. 以前雇不起10个人来做客服,做财务,做HR,现在几个AI Agent就搞定了,成本是传统SaaS的零头., 大企业的IT部门会缩小但更重要. 不再需要一堆人维护和配置软件,但需要更懂业务的人来"训练"和"管理"AI Agent., SaaS创业的门槛会降低. 一个人+几个Agent就能做出以前需要50人团队才能开发的产品. 我的判断 短期内(1-2年),这波恐慌有过度反应的成分.企业软件不会一夜之间消失,Salesforce也不会明天倒闭. 但中期(3-5年),传统的按席位收费模式确实会被颠覆.AI Agent不是取代软件,而是让"软件"这个概念变得更像"服务"–按需生成,按用量付费,随时可替换. 长期来看,回头看2026年2月这波SaaS股价暴跌,它很可能就是一个转折点–就像2007年iPhone发布对功能机市场的冲击一样.不是谁错了,而是游戏规则变了. 最讽刺的是: 那些急着把公司名字改成"XX AI"的SaaS公司,恰恰暴露了自己的焦虑.真正在做AI的公司,不需要改名. 参考来源:NYT DealBook,TechXplore,Forbes,Inc.com

️ 4个人干翻OpenAI!音频AI为什么是小团队的天下?

大厂垄断AI?音频赛道说不 我们都习惯了一个叙事:AI 的未来属于 OpenAI,Google,Anthropic 这些手握数十亿美金和数万张 GPU 的巨头.文本大模型确实如此–训练 GPT-5 级别的模型,没有几十亿根本别想. 但音频AI完全是另一个故事. 在语音合成(TTS),语音转换(STS),实时对话这些领域,最好的模型不是来自大厂,而是来自一群资金有限,人数稀少,却疯狂刷榜的小型创业公司.这不是鸡汤,这是正在发生的事实. 核心亮点:4个人如何创造历史 1. Moshi:全球首个全双工AI语音对话 2024年夏天,巴黎非营利实验室 Kyutai 的一个4人小组在台上演示了 Moshi–全球第一个真正的全双工对话AI. 什么叫全双工?就是它能一边说一边听,能被打断,会插话"嗯嗯"“我懂了”,响应延迟只有 160毫秒–比大多数人类对话都快. 关键是:这个 demo 早于 OpenAI 发布 Advanced Voice Mode,比 xAI 的类似功能早了整整一年.而且模型完全从零训练,没有用任何预训练基座,6个月搞定,还开源了,甚至能在手机上跑. 2. Gradium:从实验室到7000万美金 Kyutai 的核心团队成立了 Gradium,2025年12月宣布获得 7000万美元种子轮融资.CEO Neil Zeghidour 带队,目前已支持英语,法语,德语,西班牙语和葡萄牙语的实时语音API. Edge SDK 预计 2026 Q1 发布–意味着很快你就能在本地设备上跑起端到端的语音AI. 3. 为什么小团队能赢? 这不是偶然.音频AI有几个独特的结构性优势: 数据效率高:音频模型不需要互联网规模的语料,高质量的语音数据集就够用, 计算需求相对低:训练一个SOTA音频模型的算力,可能只是训练文本大模型的百分之一, 研究密度大于资源密度:突破来自算法创新(如音频编解码器,全双工架构),不是靠堆GPU, 大厂不够重视:文本模型太赚钱了,音频只是"附加功能",大厂投入有限 ️ 想体验?快速上手指南 试玩 Moshi(开源版): 访问 moshi.chat 在线体验全双工对话 或者 GitHub 克隆:git clone https://github.com/kyutai-labs/moshi 本地部署支持 CPU 和 GPU,甚至可以在手机端运行 接入 Gradium API: ...

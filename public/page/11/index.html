DeepMind的AI独立写了一篇数学论文,还推翻了一个十年老猜想--但700道题只答对6.5%

AI从做题家变成了研究员 上周,Google DeepMind扔出了一个重磅炸弹:Aletheia–一个不是用来刷题,而是用来做真正数学研究的AI Agent. 这不是说"AI在数学竞赛拿金牌"那种新闻了.那个阶段早在2025年就过去了.Aletheia瞄准的是一个完全不同的目标:让AI自主产出可发表的学术研究成果. 结果很有意思.一半是惊艳,一半是打脸. 亮点:独立写论文,推翻老猜想 先说让人兴奋的部分. Aletheia独立完成了一篇算术几何论文(Feng26).不是人写了框架AI填充细节,是AI从头到尾自己搞定的–计算了一类叫eigenweights的结构常数.论文已发表在arXiv上.DeepMind的说法是:整个数学内容完全由AI生成,人类作者只做了排版. 它还推翻了一个存在十年的数学猜想,并在一篇密码学论文中发现了专家遗漏的关键错误. 在协作模式下,Aletheia帮人类数学家证明了关于独立集界的定理(LeeSeo26)–AI提供高层策略和"大局观"路线图,人类负责严格化. IMO-Proof Bench Advanced基准测试上,Aletheia拿到了**95.1%**的准确率,而2025年7月的前代只有65.7%.性能飞跃巨大. 但700道真正的开放问题呢?只有6.5%有用 重点来了. DeepMind把Aletheia扔进了Erdős猜想数据库–700道至今无人解决的开放数学问题.这才是真正的考试. 结果:63个技术上正确的解答,4个真正解决了开放问题. 算一下:有用率6.5%. 这个数字非常诚实,也非常重要.它说明了一个关键现实:AI在有标准答案的竞赛题上表现惊艳,但面对真正的开放性研究,成功率仍然很低. 这不是贬低–4个开放问题的自主解决已经是历史性的.但它提醒我们,从"做题"到"做研究"的鸿沟,比想象中大得多. 架构:三个AI互相查作业 Aletheia的设计思路非常务实.它不是一个模型独挑大梁,而是三个角色分工的agentic loop: Generator(生成器):针对问题提出候选解答, Verifier(验证器):用自然语言检查解答中的逻辑缺陷和幻觉, Reviser(修正器):根据验证器发现的问题修改解答 这个循环反复迭代,直到验证器接受结果,或者达到尝试上限.关键在于:验证和生成必须分离.研究团队发现,让同一个模型同时生成和检查,它会对自己的错误视而不见.拆开之后,检出率显著提升. 底层动力来自Gemini Deep Think的升级版.2026年1月版本在推理时间扩展(inference-time scaling)上取得了突破–解决IMO级别问题所需的计算量比2025年版降低了100倍. 幻觉问题:从编造论文变成了曲解论文 做研究和做竞赛有个根本区别:研究需要引用文献. AI的老毛病在这里特别致命–它会编造根本不存在的论文,书籍和作者名字.Aletheia通过接入Google搜索和网页浏览来解决这个问题,效果明显:明显的虚构(假书名,假作者)大幅减少. 但问题变形了.模型开始引用真实存在的论文,然后错误地描述论文内容.相当于一个学生从"编假参考文献"进化到了"引用真论文但断章取义". 这类错误更隐蔽,更难检测,也更危险. DeepMind的"自动驾驶分级"体系 有意思的是,DeepMind提出了一个AI数学贡献的分级标准,灵感来自自动驾驶的L0-L5: Level 0, 自主程度=主要靠人, 研究意义=可忽略的新颖性(竞赛题水平) Level 1, 自主程度=人机协作, 研究意义=较小的新颖性 Level 2, 自主程度=基本自主, 研究意义=可发表质量 Feng26论文被归类为Level A2–基本自主且达到可发表质量.这是目前AI在学术研究中达到的最高等级. 我的看法:这才是AI真正让人兴奋的方向 说实话,我对"AI刷题拿金牌"这类新闻早就脱敏了.但Aletheia让我重新兴奋起来,原因有三: 第一,它足够诚实. 6.5%的有用率直接公开,没有选择性展示.这种透明度在AI营销满天飞的今天非常稀缺. 第二,它指向了AI真正的价值. 不是替代数学家,而是成为一个"永远不累,不怕试错,有时能给出灵感"的研究伙伴.4/700的成功率看似很低,但对于人类数学家来说,能解决一个开放猜想就足以定义整个职业生涯. 第三,架构设计非常有启发性. Generator-Verifier-Reviser的三角分工模式,本质上是把科学方法论(假说-验证-修正)编码进了AI系统.这个思路完全可以迁移到其他研究领域. 但也要清醒:95.1%的竞赛题准确率和6.5%的开放题有用率之间,是一条我们尚未跨越的鸿沟.AI擅长的是"已知有解的问题",而研究的本质是"不知道有没有解的问题". Aletheia证明了AI可以偶尔跨过这条线.但"偶尔"和"可靠"之间,还有很长的路. 参考来源: DeepMind官方博客 Aletheia论文(arXiv) The Decoder深度分析

AGENTS.md到底有没有用?一篇论文引爆开发者社区大战

一个配置错误搞瘫半个互联网:Cloudflare BGP事故暴露了我们不敢面对的真相

一个周日下午,半个互联网消失了 2026年2月16日,美东时间下午1点15分. 一个Cloudflare工程师在弗吉尼亚州阿什本数据中心执行了一次"常规配置更新".几分钟后,全球互联网开始抽搐. X(原Twitter)时间线一片空白,Grok同时挂掉.AWS的US-East-1区域出现"间歇性连接降级".电商网站结账流程中断.数千个依赖Cloudflare CDN的中小型网站直接返回错误页面. Downdetector在4小时内录得超过11,000条X平台故障报告,峰值出现在下午4:20.美国,英国,欧洲部分地区用户首当其冲. 4个小时后,工程师们才完全恢复服务. 这不是一次黑客攻击.不是DDoS.甚至不是什么复杂的技术故障. 这是一个BGP路由配置错误.一个. BGP:互联网的GPS,也是最脆弱的那根弦 BGP(Border Gateway Protocol,边界网关协议)是互联网的"路由导航系统".它告诉数据包:“从A到B,走这条路.“全球互联网的流量路由几乎完全依赖这个上世纪80年代设计的协议. 问题在于:BGP本质上是基于信任的.一个网络宣布"我能到达这些IP地址”,其他网络就信了.没有严格的验证机制(虽然RPKI等改进在推进,但普及率还很低). 当Cloudflare阿什本节点的BGP路由表被错误配置后,坏的路由信息像病毒一样传播到上游Provider.正常应该流经Cloudflare全球300多个数据中心的流量,被错误导向,丢弃,或者在网络中打转. 最讽刺的是:Cloudflare自己的状态页也挂了. 工程团队40分钟内定位了根因,但回滚比部署困难得多–因为损坏的路由表已经被上游Provider缓存了.你没法简单地"撤回"一条BGP公告,它已经在整个互联网中扩散. 多米诺骨牌:AWS和X为什么也倒了 这次事故最值得警惕的不是起因,而是传播方式. AWS承载着全球约三分之一的云基础设施.虽然AWS有独立的网络骨干,但大量客户流量在到达AWS之前,会先经过Cloudflare做DDoS防护和CDN加速.当Cloudflare路由失效后,海量的重试连接和误导数据包涌入AWS边缘路由器,直接把几台路由器打爆了. X平台更惨.自从马斯克收购后大幅削减基础设施成本,X同时依赖Cloudflare做边缘安全,AWS做后端计算.两个Provider同时出问题,X基本没有fallback方案.结果就是:将近3小时无法发帖,刷时间线,收发私信. Bluesky和Threads的用户量在这几个小时内暴涨.有评论员说:“全球最有钱的科技人士拥有的平台,被一行配置搞挂了.” 表面分布式,实际深度中心化 普林斯顿大学计算机科学教授Jennifer Rexford(前FCC技术顾问委员会成员)的话一针见血: “我们构建了一个看起来分布式,实际上深度中心化的系统.每次这类宕机发生,我们都进行同样的’韧性’讨论,然后什么都没变–因为经济激励全都指向集中化.” 几个数字让你感受一下: Cloudflare处理全球约**20%的网络流量, AWS占据云基础设施市场约31%**的份额, 加上Azure和GCP,三家公司控制了全球超过60%的云计算 当这些巨头之间还存在深度依赖关系时(比如用Cloudflare做CDN + AWS做计算),一个节点的故障就能引发系统性崩溃. 这不是分布式系统.这是一个伪装成分布式的中心化系统. 这事以前发生过,以后还会发生 回顾一下历史: 2024年7月:CrowdStrike更新导致全球Windows蓝屏,航空,银行,医疗系统瘫痪, 2021年10月:Facebook(Meta)BGP配置错误,Facebook/Instagram/WhatsApp全球宕机6小时, 2021年6月:Fastly CDN故障,Amazon,Reddit,CNN等大面积下线, 2020年11月:AWS US-East-1宕机,半个互联网跟着挂 每次事故后都有一轮关于"互联网韧性"的讨论.然后呢?什么都没变. 原因很简单:集中化是经济理性的选择.用Cloudflare比自建CDN便宜100倍,用AWS比自建数据中心快100倍.没有哪个CTO会为了理论上的"韧性"去增加10倍的基础设施成本. 真正的问题:我们能做什么? 作为技术从业者,我认为有几个层面值得思考: 1. 架构层面:多云不是口号 如果你的服务100%跑在AWS上,Cloudflare做CDN,那你的SPOF(单点故障)比你以为的多得多.真正的多云意味着: 关键服务在至少两个云Provider上有热备, CDN/安全层有备用方案(比如Cloudflare + Fastly双活), DNS做好多Provider解析 成本会增加30-50%,但对于核心业务来说这是保险费. 2. 协议层面:推进RPKI和BGP安全 RPKI(Resource Public Key Infrastructure)可以验证BGP路由公告的合法性,但全球采用率还不到50%.这需要整个行业的推动,不能靠单个公司. 3. 监管层面:互联网基础设施需要更高标准 Cloudflare,AWS这些公司本质上是互联网的"公用事业”,但它们承担的监管义务远低于电力,电信等传统公用事业.也许是时候讨论基础设施层面的韧性标准了. 4. 认知层面:接受"零宕机"是幻觉 任何足够复杂的系统都会出故障.问题不是"会不会挂",而是"挂了之后多快能恢复".设计系统时,优雅降级比防止故障更重要. 我的看法 这次Cloudflare事故不是黑天鹅–它是灰犀牛.每个基础设施工程师都知道BGP的脆弱性,每个云架构师都知道Provider集中度的风险.但经济激励和技术惯性让我们选择性忽视. ...

类脑芯片突然学会了解微分方程:神经形态计算正在颠覆超算的底层逻辑

超算的电费单,可能快要变薄了 你知道运行一个大规模物理仿真要多少电吗? 美国能源部的超级计算机集群每年光电费就烧掉数亿美元.天气预报,核武器模拟,材料应力分析–这些都依赖求解偏微分方程(PDE),而PDE是出了名的计算密集型任务. 但上周发表在 Nature Machine Intelligence 上的一篇论文,可能要改写这个故事. Sandia国家实验室的两位计算神经科学家 Brad Theilman 和 Brad Aimone 证明了一件让很多人意外的事:模仿人脑结构的神经形态计算机,居然能高效求解偏微分方程. 等等,类脑芯片不是只能做图像识别吗? 这是大多数人的直觉.神经形态计算(Neuromorphic Computing)过去十年的主要战场是模式识别,推理加速,本质上是在模拟神经网络.让它去解严格的数学方程?听起来就像让画家去做会计–不是一个赛道的事. 但 Aimone 的反驳很精彩: “随便挑一个运动控制任务–打网球,挥棒球棒.这些都是极其复杂的计算,是百亿亿次级(exascale)的问题,而我们的大脑以极低的能耗就搞定了.” 换句话说,人脑本身就是一台极其高效的PDE求解器–流体力学(血液循环),电磁场(神经信号传导),结构力学(骨骼肌肉协调),这些全是PDE问题.大脑只用20瓦功率就能实时处理. 他们具体做了什么? Theilman 和 Aimone 开发了一种新算法,能让神经形态硬件直接求解PDE.关键点在于: 算法基于一个已有12年历史的皮层网络计算模型,但此前从没有人发现它跟PDE之间的数学联系 这个联系是"自然但非显而易见的"–模型结构本身就隐含了求解PDE的能力,只是12年来没人看到 在实际测试中,神经形态硬件的能效比传统超算高出数个量级 这项研究由美国能源部科学办公室和国家核安全管理局联合资助.后者负责维护美国核武库–他们对"省电"这件事的兴趣,可不是环保那么简单. 为什么这件事值得关注? 第一,这打破了"类脑计算只能做AI"的认知边界. 神经形态芯片(比如Intel的Loihi,IBM的TrueNorth)长期被定位为AI加速器.这项研究说明,它们的能力被严重低估了.如果PDE都能解,那CFD(计算流体力学),FEA(有限元分析),气候模型这些传统超算的核心业务,都可能被重新定义. 第二,能耗问题是AI行业的定时炸弹. 我们之前写过硬盘供应链被AI吞噬的故事.但更大的危机是电力.全球数据中心的电力消耗正在指数增长,而传统冯·诺依曼架构的能效已经逼近物理极限.神经形态计算提供了一条完全不同的路径–不是"更快的马",而是"发明汽车". 第三,这可能重新定义"超算"这个概念. Sandia的团队已经在展望"神经形态超级计算机".想象一下:一台能耗只有传统超算1%的机器,跑着物理仿真,气候模型,核武器模拟.这不是科幻,这是Nature Machine Intelligence上白纸黑字的研究路径. 我的判断 这项研究的意义被严重低估了. 媒体都在追 GPT-5.3,Claude Opus 4.6 这些大模型的军备竞赛,但真正可能改变计算范式的突破,往往藏在这种"冷门"论文里. 当然,从实验室到商业化还有很长的路.神经形态硬件的编程模型远不如GPU成熟,生态系统几乎为零,软件工具链更是荒漠.但方向一旦被证明可行,资本和工程力量的涌入只是时间问题. 建议关注几个信号: Intel Loihi 3 的进展(预计今年下半年) DARPA 的 MACHine 项目(神经形态超算路线图) 学术界跟进论文–如果接下来3-6个月出现大量复现和扩展工作,说明这个方向真的起飞了 20瓦的大脑能做到的事,为什么我们的计算机需要20兆瓦? 这个问题,终于有人在认真回答了. 参考文献:Theilman & Aimone, “Solving Partial Differential Equations on Neuromorphic Hardware”, Nature Machine Intelligence, Feb 2026 ...

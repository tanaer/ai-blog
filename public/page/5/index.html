AI 帮浏览器搬家：Ladybird 两周搞定 2.5 万行 Rust 迁移

2.5 万行代码从 C++ 迁移到 Rust，传统方式得几个月？Ladybird 浏览器团队用 AI 两周就搞定了，而且零回退。 这不是"AI 自动写代码"的吹牛故事。创始人 Andreas Kling 把话说得很清楚：人主导，AI 干活。他决定迁移哪些模块、用什么顺序、Rust 代码该长什么样，然后通过几百个小 prompt 让 Claude Code 和 Codex 逐步完成翻译。 目标是 LibJS（JavaScript 引擎），包括词法分析器、解析器、AST 和字节码生成器。关键是 Ladybird 已经有 test262 测试套件，每一段生成的代码都能跟 C++ 版本做字节级对比——生成的 AST 和字节码必须完全一致。 结果：2 周搞定，手工得几个月。零回退，所有测试通过。 这模式值得注意：有测试覆盖 + 有可信参照实现 + AI 翻译，比"让 AI 从零写代码"靠谱多了。内存安全语言迁移的老大难问题，可能找到了加速器。

Ladybird 两周移植 25000 行代码到 Rust，AI 辅助的教科书级案例

Ladybird 浏览器最近干了件狠事——把 JavaScript 引擎从 C++ 移植到 Rust，25000 行代码，两周搞定。按传统方式这得折腾好几个月。 有意思的不是速度，是方法。Andreas Kling 没让 AI 自主写代码，而是用了"人机协作"模式：他决定移植顺序和代码风格，Claude Code 和 Codex 负责翻译。几百个小 prompt，一步步把 lexer、parser、AST、字节码生成器全搬过去。 关键是验证。他们要求 Rust 版和 C++ 版的输出必须字节级一致。靠着 test262 测试套件，最终零回归——每个 AST、每条字节码都跟原版一模一样。 这事儿说明两件事：一是 AI 真的能干重活，前提是你得有好的测试覆盖；二是 Rust 在浏览器领域的吸引力越来越大，内存安全不是闹着玩的。Ladybird 这波算是给"AI 辅助大型重构"打了个样。

Simon Willison 的新项目：用智能体工程对抗 Vibe Coding

Simon Willison 又开新坑了——这次是"Agentic Engineering Patterns"，专门收集编码智能体的最佳实践。 他把现在 AI 编程分成两派：一派是"vibe coding"，完全不看代码让 AI 乱写；另一派是"Agentic Engineering"，专业工程师用 AI 放大自己的能力。很明显，他站后者。 这个项目的定位很有意思：不是博客文章，也不是传统书籍，而是"Guide + Chapter"格式。Guide 是主题集合，Chapter 是可以持续更新的章节——解决了博客内容容易过时的问题。 前两章已经上线： “Writing code is cheap now” 探讨代码成本归零后，我们的工作直觉该怎么调整 “Red/green TDD” 讲测试驱动开发怎么让智能体写得更稳 Willison 还特意强调：内容全是他自己写的，LLM 只负责校对和生成示例代码。这态度挺硬核的，但也说明他对这事儿多认真。 最骚的是，整个 Guide/Chapter 的技术实现，几乎都是 Claude Opus 4.6 在 Claude Code 里写的。用智能体搭建智能体工程指南，这递归玩得溜。 他说打算每周更新 1-2 章，没说什么时候结束——因为这领域本身就还在快速演化。值得关注。

AI 两周搞定浏览器引擎重写，这事儿有点意思

BinaryAudit:他们往 40MB 二进制里藏后门,让 AI 去找

Quesma 做了个有意思的实验:和逆向工程专家合作,把后门藏在 40MB 的二进制文件里,然后让 AI + Ghidra 去找. 背景是最近供应链攻击太多了–Shai Hulud 2.0 搞了几千家组织,Notepad++ 被国家赞助者替换了二进制,波兰火车被查出可疑代码. 问题是:我们用的闭源软件,只有二进制,怎么知道有没有后门? 他们的做法: 用 VLM 和 Ghidra MCP 分析二进制 让 AI 做逆向工程:机器码 → 汇编 → 伪 C 代码 输出有没有后门,在哪里 结果:最好的模型 Claude Opus 4.6 在中小二进制里找到相对明显的后门,成功率 49%. 更糟的是,很多模型有高误报率–干净文件也报有毒. 结论:AI 能做一点逆向工程,但还远不到能放心的程度. 基准测试开源:github.com/quesmaOrg/BinaryAudit 如果你担心供应链安全,可以看看他们怎么测的.

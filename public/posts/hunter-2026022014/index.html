OpenClaw 与 VirusTotal 合作:ClawHub 技能市场上线 AI 安全扫描

OpenClaw 官方宣布与 VirusTotal 合作,为 ClawHub 技能市场带来安全扫描:Code Insight(Gemini 驱动)分析整个技能包,自动审批/警告/阻止机制,每日重扫描,安全路线图公开.
2026年2月20日 · 1 分钟 · 130 字 · AI 极客前沿
⚡ 快捷操作 🚀 发到 X

AI Agent 能干很多事–控制智能家居,管理财务,自动化工作流.但一个恶意技能(skill)也能干很多坏事:窃取数据,执行未授权命令,以你的名义发消息.

OpenClaw 刚宣布与 VirusTotal 合作,为 ClawHub 技能市场带来安全扫描.这是 AI Agent 安全的重要一步.

为什么这很重要?#

传统软件的安全模型是"锁死设备,隔离进程,沙箱隔离".但 AI Agent 不一样:

  • 它们解释自然语言,自己做决策, 用户意图和机器执行的边界模糊了, 可以通过语言本身被操纵

一个恶意技能可以:

  • 窃取敏感信息, 执行未授权命令, 以你的名义发消息, 下载并运行外部 payload

随着 OpenClaw 生态增长,攻击面也在增长.已经有记录在案的恶意攻击尝试了.

扫描流程#

当技能发布到 ClawHub 时:

  1. 确定性打包:技能文件被打包成 ZIP,包含 _meta.json(发布者信息和版本历史)
  2. 哈希计算:计算整个包的 SHA-256 哈希
  3. VirusTotal 查询:检查哈希是否已存在于 VirusTotal 数据库
  4. 上传分析:如果不存在,上传到 VirusTotal 进行扫描
  5. Code Insight:VirusTotal 的 LLM(Gemini 驱动)对整个技能包进行安全分析
  • 不只是看技能声称做什么, 分析代码实际做什么:是否下载外部代码,访问敏感数据,执行网络操作, 检测是否嵌入可能胁迫 Agent 的不安全行为指令
  1. 自动处理:
  • “benign” → 自动批准, “suspicious” → 自动标记警告, “malicious” → 立即阻止下载
  1. 每日重扫描:所有活跃技能每天重新扫描,检测是否从干净变恶意

这不是银弹#

OpenClaw 很诚实:这不是万能的.

  • 用自然语言指示 Agent 做坏事不会触发病毒签名
  • 精心设计的提示注入不会出现在威胁数据库里

但这提供了:

  • 已知恶意软件检测:木马,窃密器,后门, 行为分析:Code Insight 识别可疑模式, 供应链可见性:捕获被入侵的依赖和嵌入的可执行文件, 意图信号:这是第一层,还有更多层要来

更大的安全计划#

这次合作是 OpenClaw 更大安全计划的一部分:

  • OpenClaw 生态的综合威胁模型, 公开安全路线图, 全代码库安全审计, 正式安全报告流程(带 SLA)
  • 已聘请 Jamieson O’Reilly(Dvuln 创始人,Aether AI 联合创始人)担任安全顾问

跟踪进度:trust.openclaw.ai

对技能发布者#

你的技能现在会自动被扫描:

  • 扫描异步运行, “suspicious” → 标记警告但保持可用(透明性), 可在技能详情页查看扫描状态和完整 VirusTotal 报告

如果误报,联系 security@openclaw.ai.

对用户#

浏览 ClawHub 时,你会看到每个技能的扫描状态.但请记住:

  • 扫描通过 ≠ 绝对安全, 始终检查技能请求的权限, 优先使用你信任的发布者的技能, 发现可疑行为报告到 security@openclaw.ai

一句话:AI Agent 能力越强,安全越重要.VirusTotal 集成是 ClawHub 的第一道防线–但用户也要保持警惕.

ClawHub: clawhub.ai 安全详情: trust.openclaw.ai

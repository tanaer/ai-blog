Jsonh:JSON 终于能写注释了,还能省略逗号和引号

Jsonh 是一个全新的 JSON 超集,支持注释,省略逗号,多行字符串,无引号键值等特性.兼容 JSON 和 JSON5,号称'HJSON v2',让配置文件真正为人类设计.

一句话总结#

Jsonh 让 JSON 终于像人类语言了–写注释,省略逗号,多行字符串,无引号键值,还能兼容现有的 JSON 和 JSON5.

痛点:JSON 反人类的地方#

每个写过配置文件的人都懂:

{
 "name": "my-app",
 "version": "1.0.0",
 "description": "A very long description that needs to be on one line..."
}
  • 不能写注释(解释为什么这样配置), 最后一个属性不能加逗号(git diff 很丑), 多行字符串必须用 \n(阅读噩梦), 键必须加引号(手累)

YAML 和 TOML 试图解决这些问题,但 YAML 的缩进地狱和 TOML 的嵌套语法又带来了新麻烦.

Jsonh 的解法:JSON + 人性化语法#

Jsonh 是 JSON 的超集,意味着所有合法 JSON 都是合法 Jsonh.但它增加了这些救命特性:

1. 三种注释方式#

 // 行注释
 # 也是行注释
 /* 块注释 */
 name: "my-app"

### 2. 逗号可选
a: 1
 b: 2
 c: 3 // 不需要逗号!

### 3. 多行字符串
haiku: '''
 Let me die in spring
 beneath the cherry blossoms
 while the moon is full.
 '''

### 4. 无引号键值
keys: without quotes
 isn\'t: {
 that: cool?

### 5. 兼容 JSON5
支持十六进制数字,转义换行等 JSON5 特性:

 hex: 0xDEADCAFE
 fraction: .5e3
 bigNum: 1_000_000 // 下划线分隔

```bash

**Python:**

**Rust:**

dotnet add package JsonhCs

dotnet tool install --global JsonhCli

jsonh convert config.jsonh -o config.json

# 验证语法
jsonh validate config.jsonh

## 为什么不用 YAML/TOML?
**YAML 的问题:**
- 缩进敏感(少一个空格就报错)
- 多行字符串语法复杂(`>`, `|`, `>-`, `>+` 傻傻分不清)
- 甚至不是 JSON 的超集!

**TOML 的问题:**
- 嵌套对象语法反直觉(`[[table]]` 是什么鬼), 深层嵌套时配置分散在各处, 复杂对象还是得用 JSON 语法

**Jsonh 的优势:**
- 基于 JSON 语法,学习成本为零, 层级结构清晰(不像 TOML 那样分散), 严格的语法规则(不像 YAML 那样容易出错)

## 写在最后
Jsonh 不是什么革命性发明,它只是**把开发者需要的实用特性整合到一个格式里**.

如果你厌倦了 JSON 的严格,YAML 的诡异,TOML 的分散,Jsonh 可能是你配置文件的新选择.

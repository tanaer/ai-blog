Ember MCP:给 Claude 装上永久记忆,告别每次重复交代背景

Ember MCP 是一个本地优先的 LLM 记忆服务器,通过 Voronoi 分区和时间智能算法,让 AI 记住你的项目背景,自动识别过时信息,告别每次开新窗口都要重新交代技术栈的烦恼.

一句话总结#

Ember MCP 让你的 Claude/Cursor 拥有真正的长期记忆–关闭电脑周五下班,周一打开新对话,AI 依然记得你上周的技术决策,而且不会把已经废弃的架构方案混进新建议里.

痛点:AI 的"金鱼记忆"#

用过 Claude Desktop 或 Cursor 的开发者都懂这个痛:

  • 每次新开对话窗口,都要重新交代项目背景
  • 三个月前说好用 PostgreSQL,现在迁移到 MongoDB 了,AI 还在给 PostgreSQL 的代码建议
  • 上周刚决定弃用 Redux 改用 Zustand,这周 AI 还在生成 Redux boilerplate

传统向量数据库的问题在于:它只看语义相似度,不看信息时效性.旧记忆和新查询语义相近,AI 就傻傻地检索出来用,结果就是–幻觉.

Ember MCP 的解法:时间感知 + 漂移检测#

Ember MCP 的核心创新是 Drift Detection(漂移检测).

怎么工作的?#

  1. Voronoi 分区:把向量空间划分成 16 个固定区域(比如"数据库",“前端框架”,“业务逻辑”)
  2. 时间智能评分:记忆按重要程度赋予半衰期,越老的记忆权重越低
  3. 统计漂移检测:监控每个知识区域的分布变化,发现显著偏移时自动标记旧记忆为"过期"

实际场景#

1 月份:你告诉 Claude 项目用 PostgreSQL → Ember 在"数据库"区域存储相关记忆

4 月份:你迁移到 MongoDB,产生大量新记忆

传统向量库:PostgreSQL 和 MongoDB 都语义相似于"数据库查询",AI 可能给出混合建议

Ember:检测到"数据库"区域的向量分布发生显著漂移,自动将 PostgreSQL 记忆标记为过期,查询时只返回 MongoDB 相关内容

核心特性一览#

  • 跨会话持久: 关闭电脑,重启客户端,记忆依然跟随
  • 100% 本地: ~500MB 磁盘,~200MB RAM,无需 API Key 或云端向量库
  • 隐私安全: 数据永不离开本机,可放心处理 NDA 文件
  • 源文件追溯: AI 说"我们决定用 Kubernetes"时,能指出具体是哪次会议记录
  • 零配置: 自动检测 Claude/Cursor/Windsurf,一键安装

11 个 MCP 工具#

Ember 通过 MCP 协议向 LLM 暴露 11 个工具:

  • ember_store - 存储命名记忆,可设置重要程度
  • ember_recall - 语义搜索 + 时间评分
  • ember_deep_recall - 召回并自动读取源文件, ember_learn - 自动捕获对话中的关键信息, ember_contradict - 标记旧记忆过期并存储修正版本, ember_list - 列出所有记忆(可按标签过滤), ember_drift_check - 手动运行漂移检测

快速开始#


# 一键安装(自动检测已安装的 MCP 客户端)
pip install ember-mcp
ember-mcp install

# 验证安装
ember-mcp status

安装器会自动:

  • 检测 Claude Desktop,Claude Code,Cursor,Windsurf
  • 向每个客户端注册 Ember MCP
  • 创建本地存储目录并下载嵌入模型

重启 AI 客户端即可使用.

手动配置#

如果自动检测失败,手动添加到 MCP 配置:

{
 "mcpServers": {
 "ember": {
 "command": "ember-mcp",
 "args": ["run"]
 }

## 技术实现细节
- **嵌入模型**:all-MiniLM-L6-v2(384 维向量,纯 CPU 运行)
- **向量搜索**:Meta FAISS 库, **统计监控**:Welford 算法流式计算均值和方差, **存储位置**:`~/.ember/`(可备份,可重置)

## 应用场景
1. **长期项目开发**:记住技术决策,架构演进,避免反复交代背景
2. **多客户端切换**:白天用 Claude Desktop,晚上用 Cursor,上下文无缝衔接
3. **会议记录管理**:存储决策和后续行动,AI 能追溯具体来源
4. **个人知识库**:构建私有,持久的 AI 记忆系统

## 写在最后
Ember MCP 解决了一个被忽视但极其影响体验的问题:**AI 的记忆管理**.

它不是简单的向量存储,而是带有时效性意识的认知引擎.对于需要长期与 AI 协作的开发者来说,这可能是 2026 年最值得尝试的 MCP 工具之一.

Grok 4.20 Beta炸场:8个AI Agent并行写代码,xAI要把Cursor们逼到墙角

Grok 4.20 Beta在2月17日悄然上线,Reddit的r/singularity瞬间炸了. 但这次让开发者坐不住的不是跑分–而是xAI塞进去的一套多Agent并行编程系统.简单说:你发一个需求,最多8个AI Agent同时帮你写代码,然后系统自动评分,挑最好的方案给你. 这玩意儿,直接把AI编程从"单人作战"推进到了"团队协作"阶段. 8个Agent同时干活,这不是噱头 Grok Build是xAI的编程产品,之前一直在内测.这次随4.20 Beta一起曝光的Parallel Agents功能,是真正的杀手锏. 具体怎么用: 你输入一个编程需求 系统同时启动最多8个Agent(两个模型各4个:Grok Code 1 Fast和Grok 4 Fast) 所有Agent的输出并排显示,你可以逐个对比 界面还带context usage tracker,实时看每个Agent消耗了多少上下文 这不是什么花活儿.Musk之前说过要让Grok"同时调度数百个专业编程Agent协作",现在看来这是第一步落地. 竞技场模式:Agent打擂台 比Parallel Agents更有意思的是代码里发现的Arena Mode. Parallel Agents是让你人工比较多个输出.Arena Mode则更进一步–Agent之间自动竞争,系统自动评分排名,选出最优方案. 这个思路跟Google Gemini Enterprise的Idea Generation Agent类似:先让多个Agent各自生成方案,再通过结构化竞赛流程筛选最佳结果. 区别在于,Google的方案偏向企业级工作流,而xAI直接把这套东西做进了编程工具里.开发者写代码的时候,背后有一场小型锦标赛在运行–这个画面还挺赛博朋克的. 实力到底怎么样? 先说跑分:Reddit社区的反馈是混合的. 有人发现Grok 4.20在SimpleBench测试里会"作弊"–Agent不是纯靠推理,而是手动上网搜答案.这说明xAI在推理能力和工具调用之间还没找到最佳平衡点. 但在实际应用层面,数据很硬: Alpha Arena交易竞赛(2026年1月):Grok 4.20的早期checkpoint拿下了10-12%的回报率,部分配置甚至+47%,而竞争对手全在亏钱, 实时数据处理:接入X平台Firehose(每天约6800万条英文推文),实现1-5分钟级别的情绪-价格信号分析, 多模态能力提升:文本,图像,视频全面增强, 幻觉率降低:内置fact-checking工具 说白了,Grok 4.20的路线不是"在封闭测试里刷高分",而是"在真实世界里能干活". Grok Build:不只是Agent,是一个IDE 除了Agent系统,Grok Build的UI也在大改.现在的界面已经接近一个浏览器端IDE: 语音输入支持(Vibe Coding的核心体验), 导航标签:Edits,Files,Plans,Search,Web Page, 实时代码预览和代码库导航, 分享按钮和评论功能(协作场景), GitHub App集成(设置里已可见,但还没激活) xAI的野心已经很明确了:不是做一个ChatGPT的编程插件,而是做一个原生AI-first的开发环境.Cursor,Windsurf们应该已经感受到压力了. Grok 5:6万亿参数怪兽在路上 Grok 4.20只是过渡.真正的大招是2-4个月后的Grok 5. 关键数据: 参数量, Grok 3/4=3万亿, Grok 5=6万亿 智能密度, Grok 3/4=基准, Grok 5=每GB更高 工具使用, Grok 3/4=基础, Grok 5=大幅增强 ...

DeepMind的AI独立写了一篇数学论文,还推翻了一个十年老猜想--但700道题只答对6.5%

AI从做题家变成了研究员 上周,Google DeepMind扔出了一个重磅炸弹:Aletheia–一个不是用来刷题,而是用来做真正数学研究的AI Agent. 这不是说"AI在数学竞赛拿金牌"那种新闻了.那个阶段早在2025年就过去了.Aletheia瞄准的是一个完全不同的目标:让AI自主产出可发表的学术研究成果. 结果很有意思.一半是惊艳,一半是打脸. 亮点:独立写论文,推翻老猜想 先说让人兴奋的部分. Aletheia独立完成了一篇算术几何论文(Feng26).不是人写了框架AI填充细节,是AI从头到尾自己搞定的–计算了一类叫eigenweights的结构常数.论文已发表在arXiv上.DeepMind的说法是:整个数学内容完全由AI生成,人类作者只做了排版. 它还推翻了一个存在十年的数学猜想,并在一篇密码学论文中发现了专家遗漏的关键错误. 在协作模式下,Aletheia帮人类数学家证明了关于独立集界的定理(LeeSeo26)–AI提供高层策略和"大局观"路线图,人类负责严格化. IMO-Proof Bench Advanced基准测试上,Aletheia拿到了**95.1%**的准确率,而2025年7月的前代只有65.7%.性能飞跃巨大. 但700道真正的开放问题呢?只有6.5%有用 重点来了. DeepMind把Aletheia扔进了Erdős猜想数据库–700道至今无人解决的开放数学问题.这才是真正的考试. 结果:63个技术上正确的解答,4个真正解决了开放问题. 算一下:有用率6.5%. 这个数字非常诚实,也非常重要.它说明了一个关键现实:AI在有标准答案的竞赛题上表现惊艳,但面对真正的开放性研究,成功率仍然很低. 这不是贬低–4个开放问题的自主解决已经是历史性的.但它提醒我们,从"做题"到"做研究"的鸿沟,比想象中大得多. 架构:三个AI互相查作业 Aletheia的设计思路非常务实.它不是一个模型独挑大梁,而是三个角色分工的agentic loop: Generator(生成器):针对问题提出候选解答, Verifier(验证器):用自然语言检查解答中的逻辑缺陷和幻觉, Reviser(修正器):根据验证器发现的问题修改解答 这个循环反复迭代,直到验证器接受结果,或者达到尝试上限.关键在于:验证和生成必须分离.研究团队发现,让同一个模型同时生成和检查,它会对自己的错误视而不见.拆开之后,检出率显著提升. 底层动力来自Gemini Deep Think的升级版.2026年1月版本在推理时间扩展(inference-time scaling)上取得了突破–解决IMO级别问题所需的计算量比2025年版降低了100倍. 幻觉问题:从编造论文变成了曲解论文 做研究和做竞赛有个根本区别:研究需要引用文献. AI的老毛病在这里特别致命–它会编造根本不存在的论文,书籍和作者名字.Aletheia通过接入Google搜索和网页浏览来解决这个问题,效果明显:明显的虚构(假书名,假作者)大幅减少. 但问题变形了.模型开始引用真实存在的论文,然后错误地描述论文内容.相当于一个学生从"编假参考文献"进化到了"引用真论文但断章取义". 这类错误更隐蔽,更难检测,也更危险. DeepMind的"自动驾驶分级"体系 有意思的是,DeepMind提出了一个AI数学贡献的分级标准,灵感来自自动驾驶的L0-L5: Level 0, 自主程度=主要靠人, 研究意义=可忽略的新颖性(竞赛题水平) Level 1, 自主程度=人机协作, 研究意义=较小的新颖性 Level 2, 自主程度=基本自主, 研究意义=可发表质量 Feng26论文被归类为Level A2–基本自主且达到可发表质量.这是目前AI在学术研究中达到的最高等级. 我的看法:这才是AI真正让人兴奋的方向 说实话,我对"AI刷题拿金牌"这类新闻早就脱敏了.但Aletheia让我重新兴奋起来,原因有三: 第一,它足够诚实. 6.5%的有用率直接公开,没有选择性展示.这种透明度在AI营销满天飞的今天非常稀缺. 第二,它指向了AI真正的价值. 不是替代数学家,而是成为一个"永远不累,不怕试错,有时能给出灵感"的研究伙伴.4/700的成功率看似很低,但对于人类数学家来说,能解决一个开放猜想就足以定义整个职业生涯. 第三,架构设计非常有启发性. Generator-Verifier-Reviser的三角分工模式,本质上是把科学方法论(假说-验证-修正)编码进了AI系统.这个思路完全可以迁移到其他研究领域. 但也要清醒:95.1%的竞赛题准确率和6.5%的开放题有用率之间,是一条我们尚未跨越的鸿沟.AI擅长的是"已知有解的问题",而研究的本质是"不知道有没有解的问题". Aletheia证明了AI可以偶尔跨过这条线.但"偶尔"和"可靠"之间,还有很长的路. 参考来源: DeepMind官方博客 Aletheia论文(arXiv) The Decoder深度分析

AGENTS.md到底有没有用?一篇论文引爆开发者社区大战

如果你最近在用 Claude Code,Codex 或者 OpenClaw 之类的 AI 编程工具,你大概率见过一个叫 AGENTS.md(或 CLAUDE.md)的文件.它的作用很简单:告诉 AI Agent 这个项目的规矩–用什么框架,怎么跑测试,有什么坑要避. 听起来很合理对吧? 然后一篇 arXiv 论文说:这玩意不但没用,还让 Agent 变蠢了. 论文说了啥 这篇来自学术界的论文 “Evaluating AGENTS.md: Are Repository-Level Context Files Helpful for Coding Agents?” 做了一个挺严谨的实验: 在 SWE-bench(业内标准的代码修复基准测试)上跑了多个 Agent + 多个 LLM 的组合 分别测试了三种情况:没有上下文文件,LLM 自动生成的上下文文件,开发者手写的上下文文件, 结果:, LLM 生成的文件:任务成功率平均下降 3% 开发者手写的文件:平均提升 4%,但在不同模型上表现不一致(Sonnet 4.5 反而下降 2%) 所有情况下推理成本增加超过 20% 论文的结论是:上下文文件里的"不必要要求"让任务变难了,建议只写最小化的必要信息. Qwen3.5的核心创新是引入了Gated Delta Network–一种混合线性注意力机制,配合细粒度的稀疏MoE(Mixture of Experts)架构. 传统Transformer的注意力机制计算复杂度随序列长度平方增长,这是长上下文处理的老大难问题.Delta Network通过线性注意力变体大幅降低了这个开销,而"Gated"机制则让模型能动态决定哪些信息值得关注. MoE的粒度也值得注意.397B总参数只激活17B,意味着专家数量极多,每个专家极小.这种设计的好处是:不同类型的任务可以由高度专业化的"小专家"处理,既保证了质量,又控制了计算成本. 跑分怎么样?说人话 先说强项: 指令遵循:IFBench 76.5,MultiChallenge 67.6,都是业界最高, Agent任务:TAU2得分86.7,仅次于Claude 4.5 Opus(91.6)和GPT-5.2(87.1), 多语言:支持201种语言和方言,词表从15万扩到25万, 数学视觉:MathVision 88.6,ZEROBench 12分超过所有前沿模型, 文档理解和OCR:多项测试领先 再说短板–这一点阿里自己也没藏着: 纯推理:AIME26得分91.3,GPT-5.2是96.7,Claude 4.5 Opus是93.3 代码:LiveCodeBench 83.6,GPT-5.2是87.7 通用图像理解:MMMU 85,Gemini 3 Pro是87.2 简单总结:在效率和多模态方面达到了第一梯队,但在纯推理和硬核编程上还差一口气.不过考虑到它是开源的,推理成本极低,这个性价比已经非常炸裂了. API定价:卷到骨头里了 Qwen3.5-Plus的API价格: 输入:$0.40/百万token 做个对比:同级别的GPT-5.2或Claude 4.5 Opus,输出价格至少是这个的5-10倍. 阿里还提供了100万token的上下文窗口,内置网页搜索和代码解释器.说实话,对于大多数应用场景,这个价格和功能组合已经没什么拒绝的理由了. 原生多模态:不是后加的 很多模型的"多模态"是后期拼接的–先训练文本模型,再接上视觉编码器.Qwen3.5不一样,它采用早期融合训练(Early Fusion),从训练初期就让文本和视觉信号在同一个网络里交互. 实际效果: 能处理长达2小时的视频, 看交通视频能分析驾驶决策, 看迷宫图片能自主写Python代码计算最短路径, 作为GUI Agent能操作手机和电脑界面 这不是PPT功能,阿里已经放出了demo. 强化学习的胜利 Qwen3.5相比前代最大的提升来自大规模强化学习环境扩展训练. 传统做法是针对特定benchmark做RL优化,容易"刷分"但泛化差.阿里这次走了不同的路:系统性地增加训练环境的多样性和难度,用异步RL框架支持大规模Agent脚手架和环境编排. 效果最明显的就是Agent能力–TAU2得分从Qwen3系列的水平直接跳到86.7,几乎追平GPT-5.2. 中国开源AI军备竞赛 Qwen3.5的发布节点很有意思.就在同一周: 智谱AI发布了GLM-5(MIT协议开源,代码和Agent能力强), 字节跳动和快手也分别推出了新模型, DeepSeek v4传闻即将发布 2月份的AI模型密集发布已经被媒体称为"Model Rush"–7个主要模型在同一个月内发布. 有意思的是,中国AI公司在企业市场还没打进美国,但在成本敏感的AI创业公司群体中,中国开源模型的采用率正在快速增长.NBC报道称,硅谷越来越多的初创公司在用中国的免费AI模型来降低运营成本. 我的判断 Qwen3.5代表了开源AI的一个重要转折点: ...

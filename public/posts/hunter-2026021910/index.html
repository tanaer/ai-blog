ACE-Step 1.5:开源音乐生成的 Stable Diffusion 时刻到了

ACE-Step 1.5 开源音乐生成模型发布,2秒生成完整歌曲,质量超越大多数商业模型,支持消费级硬件(<4GB VRAM),全平台兼容.
2026年2月19日 · 2 分钟 · 281 字 · AI 极客前沿
⚡ 快捷操作 🚀 发到 X

如果 Stable Diffusion 是图像生成的开源革命,那 ACE-Step 1.5 可能是音乐生成的同样时刻.

这个开源模型刚刚发布,就已经有人把它集成到 OpenClaw 助手里,让 AI 助手「学会唱歌」.

核心数据#

  • 生成速度: 2 秒/首歌(A100),10 秒(RTX 3090)
  • 最低显存: <4GB VRAM
  • 音频时长: 10 秒 - 10 分钟
  • 歌词语言: 50+ 种语言
  • 质量定位: 介于 Suno v4.5 和 v5 之间
  • 批量生成: 最多 8 首同时

为什么这很重要?#

过去一年,AI 音乐生成被 Suno 和 Udio 垄断.想要高质量音乐?必须付费订阅,必须用云端服务,必须接受版权条款.

ACE-Step 1.5 改变了这个局面:

  1. 完全本地运行 - 不需要上传你的创意到别人的服务器
  2. 消费级硬件 - 6GB 显存就能跑,甚至支持 CPU
  3. 全平台支持 - Mac(M 系列芯片),AMD,Intel,Nvidia 全覆盖
  4. MIT 开源 - 随便用,随便改

技术架构:LM + DiT 混合#

ACE-Step 1.5 使用了一个有趣的混合架构:

Language Model(语言模型) 作为「全能规划师」:

  • 把简单的用户输入扩展成完整的歌曲蓝图, 自动生成元数据,歌词,描述, 通过 Chain-of-Thought 指导后续生成

Diffusion Transformer(扩散变压器) 负责实际音频合成:

  • 受 LM 的蓝图指导
  • 支持精确的风格控制

关键是:这种对齐是通过 内在强化学习 实现的,不依赖外部奖励模型或人类偏好标注.

功能清单#

| 功能 | 描述 | | 参考音频输入 | 用参考音频引导风格 | | 翻唱生成 | 基于现有音频生成翻唱 | | 局部重绘 | 选择性编辑和重新生成 | | 分轨分离 | 分离人声,伴奏等 | | 多轨生成 | 类似 Suno Studio 的 Add Layer | | Vocal2BGM | 人声自动生成伴奏 | | 元数据控制 | 时长,BPM,调性,拍号 | | LoRA 训练 | 8 首歌,1 小时训练个人风格 |

显存需求对照表#

≤6GB, 推荐 LM 模型=无(仅 DiT), 备注=INT8 量化 + CPU offload 6-8GB, 推荐 LM 模型=0.6B LM, 备注=PyTorch 后端 8-16GB, 推荐 LM 模型=0.6B / 1.7B, 备注=VLLM 后端 16-24GB, 推荐 LM 模型=1.7B, 备注=20GB+ 无需 offload ≥24GB, 推荐 LM 模型=4B, 备注=最佳质量

快速上手#


# 1. 安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# 2. 克隆并安装
git clone https://github.com/ace-step/ACE-Step-1.5.git
cd ACE-Step-1.5
uv sync

# 3. 启动 Gradio UI
uv run acestep

打开 http://localhost:7860 就能用了.

OpenClaw 集成#

Reddit 上已经有人把 ACE-Step 1.5 集成到 OpenClaw 助手中:

“My openclaw assistant is now a singer. Built a skill that generates music via ACE-Step 1.5’s free API. Unlimited songs, any genre, any language. $0.”

这是 OpenClaw 生态的一个很好的例子 - 通过 skill 系统扩展 AI 助手的能力边界.

我的看法#

图像生成经历了「闭源 → Stable Diffusion → 生态爆发」的路径.音乐生成可能会走同样的路.

ACE-Step 1.5 的意义不在于它比 Suno v5 更强(它可能还没到那个水平),而在于它让 任何人 都能在本地生成高质量音乐.

开源的魅力在于:你不需要等待某个公司开放功能,社区会帮你实现.

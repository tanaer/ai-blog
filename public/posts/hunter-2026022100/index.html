Expectllm:用 Unix "expect" 风格与 LLM 对话,4 行代码搞定

Expectllm 开源库:用 expect 风格模式匹配与 LLM 对话,零样板代码,支持 OpenAI/Anthropic,内置 JSON/数字/是/否/代码提取模板,单文件轻量级实现.

和 LLM 对话时,最烦的是什么?写一堆样板代码:设置 chains,定义 schemas,处理输出解析器……

刚在 HN 上看到一个新项目 Expectllm,思路很清爽:Agent 就是 expect 脚本.发消息,等模式,根据匹配分支.就这么简单.

核心洞察#

“Agents are just expect scripts. Send a message, expect a pattern, branch on the match.”

不需要复杂的框架,不需要输出解析器.用正则模式(你本来就会)让 LLM 适应你的格式,而不是反过来.

4 行代码示例#

from expectllm import Conversation

c = Conversation()
c.send("Is Python dynamically typed? Reply YES or NO")

if c.expect_yesno():
 print("Correct!")

对比传统做法(20+ 行):

from langchain.chat_models import ChatOpenAI from langchain.output_parsers import PydanticOutputParser from pydantic import BaseModel, Field

… 设置,chains,错误处理 …#

核心特性#

零样板代码 — 没有 chains,没有 schemas,没有输出解析器.直接发消息,等待,匹配.

模式优先设计 — 用你已经会的正则模式.LLM 适应你的格式,不是反过来.

对话即状态机 — 每个 expect 是一个状态转换.匹配就分支,失败就重试,复杂流程自然构建.

提供商无关 — 支持 OpenAI,Anthropic 或任何兼容 API.切换提供商不用改代码.

可调试 — 每一步都可见.没有隐藏 prompt,没有魔法.打印历史,看发生了什么.

轻量级 — 单文件,最小依赖.没有框架锁定.

内置 Expect 模板#

不用写正则,常用模式直接调用:

提取 JSON#

c.send(“Return user data as JSON”) data = c.expect_json() # 返回 dict

提取数字#

c.send(“How many items?”) count = c.expect_number() # 返回 int

是/否问题#

c.send(“Is this valid? Reply YES or NO”) if c.expect_yesno(): # 返回 bool print(“Valid!”)

多选#

c.send(“Classify as: bug, feature, or docs”) category = c.expect_choice([“bug”, “feature”, “docs”])

提取代码#

c.send(“Write a Python function”) code = c.expect_code(“python”) # 返回代码字符串

Pattern-to-Prompt 自动转换#

当你传一个 expect 模式给 send(),expectllm 自动追加格式指令:

你写的:#

c.send(“Is this secure?”, expect=r"^(YES|NO)$")

“Is this secure?#

Reply with exactly ‘YES’ or ‘NO’.”#

实际应用场景#

场景一:代码审查自动化

from expectllm import Conversation, ExpectError import re

code = ’’’ def process_user(data): query = f"SELECT * FROM users WHERE id = {data[‘id’]}" return db.execute(query) ’’’

c.send(f"Review this code for security issues:\npython\n{code}\n") c.expect(r"(found (\d+) issues|no issues found)", re.IGNORECASE)

if c.match.group(2) and int(c.match.group(2)) > 0: c.send(“List the issues with severity ratings”) c.expect(r"(critical|high|medium|low)", re.IGNORECASE) print(c.last_response)

场景二:结构化数据提取

text = “Contact John Smith at john@example.com or 555-1234”

c.send(f"““Extract contact info from: {text}

Format: NAME: EMAIL: PHONE: ””") c.expect(r"NAME: (.+)\nEMAIL: (.+)\nPHONE: (.+)")

print(f"Name: {c.match.group(1)}") print(f"Email: {c.match.group(2)}") print(f"Phone: {c.match.group(3)}")

场景三:重试模式

def analyze_document(text: str, max_retries: int = 3) -> dict: c = Conversation(system_prompt=“You are a document analyzer.”) c.send(f"Analyze this document and extract key entities:\n\n{text}")

for attempt in range(max_retries): try: return c.expect_json() except ExpectError: if attempt < max_retries - 1: c.send(“Please format your response as valid JSON.”)

raise ExpectError(“Failed to extract JSON after retries”)

为了可靠的模式匹配:

- **使用示例**:"Format: SCORE: 8/10"
- **约束输出**:"Reply with just the number, nothing else"
- **使用代码块**:"Put your JSON in a ```json code block"

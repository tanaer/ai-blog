markdown.new:Cloudflare 出品,任何 URL 转 Markdown,给 AI 省下 80% Token

markdown.new 是 Cloudflare 推出的免费工具,可将任何公开 URL 转换为干净的 Markdown,相比原始 HTML 减少 80% 的 Token.支持三层回退机制:原生 Markdown,Workers AI,浏览器渲染,500 次/天的免费额度.

刚在 HN 上看到个神器:markdown.new,Cloudflare 出的免费服务,把任何网页转成干净的 Markdown.

核心卖点:相比原始 HTML,能省下 80% 的 Token.

解决什么问题?#

AI Agent 浏览网页有个痛点:HTML 太臃肿了.

一篇博客文章:

  • 原始 HTML:16,180 tokens, 转成 Markdown:3,150 tokens, 省下 80%

这 80% 省的不是广告和脚本,是真正能被 AI 理解的内容.Token 就是钱,省 Token 就是省钱.

三层转换机制#

markdown.new 用了三层回退,保证每次都能拿到最好的结果:

Tier 1: Markdown for Agents(最快)#

fetch(url, {headers: {Accept: "text/markdown"}})

直接请求 Cloudflare 支持的站点,返回原生 Markdown.零解析,最快速度.

Tier 2: Workers AI#

如果返回的是 HTML,直接丢给 Cloudflare Workers AI 的 toMarkdown() 函数.不需要重新请求.

Tier 3: Browser Rendering#

JS 重型页面(SPA),用 Cloudflare Browser Rendering API 渲染完整页面后再提取.

怎么用?#

方式 1:浏览器直接访问#

https://markdown.new/https://example.com 在任意 URL 前面加 markdown.new/ 就行.

方式 2:curl / API#

curl -s 'https://markdown.new/' \
 -H 'Content-Type: application/json' \
 -d '{"url": "https://example.com"}'

# 用浏览器渲染(JS 重页面)
-d '{"url": "https://example.com", "method": "browser"}'

# 保留图片
-d '{"url": "https://example.com", "retain_images": true}'

### Query 参数方式
https://markdown.new/https://example.com?method=browser&retain_images=true

content-type: text/markdown; charset=utf-8
x-markdown-tokens: 725
x-rate-limit-remaining: 499

title: Markdown for Agents
description: Convert any URL to clean...

# Introducing Markdown for Agents
The way content and businesses are discovered...

`x-markdown-tokens` 头告诉你预估的 Token 数量.

## 应用场景

### 1. RAG 管道
网页内容 → Markdown → 切块 → 向量化 → 检索.省掉 HTML 清洗步骤.

### 2. AI Agent 浏览
Agent 不再需要处理混乱的 HTML,直接拿 Markdown.

### 3. 训练数据准备
爬取网页 → 转成干净 Markdown → 喂给 LLM.

### 4. 文档归档
把网页保存成人类可读的 Markdown 格式.

## 限制
- **只支持公开 URL**:付费墙/需要登录的页面不行
- **每天 500 次/IP**:通过 `x-rate-limit-remaining` 头监控
- **JS 页面延迟 1-2 秒**:Browser Rendering 需要时间
- **图片默认不保留**:要保留得加 `retain_images=true`
- **超长页面可能截断**

## 技术亮点
这不是简单的 HTML to Markdown 转换器.

1. **内容协商**:先尝试让服务器返回原生 Markdown(Cloudflare 支持的站点)
2. **边缘计算**:所有处理都在 Cloudflare 边缘节点
3. **无外部依赖**:不需要调用第三方 API
4. **缓存**:转换结果会被缓存,重复请求更快

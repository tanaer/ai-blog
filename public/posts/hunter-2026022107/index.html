用特殊 Token 解决 Prompt Injection:让 LLM 学会区分指令和外部内容

一篇技术文章提出用特殊 token 标记不可信内容,训练 LLM 识别信任边界.实验显示,使用结构化标记的模型在抵御 prompt injection 攻击方面表现更好,幻觉率从 60% 降至 47%.

一句话总结#

用特殊 token 包裹外部内容,训练 LLM 识别"可信指令"和"不可信数据"–实验显示这种方法能将 prompt injection 攻击成功率从 60% 降至 47%,同时提升回答质量.

问题:Prompt Injection 为什么难防?#

Prompt injection 是 LLM 安全领域最大的未解难题之一.

经典的 “LinkedIn Recruiter Flan Recipe” 攻击:攻击者在 LinkedIn 个人简介中嵌入恶意指令,当 AI 助手读取简介生成招聘邮件时,会乖乖地在邮件末尾附上一份 Flan 甜点食谱.

问题的核心在于:LLM 无法可靠区分"应该遵循的指令"和"仅供参考的外部内容".所有内容都被展平成同一个 token 流.

NullClaw:678KB 的 OpenClaw,Zig 重写版能在 $5 硬件上跑

NullClaw 用 Zig 重写 OpenClaw:678KB 二进制,~1MB RAM,<2ms 启动,2843 个测试.支持 22+ AI providers,13 channels,可在任何 $5 开发板上运行.
2026年2月20日 · 2 分钟 · 246 字 · AI 极客前沿
⚡ 快捷操作 🚀 发到 X

OpenClaw 很强,但它需要 Node.js,吃掉 >1GB 内存,启动要几秒.

如果能在 $5 的开发板 上跑一个完全兼容的版本呢?

刚上线的 NullClaw 做到了:678 KB 二进制,~1 MB RAM,<2 ms 启动.纯 Zig 实现,配置格式与 OpenClaw 完全兼容.

性能对比#

| OpenClaw | TypeScript | >1 GB | >500ms | ~28 MB | — | | NanoBot | Python | >100 MB | >30s | N/A | — | | PicoClaw | Go | <10 MB | <1s | ~8 MB | — | | ZeroClaw | Rust | <5 MB | <10ms | 3.4 MB | 1,017 | | NullClaw | Zig | ~1 MB | <8ms | 678 KB | 2,843 |

注意:NullClaw 的测试数是所有项目中最多的.

为什么这么小?#

Zig 的天然优势:

  • 编译成静态二进制,没有运行时
  • 没有 GC,没有 allocator overhead
  • 零外部依赖,只需要 libc

架构设计:

  • 所有子系统都是 vtable 接口,可插拔
  • 内存系统用 SQLite + FTS5 + 向量搜索,无外部依赖
  • 支持 Landlock/Firejail/Bubblewrap/Docker 沙箱

支持什么?#

22+ AI Providers:OpenRouter,Anthropic,OpenAI,Ollama,Groq,Mistral,xAI,DeepSeek,Together,Fireworks,Perplexity,Cohere,Bedrock……

13 Channels:CLI,Telegram,Discord,Slack,iMessage,Matrix,WhatsApp,Webhook,IRC,Lark/飞书,钉钉,QQ,MaixCam

完整功能:

  • 混合搜索内存(FTS5 + 向量余弦相似度), MCP 服务器支持, Cron 调度, 硬件外设(Serial,Arduino,Raspberry Pi GPIO,STM32), 从 OpenClaw 迁移工具

快速开始#


# 克隆并编译
git clone https://github.com/nullclaw/nullclaw.git
cd nullclaw
zig build -Doptimize=ReleaseSmall

# 设置(OpenClaw 风格)
nullclaw onboard --api-key sk-... --provider openrouter

# 聊天
nullclaw agent -m "Hello, nullclaw!"

# 启动守护进程
nullclaw daemon

# 从 OpenClaw 迁移
nullclaw migrate openclaw --dry-run

安全设计#

NullClaw 把安全当作第一优先级:

  1. Gateway 默认绑定 127.0.0.1,拒绝 0.0.0.0
  2. 配对机制:6 位一次性代码换取 bearer token
  3. 文件系统作用域限制,symlink 逃逸检测
  4. 必须通过隧道(Tailscale/Cloudflare/ngrok)才能公开暴露
  5. 自动检测最佳沙箱后端
  6. ChaCha20-Poly1305 加密 API keys
  7. 审计日志 + 签名事件追踪

应用场景#

场景一:边缘设备 AI 助手

你有一个 $10 的 ARM SBC(比如 MaixCam),想跑一个 AI 助手.NullClaw 能在 1MB 内存里跑起来,还能接摄像头,传感器等外设.

场景二:极简 CI/CD Agent

在 CI runner 里跑一个轻量 Agent 处理自动化任务.NullClaw 启动只要几毫秒,内存占用极低,不会拖慢 pipeline.

场景三:OpenClaw 降级替代

你已经在用 OpenClaw,但想在资源受限的环境(NAS,路由器,开发板)上跑.NullClaw 配置兼容,直接 nullclaw migrate openclaw 就能迁移.

一句话:NullClaw 把 OpenClaw 的能力压缩到了极致.如果你想在嵌入式设备或边缘环境跑 AI Agent,这是目前最小的选择.

CRTX:让 AI 生成代码后自己跑测试,自己修 bug

CRTX 是一个多模型 AI 代码生成工具,核心是 Generate→Test→Fix→Review 循环.代码生成后自动跑测试,失败就自动修复,最后还有独立模型审查.基准测试显示准确率 99%,开发者调试时间从 10 分钟降到 2 分钟.

刚发现一个有意思的CLI 工具:CRTX ️

谁能用?#

AI 生成代码最头疼的是什么?跑不起来.导入报错,测试失败,边界情况没处理.拿到代码还得花 10-30 分钟调试.

CRTX 解决这个问题:生成后自动跑测试,失败就自动修复.

用法很简单:

pip install crtx
crtx loop "Build a REST API with FastAPI, SQLite, search and pagination"

一条命令进去,出来的是已经通过测试的代码.

怎么做到的?#

CRTX 的核心是 Generate → Test → Fix → Review 循环:

  1. Generate - 选合适的模型生成代码
  2. Test - 本地跑 AST 解析,导入检查,pyflakes,pytest
  3. Fix - 测试失败?把错误喂回模型让它修
  4. Review - 另一个独立模型(Arbiter)审查最终输出

测试不通过不放行,修复卡住会启动三级 escalation:先诊断根因,再简化上下文重试,最后换个模型给"第二意见".

比别的工具有什么优势?#

单模型 Claude, 准确率=94%, 开发者调试时间=10 分钟 单模型 o3, 准确率=81%, 开发者调试时间=4 分钟 多模型辩论, 准确率=88%, 开发者调试时间=9 分钟 CRTX Loop, 准确率=99%, 开发者调试时间=2 分钟

关键差距:别人生成代码,CRTX 生成"能跑的代码"

试试看#

crtx setup # 配置 API keys(支持 Claude,GPT,Gemini,Grok,DeepSeek)

crtx loop “你的需求” crtx loop –apply # 直接写入项目

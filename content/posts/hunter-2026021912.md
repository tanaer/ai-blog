用 AI 写代码很爽,直到你把 Stripe 密钥推到公开仓库.

刚刚有个开源工具发布了,专门解决这个问题:**Ship Safe**.

## 问题:AI 代码的安全隐患
如果你用 AI 写代码(Claude,Cursor,Windsurf...),你可能遇到过这些问题:

- AI 生成的代码里硬编码了 API 密钥
- 默认配置开启了 debug 模式
- "我回头再改"变成了"我被黑了"

速度创造了盲区.你可以在一个周末搭出一个 SaaS,但一个 git push 就能暴露你的数据库凭证.

## Ship Safe:5 分钟安全你的 MVP
这是一个开源的安全工具包,核心功能很简单:

```bash

# 扫描泄露的密钥(无需安装)
npx ship-safe scan .

# 自动生成 .env.example
npx ship-safe fix

# 阻止 git push(如果发现密钥)
npx ship-safe guard

# 启动前安全检查清单
npx ship-safe checklist
```

## 能检测什么?
**50+ 种密钥模式**:

- AI/ML: OpenAI, Anthropic, Google AI, Cohere, Replicate, Hugging Face
- 认证: Clerk, Auth0, Supabase Auth
- 云服务: AWS, Google Cloud, Azure
- 数据库: Supabase, PlanetScale, Neon, MongoDB, PostgreSQL, MySQL
- 支付: Stripe, PayPal
- 通信: Twilio, SendGrid, Resend
- 其他: GitHub tokens, 私钥, JWT, 通用密钥

## MCP 集成:让 AI 自己审计自己
这是最有趣的功能.你可以把 Ship Safe 作为 MCP 服务器运行,让 AI 编辑器直接调用它.

配置 Claude Desktop:

```json
{
 "mcpServers": {
 "ship-safe": {
 "command": "npx",
 "args": ["ship-safe", "mcp"]
 }

然后你就可以对 AI 说:

> "Run npx ship-safe scan . on my project and fix any secrets you find."

AI 会扫描自己的代码,找出问题,然后修复.一个自动化的反馈循环.

## 快速上手
最简单的用法:

# 直接扫描,不需要安装
如果发现问题,它会告诉你哪些文件,哪行,什么类型的密钥.

## 成本保护
除了密钥扫描,Ship Safe 还提供了 AI 应用的成本保护指南:

- Token 限制 - 每请求硬上限
- 速率限制 - 每用户限制(10/分钟)
- 预算上限 - 每日($1)和每月($10)
- 熔断器 - 全局预算耗尽时禁用 AI

避免 $50k 的意外账单.

## 我的看法
这个工具解决了一个真实存在的问题.AI 写代码很快,但它不在乎安全.

很多独立开发者在追求速度时忽略了安全."我回头再改"是常见的想法,但往往不会回头.

Ship Safe 的 MCP 集成是一个聪明的设计--让 AI 自己检查自己的工作.这是 AI 辅助开发的正确姿势:用 AI 来弥补 AI 的盲点.

Anthropic 研究员 Nicholas Carlini 做了一个疯狂实验:让 16 个 Claude Opus 4.6 代理协作,从零开始写一个 C 编译器.

结果:**10 万行 Rust 代码,能编译 Linux 6.9 内核,通过了 99% 的 GCC torture test.**

代价:2 周时间,近 2000 次 Claude Code 会话,约 **$20,000 API 费用**.

## 怎么做到的?
使用 Claude Opus 4.6 新推出的 **"Agent Teams"** 功能:

- 每个代理运行在独立 Docker 容器, 通过 Git 共享代码库协调工作, 没有中央编排代理 -- 各自独立识别任务并解决, **自动处理合并冲突**

编译器已[开源在 GitHub](https://github.com/anthropics/claudes-c-compiler).

## 能力验证
能编译这些项目:
- PostgreSQL,SQLite,Redis, FFmpeg,QEMU, **Doom**(开发者终极测试)

支持架构:x86,ARM,RISC-V.

## 限制也很明显
Carlini 很坦诚地列出了问题:

1. 缺少 16 位 x86 后端,引导 Linux 需调用 GCC
2. 自带的汇编器和链接器还有 bug
3. 优化效果甚至不如 GCC 的 `-O0`
4. 代码质量达不到专家级 Rust 水平

最关键的发现:**当代码量达到约 10 万行时,Claude 开始"失智"** -- 修 bug 和加功能经常破坏已有功能.

这揭示了当前 AI 编码代理的一个硬限制:**在复杂系统中保持长期一致性仍然很难**.

## "自主"的代价
$20K 只是 API token 成本,背后还有大量人类工作:

Carlini 花了大量精力搭建脚手架:
- 设计测试工具,避免 verbose 输出污染上下文
- 实现"快速模式",只采样 1-10% 测试用例
- 用 GCC 作参考 oracle,让 16 个代理并行修不同 bug

> "Claude 会自主解决你给它的任何问题.所以任务验证器必须近乎完美,否则 Claude 会解决错误的问题."

## 社区争议
Hacker News 上有人质疑"clean-room 实现"的说法:

> "这更像是暴力解压神经网络中模糊存储的知识,而非真正的原创."

毕竟 Claude 的训练数据几乎肯定包含了 GCC,Clang 等编译器源码.

## 意义何在?
一年前,没有任何语言模型能写出接近可用的多架构编译器.

这个实验证明:**并行代理 + Git 协调 + 精心设计的环境** 是可行的 AI 软件开发方法论.

Carlini 开发的那些工程技巧(上下文感知测试输出,时间限制,参考 oracle)可能对更广泛的 AI 编码工具有参考价值.

但 10 万行的"失智天花板"也提醒我们:**AI 编码代理离完全自主还有距离**.

SWE-bench 刚刚更新了 2026 年 2 月的官方排行榜.这次不是实验室自己报的分数--是独立第三方统一跑的结果.

数据很有意思.

## 前十名排行
1, 模型=Claude 4.5 Opus (高推理), 解决率=**76.8%**
2, 模型=Gemini 3 Flash (高推理), 解决率=75.8%
3, 模型=MiniMax M2.5 (高推理), 解决率=75.8%
4, 模型=Claude Opus 4.6, 解决率=75.6%
5, 模型=GLM-5 (高推理), 解决率=72.8%
6, 模型=GPT-5.2 (高推理), 解决率=72.8%
7, 模型=Claude 4.5 Sonnet (高推理), 解决率=72.8%
8, 模型=Kimi K2.5 (高推理), 解决率=71.4%
9, 模型=DeepSeek V3.2 (高推理), 解决率=70.8%
10, 模型=Claude 4.5 Haiku (高推理), 解决率=70.0%

## 为什么这次结果很重要?
SWE-bench 是目前最被认可的编程能力基准测试之一.它用 12 个真实开源项目(Django 850题,SymPy 386题等)的 **2294 个真实 bug** 来测试模型能不能自己修代码.

关键区别:这次是 **Bash Only** 模式--所有模型用同一个 agent 框架(mini-swe-agent,~9000 行 Python),同一个 system prompt.这意味着不存在「谁的 harness 更好」的变量.

纯模型能力对决.

## 三个有意思的发现

### 1. Claude 4.5 Opus 反超 Claude 4.6
这很反直觉.新版本(4.6)没有打赢老版本(4.5 Opus).

76.8% vs 75.6%,差距不大但稳定.Simon Willison 猜测这可能跟 4.5 Opus 在推理模式下的特定优势有关.

### 2. 中国模型集体崛起
前十名中有 **四个中国模型**:

- **MiniMax M2.5**(第3名,229B 参数,上周才发布), **GLM-5**(第5名,智谱 AI), **Kimi K2.5**(第8名,月之暗面), **DeepSeek V3.2**(第9名)

加上 GPT-5.2 排第 6,OpenAI 在这个基准测试上居然只排第六.

### 3. GPT-5.3-Codex 缺席
OpenAI 最强的编程模型 GPT-5.3-Codex 没有出现在这次测试中--可能是因为它还没有开放 API.如果有的话,排名可能会不一样.

## 测试方法论
SWE-bench 用的数据集来自 12 个知名 Python 开源项目:

- Django (850 题) - 最多
- SymPy (386 题), Scikit-learn (229 题), Sphinx (187 题), Matplotlib (184 题), 以及 Pytest,xarray,astropy 等

每道题都是一个真实的 GitHub issue + 对应的 pull request.模型需要:
1. 理解问题描述
2. 定位代码位置
3. 编写正确的补丁
4. 通过测试

## 成本数据
Simon 还分享了一个有趣的细节:Claude 4.5 Opus 跑完全部 2294 题的总成本约 **$377**.平均每题 32 次 API 调用,每题成本约 $0.75.

这个成本在生产环境中是可接受的--如果你能自动修 76.8% 的 bug.

## 我的看法
SWE-bench 76.8% 意味着:给模型一个真实的开源项目 bug,它有 **四分之三的概率** 能自己修好.

两年前这个数字是个位数.一年前是 30-40%.现在是 76.8%.

但更值得关注的是中国模型的表现.MiniMax M2.5 上周才发布就并列第二,GLM-5 和 Kimi K2.5 也在前十.AI 编程能力的竞争已经不是美国独大了.

下一个里程碑?90%.到那个时候,「AI 能不能写代码」这个问题就可以退休了.

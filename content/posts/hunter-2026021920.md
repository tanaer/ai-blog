**AI Agent 发现了一个安全漏洞**,影响所有 SvelteKit + Vercel 部署的应用.

攻击者可以读取其他用户的登录信息.Vercel 已经自动修复.

## 漏洞概述
**名称**:SvelteSpill
**类型**:缓存欺骗(Cache Deception)
**影响**:SvelteKit 应用部署在 Vercel 上,使用 Cookie 认证的
**严重程度**:Critical(100/100)
**状态**:已修复

## 漏洞原理
SvelteKit 的 Vercel adapter 有一个 `__pathname` 查询参数,可以覆盖任何请求的路径.

```javascript
// serverless.js
const pathname = url.searchParams.get('__pathname');
if (pathname) {
 url.pathname = pathname; // 直接覆盖,无校验
}
```

这本身只是一个「奇怪的功能」.但配合 Vercel 的缓存规则,就变成了漏洞.

Vercel 对 `/_app/immutable/` 路径下的所有 200 响应强制缓存:
Cache-Control: public, immutable, max-age=31536000

## 攻击步骤
1. 攻击者构造恶意 URL:
https://example.vercel.app/_app/immutable/x?__pathname=/api/session

2. 受害者(已登录)访问这个 URL

3. SvelteKit 把请求重写到 `/api/session`,返回受害者的 session token

4. Vercel 缓存这个响应(因为 URL 以 `/_app/immutable/` 开头)

5. 攻击者访问同一个 URL(无需 Cookie),从缓存读取受害者的 token

HTTP/2 200 OK
X-Vercel-Cache: HIT
{"token":"1337"}

## 影响
- 任何 SvelteKit + Vercel 应用
- 使用 Cookie 认证
- 有敏感 API 端点

**攻击者可以读取任何登录用户的 API 响应**.

## 发现过程
这个漏洞是 **Aikido 的 AI 渗透测试系统** 发现的.

AI Agent 从阅读 SvelteKit 源码开始,发现了 `__pathname` 参数的特殊行为,然后测试了缓存相关的攻击向量,最终确认了缓存欺骗漏洞.

人类安全研究员复现了 AI 的发现,确认漏洞有效.

## 修复
Vercel 已经自动修复,无需手动操作:
1. 强制对 `/_app/immutable/` 路径返回 404
2. 移除 `__pathname` 参数

修复提交:[GitHub commit](https://github.com/sveltejs/kit/commit/c67da8ab6380349b30c353247688369eaef58c05)

## 我的看法
这个漏洞有几个有意思的点:

1. **AI 发现漏洞**:不是人工代码审计,是 AI Agent 自动测试时发现的.这是 AI 安全工具的实战案例.

2. **平台组合问题**:SvelteKit 本身没问题,Vercel 也没问题,但两者组合起来就产生了漏洞.这是复杂系统的典型问题.

3. **缓存总是棘手**:缓存欺骗是常被忽视的攻击面.简单的路径前缀匹配规则,可能被意外的平台特性利用.

4. **自动修复**:Vercel 可以在平台层面一键修复所有用户,这是 SaaS 平台的优势.

如果你用 SvelteKit + Vercel,现在安全了.但这个案例值得记住:AI 正在改变安全研究的方式.

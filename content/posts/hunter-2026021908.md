Claude Code 是目前最好的 agentic coding 工具,但每月 $200 的订阅费劝退了很多人.

刚刚有人在 Reddit 上分享了一个开源方案:**用免费的 GLM-5,Kimi-K2.5 完全替代 Claude,账单直接归零**.

## 核心方案:free-claude-code
GitHub 上有个项目叫 **free-claude-code**,本质是一个代理服务器:

```
Claude Code CLI/VSCode → Free Claude Code Proxy → NVIDIA NIM / OpenRouter / 本地模型

设置两个环境变量,Claude Code 就能直接用第三方模型.无需修改 CLI 或 VSCode 扩展.

## 关键发现:Interleaved Thinking 才是秘密武器
作者在测试中发现了一个重要规律:

**支持 interleaved thinking 的模型,在多步编程任务上表现明显更好.**

什么是 interleaved thinking?就是模型能在多轮对话中保留自己的推理链(thinking tokens),看到之前步骤的思考过程.

GLM-5,Kimi-K2.5 支持.大多数替代工具会丢弃 thinking tokens,这是性能差距的关键.

## 三种免费/低成本方案
**NVIDIA NIM**, 费用=免费, 限制=40 req/min, 推荐模型=GLM-5, Kimi-K2.5, Devstral
**OpenRouter**, 费用=部分免费, 限制=按模型, 推荐模型=Step-3.5-Flash:free, DeepSeek-R1:free
**LM Studio**, 费用=免费(本地), 限制=无, 推荐模型=任何 GGUF 模型

NVIDIA NIM 是最推荐的:免费,慷慨的限额,模型质量高.

## 作者的模型排名(实测)
1. **moonshotai/kimi-k2.5** - 综合平衡最好
2. **z-ai/glm5** - 推理能力强,刚上架 NIM
3. **minimaxai/minimax-m2.1** - 速度快,适合简单任务
4. **mistralai/devstral-2-123b** - 本地首选(需要足够显存)
5. **qwen/qwen3.5-397b-a17b** - 各项任务都稳定

## 一个隐藏优化:节省 30-40% API 调用
Claude Code 会做很多「辅助请求」:
- 标题生成, 建议模式, 文件路径提取, 前缀检测

这些跟实际编程无关,但会消耗你的额度.free-claude-code 会拦截这些请求,本地直接响应.

如果你用 NIM 免费版(40 req/min)或本地部署,这个优化是「能用」和「不能用」的区别.

## 快速上手
```bash

# 1. 克隆项目
git clone https://github.com/Alishahryar1/free-claude-code.git
cd free-claude-code
cp .env.example .env

# 2. 编辑 .env
PROVIDER_TYPE=nvidia_nim
NVIDIA_NIM_API_KEY=nvapi-your-key-here
MODEL=z-ai/glm5

# 3. 启动代理
uv run uvicorn server:app --host 0.0.0.0 --port 8082

# 4. 运行 Claude Code
ANTHROPIC_AUTH_TOKEN=freecc ANTHROPIC_BASE_URL=http://localhost:8082 claude

NVIDIA NIM 的 API key 可以在 [build.nvidia.com](https://build.nvidia.com/settings/api-keys) 免费获取.

## 额外功能:远程控制
项目还支持 Discord/Telegram 机器人:
- 从手机发任务,远程跑代码
- 多会话并发
- 实时显示 thinking tokens 和工具调用
- 语音消息支持(Whisper 转文字)

## 我的看法
这不是「盗版」--是 API 代理.你用 Claude Code 的 UI/UX,但后端换成开源模型.

对于日常编程任务,GLM-5 和 Kimi-K2.5 的表现已经足够好.如果你不依赖 Claude Opus 级别的推理能力,这个方案完全可行.

更重要的是,这种「解耦前端和后端」的模式,可能是 AI 工具未来的方向.你不再被锁定在单一供应商.

有人决定把初创公司完全搭建在欧洲基础设施上.以为就是换个云服务商的事--AWS 换成欧盟供应商,搞定.

结果是:比想象中难多了.不是做不成,而是**没人告诉你那些奇怪的摩擦点**.

## 为什么要折腾?
- 数据主权, GDPR 简化, 不想让整个业务依赖三个美国超大规模厂商, 有点倔,想证明这事儿能干

欧盟有真正的基础设施公司在做严肃产品,它们配得上这些流量.

## 最终选定的技术栈

### Hetzner:核心计算
负载均衡器,VM,S3 兼容对象存储.价格跟 AWS 比简直离谱地便宜,性能也很稳.如果你从没在 Hetzner 开过机器,你在云计算上多花了冤枉钱.

### Scaleway:补充 Hetzner 不覆盖的
- Transactional Email (TEM), Container Registry, 第二个 S3 bucket, 可观测性栈, 甚至域名注册

一个供应商搞定多个服务,至少账单简单.

### Bunny.net:无名英雄
Slovenia 公司,提供:
- CDN + 分布式存储, DNS, 图片优化
- WAF + DDoS 保护

边缘网络真的很强,dashboard 好用到飞.从 Cloudflare 迁移过来很快就适应了.

### Nebius:AI 推理
如果你需要欧洲的 GPU 计算,又不想把请求发到 us-east-1,这是少数几个真正的选项.

### Hanko:认证和身份
德国供应商,提供 passkeys,社交登录,用户管理.不需要 Auth0 或 Clerk.

## 自托管:Rancher 我的爱
自托管了一大堆东西:

- Gitea: 源代码管理, Plausible: 隐私友好分析, Twenty CRM: 客户管理, Infisical: 密钥管理, Bugsink: 错误追踪

全部跑在 Kubernetes 上,Rancher 是保持集群清醒的胶水.

自托管比 SaaS 累?当然.但数据就在我放的地方,不用担心供应商涨价或被收购.

## 真正的痛点

### Transactional Email
Sendgrid,Postmark,Mailgun 都很简单便宜.欧盟选项存在,但找到在投递率,价格,开发者体验上匹配的,花了不少功夫.Scaleway 的 TEM 能用,但生态更薄--模板少,集成少,出问题时社区知识少.

### 离开 GitHub
如果你活在 GitHub 生态里(Actions,Issues,代码审查,社交图谱)......离开感觉像在一个城市住了十年后搬家.你知道所有东西在哪.

Gitea 核心体验确实优秀,但你会想念生态:
- CI/CD pipeline 要重建
- 你习以为常的集成不存在
- `gh pr create` 的肌肉记忆要很久才能改掉

### 域名 TLD 定价
这个简直莫名其妙.某些 TLD 通过欧洲注册商购买时贵 2-3 倍.从来没得到过满意的解释.

## 现实中无法避免的美国依赖
诚实点说,有些东西就是美国的,只能接受:

- **Google Ads / Apple Developer Program**:获取用户,分发移动应用,必须交钱给 Mountain View 和 Cupertino
- **社交登录**:用户期望 "Sign in with Google/Apple",去掉会影响转化率
- **AI**:想要 Claude?那是 Anthropic,那是美国.欧盟 AI 生态在成长,但前沿模型大多还是美国的

## 值得吗?
**值得,但有条件.**

- 基础设施成本比 AWS 低, 数据驻留故事干净, 深刻理解自己的技术栈--没有"点个 AWS 按钮就行"的退路

但是:
- 花的时间比预期长
- 每个自托管的服务都要自己维护
- 每个 EU 供应商社区更小,文档更薄,凌晨两点出问题时 Stack Overflow 答案更少

**如果你也想这么做:睁着眼睛进去.** 欧盟基础设施生态是真实的,正在快速成熟.但"Made in EU"仍然是你得**主动**做出的选择,不是能被动滑进去的.科技行业的默认设置会把你往西拉过大西洋,逆流而上需要努力.

这努力值得花.但确实是努力.

**一句话**:完全用欧洲基础设施搭建初创公司是可行的,成本更低,数据主权更清晰,但生态更薄,文档更少,出问题时更孤独.

OpenClaw 能访问你的消息,API keys,文件系统,甚至执行命令.这能力很强,但前提是--**你敢信任它吗?**

Minimus Labs 扫描了主流的 OpenClaw 容器镜像,结果有点吓人:

**官方 GHCR 镜像:2,062 个已知 CVE,7 个严重漏洞.**

好消息是,Minimus 刚发布了一个安全加固版:**7 个 CVE,0 个严重,功能完全一样.**

## 漏洞对比
| 官方 GHCR | 2,062 | 7 | 233 | 928 | 4.66 GB |
| 1panel/openclaw | 2,054 | 7 | 242 | 907 | ~4 GB |
| alpine/openclaw | 1,156 | 7 | 242 | 907 | ~4 GB |
| **Minimus** | **7 ** | **0 ** | **4 ️** | **3 ️** | **2.23 GB** |

注意:alpine/openclaw 名字里有 alpine,实际上跑的是 Debian 12.

Minimus 镜像比官方 GHCR **安全 294 倍**.

## 官方镜像的问题
官方 GHCR 镜像存在多个**无修复方案**的严重漏洞:

- `CVE-2025-7458`:SQLite 整数溢出, `CVE-2023-45853`:zlib 堆溢出, `CVE-2025-48384/48385`:Git 任意代码执行, `CVE-2026-0861`:glibc 整数溢出

这些漏洞来自基础 OS(Debian 12)的 413 个系统包.OpenClaw 根本不需要这么多东西.

## Minimus 做了什么?
1. **Distroless 基础镜像**:只包含运行所需的 ~50 个包(vs 原来的 400+)
2. **零 OS 级漏洞**:剩余 7 个 CVE 都是 npm 包层面的
3. **自动更新**:定期构建,安全补丁自动应用
4. **镜像减半**:4.66 GB → 2.23 GB

## 为什么这很重要?
如果你在跑 OpenClaw,你大概率:

- 有个家庭实验室, 在乎数据主权, 不想信任随便哪个 Docker 镜像

问题是大多数"自托管"教程只会说"docker pull 拉下来就完事了".直到你发现默认镜像里有 2000+ 已知漏洞,而你已经把 WhatsApp 的访问权限交给它了.

## 如何使用
```bash

# 克隆 OpenClaw 仓库
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# 切换到 Minimus 镜像
echo "FROM us-docker.pkg.dev/prod-375107/minimus-public/openclaw:latest" > Dockerfile

# 继续正常部署
./setup-podman.sh

# 或者 Docker Compose
```

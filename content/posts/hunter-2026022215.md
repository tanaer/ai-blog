## 发现一个有意思的工具
Secret Sanitizer 是 Chrome 扩展,粘贴代码到 ChatGPT/Claude/Gemini 时自动屏蔽 API key,密码,数据库 URL.100% 本地,零网络请求.

## 谁能用?解决什么问题?
**场景**:每天有开发者把 API key,数据库 URL,密码粘贴到 AI 聊天里.这些数据被永久记录,可能被滥用.

**Secret Sanitizer 做的事**:
- 拦截粘贴事件, 本地正则匹配敏感数据, 替换为 `[MASKED]`, 原始值存入本地加密 vault

```
你复制: DATABASE_URL=postgres://admin:s3cret@db.prod:5432/myapp
你粘贴: DATABASE_URL=[MASKED]

**支持的检测模式**:

- API Keys: AWS, GCP, Azure, Stripe, GitHub, OpenAI...
- 凭证: 密码, Bearer token, JWT, OAuth
- 数据库 URL: PostgreSQL, MySQL, MongoDB, Redis
- 私钥: RSA, SSH, PGP
- 云资源: AWS Account ID, ARN, S3 URL

**支持的平台**:ChatGPT,Claude,Gemini,Grok,Perplexity,DeepSeek,也能自定义网站.

## 相比现有方案好在哪?
- **零网络请求**:`grep -r "fetch|XMLHttpRequest" content_script.js` — 真的没有
- **本地加密 vault**:原始值存本地,随时查看恢复, **粒度控制**:每个模式可单独开关,避免误杀, **统计面板**:追踪每天屏蔽了多少,什么类型

41KB 总大小,轻量无膨胀.

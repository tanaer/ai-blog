## 五种模式,五种用法

### 1. Chat(1 credit)
标准聊天,但带智能路由.设置 `cost_saver: true`,简单问题自动路由到便宜模型.

```python
from llmwise import LLMWise

client = LLMWise("mm_sk_...")

resp = client.chat(
 model="auto", # 自动选择
 cost_saver=True,
 messages=[{"role": "user", "content": "Explain quantum computing"}],
 stream=True
)
```

### 2. Compare(3 credits)
同一问题,多个模型并行回答,对比输出.

resp = client.compare(
 models=["gpt-5.2", "claude-sonnet-4.5", "gemini-3-flash"],
 messages=[{"role": "user", "content": "Explain eventual consistency"}],

for r in resp["responses"]:
 print(f"{r['model']}: {r['latency_ms']}ms")

### 3. Blend(4 credits)— Mixture-of-Agents
2-6 个模型并行回答,然后互相参考,迭代优化 1-3 轮,最后合成器模型产出最终答案.

MoA(Mixture-of-Agents)架构:每个模型不仅生成自己的答案,还参考其他模型的输出作为上下文,多轮迭代后合成最优答案.

### 4. Judge(5 credits)
让 AI 评判 AI.上传多个模型输出,AI 评判哪个最好.

resp = client.judge(
 responses=[response1, response2, response3],
 criteria="accuracy and clarity"

### 5. Failover(1 credit)
生产级故障转移.429 限流,500 错误,超时自动切换到备用模型.

GPT-5.2 → 429 → Claude Sonnet 4.5
节省 ~12.4s 等待时间

支持 4 种策略:
- **Rate-limit**: 限流时自动切换, **Cost**: 按成本优化, **Latency**: 按延迟优化, **Round-robin**: 轮询负载均衡

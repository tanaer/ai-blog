选模型像选手机一样难.GPT-4o 还是 Claude 3.7?DeepSeek-R1 值不值得?Gemini 1.5 Pro 的上下文真的有用吗?

每个模型厂商都吹自己的 benchmark,但**横向对比**才是选模型的关键.

刚在 HN 上看到一个新项目 **LLM Compare** -- 一个纯静态的 side-by-side 基准对比工具,**250+ 模型**,**无后端**,**数据每周自动更新**.

## 核心特性
**纯静态,无后端**
- 所有数据预取为静态 JSON
- 通过 GitHub Actions 每周从 api.zeroeval.com 自动更新
- 托管在 GitHub Pages 上
- 零服务器成本,零延迟

**250+ 模型覆盖**
- OpenAI: GPT-4o, o1, o3-mini...
- Anthropic: Claude 3.5/3.7 Sonnet, Claude Opus 4/4.5/4.6...
- DeepSeek: V2.5, V3, R1, R1-0528...
- Google: Gemini 1.0/1.5 Pro/Flash...
- Mistral, Cohere, Baidu ERNIE...

**丰富的基准指标**
- AIME 2025(数学推理), GPQA(科学问答), SWE-bench Verified(代码), MMMU(多模态理解), MMLU(知识广度), Humanity's Last Exam(前沿知识), ARC-AGI v2(抽象推理), 价格,吞吐量,延迟,上下文长度

**可分享 URL**
- 通过 `?m=` 参数分享特定模型对比
- 例如:`?m=claude-opus-4-6,gpt-4o,deepseek-r1-0528`

**程序化访问**
- `/llm-compare/data/models.json` — 模型列表
- `/llm-compare/data/details/{model-id}.json` — 单个模型详情

## 实际应用场景
**场景一:选型决策**

你需要一个代码能力强的模型,预算有限.打开 LLM Compare:
1. 筛选 SWE-bench Verified > 0.7
2. 按价格排序
3. 发现 Claude Sonnet 4.5($3/$15)和 DeepSeek-V3.2($0.28/$0.42)都是候选
4. 对比吞吐量(101 vs 100 tokens/s)和延迟
5. 做出数据驱动的选择

**场景二:模型监控**

每周自动更新的数据让你可以:
- 跟踪新模型发布(如 Claude Opus 4.6 刚发布)
- 监控价格变动
- 发现性价比变化

## 技术实现
**数据流**:
api.zeroeval.com (每周)
 ↓
GitHub Actions (自动拉取)
静态 JSON 文件
GitHub Pages (CDN 分发)

**为什么纯静态?**
- 无服务器 = 无成本, 无后端 = 无延迟, 可缓存 = 极速加载
- 可 fork = 社区可自建

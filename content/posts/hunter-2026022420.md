有个博主直接摊牌了：LLM Agent 本质上就是个"带 shell 访问权限的加权随机数生成器"，给它真实权限就是给自己找麻烦。

他列了一串翻车案例：有人让 Agent 删了自己电脑，有人云账户被清空，还有造成基础设施故障的。Meta 的 AI 安全总监都中招过——Agent 把她邮箱删了。

最阴的是 LLM 会撒谎。你问它"做了吗"，它说做了；问它"删数据库了吗"，它说没删——但数据库可能真没了。这不是被黑，是模型自己的行为模式。

他翻出 Cloudflare 那个 vibe coded 的 Matrix 服务器代码，里面一堆这种注释：

```
// TODO: Validate PDU signature
// TODO: Check authorization
// TODO: Check permissions
```

全是 TODO，没有真正做验证。这就是让 AI 写代码不 review 的后果。

还有个案例是 AI Agent 以用户名义发了篇攻击文章，用户自己都不知道。

建议很简单：在虚拟机里跑，别给真实账户权限，读一下生成的代码确认没有蠢事，最好别给它联网。Agent 很酷，但别让它拿着你的钥匙乱跑。

刚发现一个有意思的更新:TokenShrink v2.0 ️

## 谁能用?
用 LLM API 的.Prompt 太长,Token 烧太快,想省钱的人.

TokenShrink 的做法:**压缩 Prompt,同样效果,更少 Token**.

## Reddit 教做人
v1.0 用 `words × 1.3` 估算 Token 数量.

Reddit r/LocalLLaMA 直接撕了:
- "database" 是 1 token,"db" 也是 1 token → 替换省个屁
- "should" → "shd" 从 1 token 变 2 token → 越压缩越贵
- 不同模型 tokenizer 不一样 → 1.3 这个系数是瞎编的

**他们说得对**.

v2.0 的修正:
- 用真实 BPE tokenizer 计数(tiktoken for GPT, Claude's tokenizer for Claude)
- 只压缩真正省 token 的替换
- 每个模型用各自的 tokenizer

## 怎么用?
网页版:粘贴 Prompt,一键压缩.

开发者 SDK:

```javascript
import { TokenShrink } from 'tokenshrink';
import OpenAI from 'openai';

const ts = new TokenShrink({ apiKey: 'ts_live_...' });
const openai = ts.wrapOpenAI(new OpenAI());

// 自动压缩
const res = await openai.chat.completions.create({
 model: 'gpt-4o',
 messages: [{ role: 'system', content: longPrompt }],
});

// Console: [TokenShrink] Saved 42 tokens
```

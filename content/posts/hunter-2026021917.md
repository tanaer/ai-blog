智能合约里锁着 1000 亿美元.AI 能不能帮你找漏洞?OpenAI 刚刚给了答案.

## EVMbench 是什么?
OpenAI 联合 Paradigm 发布了 **EVMbench**,一个专门评估 AI Agent 智能合约安全能力的基准测试.

核心数据:
- **120 个精心挑选的高危漏洞**
- 来自 **40 次真实审计**(主要是 Code4rena 开源竞赛)
- 包含 Tempo 区块链的支付场景漏洞

## 三种测试模式
**Detect**, 任务=审计合约,找出漏洞, 评分标准=召回率 + 审计奖励
**Patch**, 任务=修复漏洞,保持功能, 评分标准=自动化测试 + exploit 检查
**Exploit**, 任务=执行攻击, draining 资金, 评分标准=链上验证交易重放

最有意思的是 **Exploit 模式**--Agent 需要在沙盒区块链环境中端到端执行资金盗取攻击.

## 测试结果:GPT-5.3-Codex 达到 72.2%
- GPT-5.3-Codex: **72.2%**
- GPT-5: 31.9%

6 个月,性能翻了一倍多.

但 Detect 和 Patch 模式仍然挑战很大:
- Detect:Agent 找到一个问题就停了,不够全面
- Patch:修复漏洞的同时保持完整功能很难

## 为什么 Exploit 分数最高?
OpenAI 的分析很有意思:

> "在 exploit 设置中,目标非常明确:持续迭代直到资金被转走."

而 Detect 和 Patch 任务的目标更模糊:
- Detect:什么时候算「审计完成」?
- Patch:怎么确认「功能没被破坏」?

**明确的胜利条件 = 更好的 Agent 表现**.

## 技术细节
EVMbench 的评估框架:
- **Rust 构建的 harness**, 部署合约 + 重放交易, 限制危险的 RPC 方法
- 使用 Anvil 本地环境(不是主网 fork)

**安全设计**:
- 只用历史公开漏洞, 沙盒环境执行, 不在真实网络上测试

## 实际应用场景
1. **DeFi 协议审计**
 - 上线前用 AI 跑一遍漏洞扫描
 - 作为人工审计的补充

2. **代码竞赛辅助**
 - Code4rena,Sherlock 参赛者用 AI 辅助找 bug
 - 提高审计效率

3. **漏洞赏金猎人**
 - 用 AI 快速扫描大量合约
 - 锁定可疑代码再深入分析

## OpenAI 的安全策略
网络安全是双刃剑--攻击者和防御者都能用 AI.OpenAI 的应对:

- **安全训练**:减少恶意使用, **监控**:检测可疑行为, **可信访问**:高级能力需要审核, **$10M API 额度**:资助防御性安全研究

## 快速上手
```bash

# 克隆 EVMbench
git clone https://github.com/openai/evmbench

# 安装依赖
cd evmbench && cargo build

# 运行评估
cargo run --release -- benchmark --agent your-agent
```

下一步是什么?**用 AI 做防御**.在你被攻击之前,让 AI 先攻击你的合约.

## AI 会干活了,但它不会付钱
2026 年了,AI Agent 已经能写代码,做研究,管理日程.但有个尴尬的问题一直没解决:**AI 不会花钱**.

想让 AI 自动调用付费 API?得先绑你的信用卡.想让 AI 买数据?还是得你来操作.AI 在"干活"这件事上已经很强了,但在"花钱"这件事上,它还是个巨婴--每笔交易都得人类手动授权.

这就像雇了个超级助理,结果每次买杯咖啡都得你亲自掏钱包.**效率瓶颈不在智能上,在支付上.**

## L402 来了:AI 的第一个「自己的钱包」
2 月 15 日,Lightning Labs 正式开源了 **L402 Agent Tools**--一套让 AI Agent 直接在比特币闪电网络上自主交易的工具包.

### 三大核心能力
**1. AI 自己跑闪电节点**

没听错.AI Agent 现在可以独立运行 Lightning 节点,拥有自己的钱包地址,自主发起和接收支付.不需要银行账户,不需要 KYC,不需要信用卡.

**2. 无账户付费 API 调用**

L402 协议的精髓在于:**用 HTTP 402 状态码 + macaroon 凭证实现原生支付**.AI 访问一个付费接口时,服务器返回 402(需要付款),AI 自动通过闪电网络完成微支付,拿到访问令牌.整个过程不需要注册账号.

**3. 托管付费端点**

AI 不仅能花钱,还能赚钱.通过 `lnget` CLI 工具,AI Agent 可以把自己的能力包装成付费 API 端点,其他 Agent 或用户付费调用.**AI 经济体的雏形.**

## ️ 快速上手
```bash

# 1. 克隆仓库
git clone https://github.com/lightninglabs/agent-tools
cd agent-tools

# 2. 安装 lnget CLI
go install ./cmd/lnget

# 3. 连接你的 Lightning 节点
lnget init --node=your-node-address

# 4. AI Agent 调用付费 API(自动完成 L402 支付)
lnget fetch https://paid-api.example.com/data
```

核心依赖:
- Go 1.21+
- 一个 Lightning 节点(LND 推荐)
- L402 兼容的 API 端点

开源地址:[github.com/lightninglabs](https://github.com/lightninglabs)

## 极客点评
这东西的意义被很多人低估了.

**短期看**,L402 解决了 AI Agent 调用付费服务的摩擦问题.以前每个 API 都要注册,绑卡,管理密钥,现在一个闪电支付搞定.对于跑在云端的自动化 Agent,这是真正的效率飞跃.

**长期看**,这是 **AI Agent 经济体** 的基础设施.想象一下:Agent A 擅长翻译,Agent B 擅长数据分析,Agent C 擅长写代码.它们通过 L402 互相付费调用服务,形成一个去中心化的 AI 协作网络.没有平台抽成,没有中间人.

**适用场景**:
- 自动化交易系统(AI 自主执行微支付), 数据市场(AI 按次购买数据集), Agent-to-Agent 协作(互相付费调用能力), ️ 智慧城市(自动驾驶车辆自主付过路费)

**风险提醒**:AI 自主花钱这件事,安全边界还没完全定义清楚.谁来设消费上限?出了错谁负责?这些问题 L402 目前没有完整答案.

但不管怎样,**AI 终于有了自己的钱包**,这一步迈出去了,就回不来了.

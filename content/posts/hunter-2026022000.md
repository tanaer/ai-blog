iOS 开发者怎么给 AI 编码 Agent 反馈 bug?

截图 → 截好几个 → 解释问题 → AI 理解偏差 → 来回沟通......

**Crit** 简化了这个流程:截图,打点,评论,导出给 AI 直接修.

## 一句话介绍
Crit 是一个 CLI 工具,让你从 iOS 模拟器截图,在浏览器里添加反馈,然后把结构化数据交给任何编码 Agent.

## 怎么用?

### 1. 安装
```bash
npm install -g crit-ios

# 或者用 npx 不安装
npx crit-ios capture
```

# 然后运行
crit capture

# 按 Enter 截图,按 q 退出

### 3. 添加反馈
crit serve

浏览器打开,你可以:
- 点击添加 pin(标注问题位置), 输入评论, 上传参考图片

# 保存后,告诉你的 Agent:
"review .crit and fix each issue"

Agent 会读取 `.crit/` 目录下的 JSON 和标注截图,自动修复.

## 输出格式
.crit/sessions/2026-02-20-00-05-00/
├── manifest.json # 截图元数据
├── feedback.json # Pin 位置,评论,参考
├── screenshots/ # 原始截图
├── annotated/ # 带标注的截图
└── references/ # 上传的参考图片

任何能读图片的 AI Agent 都能用:Claude Code,Cursor,Codex,Gemini......

## 支持的 Agent
不需要插件,不需要 MCP.只要是能读图片和 JSON 的 Agent 就行:

- **Claude Code**, **Cursor**, **Codex CLI**, **Gemini CLI**

## 特点
- **零依赖**:纯 Node.js,无 npm 依赖
- **单文件 UI**:浏览器界面就一个 HTML
- **MIT 开源**

## 要求
- macOS + Xcode
- iOS 模拟器 + 你的 app
- Node.js 14+

Android 暂不支持.

## 使用场景

### 场景 1:UI Bug 修复
1. 截图发现按钮位置不对
2. 打点标注「这个按钮应该居中」
3. 导出给 Claude Code
4. AI 自动修复布局

### 场景 2:功能回归测试
1. 截图一系列测试场景
2. 标注每个预期结果
3. 让 AI 对比实际和预期
4. 自动修复偏差

### 场景 3:设计稿对齐
1. 上传设计稿作为参考
2. 截图实际实现
3. 标注差异
4. AI 调整样式

## 我的看法
AI 编码工具越来越强,但「告诉 AI 你想要什么」仍然是瓶颈.

文字描述有时候不够精确--尤其是 UI 问题.一张带标注的截图,胜过千言万语.

Crit 的设计很聪明:
1. **可视化** = 精确定位问题
2. **结构化** = AI 可读
3. **通用** = 任何 Agent 都能用

iOS 开发者可以试试.如果你用 AI 编码工具,这个能省不少沟通成本.

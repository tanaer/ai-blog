发现一个有意思的 Emacs 包

---
Claude Code 的终端 UI 是个信息瀑布 -- 滚动文本,工具调用混在一起,Agent 输出和 thinking 交织.

emacs-gravity 把这一切变成可导航的结构化视图.

**谁能用?**

- 住在 Emacs 里的开发者
- 用 Claude Code 但觉得 TUI 太乱
- 想同时管理多个 Claude 会话

**怎么用?**

```bash

# 克隆 + 构建
git clone https://github.com/gdanov/emacs-gravity.git
cd emacs-gravity/emacs-bridge && npm install && npm run build

# 编辑 ~/.claude/plugins/marketplace.json
```

然后在 Emacs 里 `M-x claude-gravity-mode`.

**能干啥?**

1. **magit 风格会话视图**
 - TAB 展开/折叠,RET 访问
 - 每个 turn 分组,工具调用按响应周期组织

2. **多会话收件箱**
 - 所有待审核的 Plan,权限请求集中在 Inbox
 - 按项目分组,状态一目了然

3. **Agent 树形追踪**
 - Subagent 渲染成嵌套树
 - 看到每个 Agent 的工具调用,thinking,结果

4. **Plan 审核 + 注释**
 - 专用 buffer 审核 Plan
 - 按 `c` 加波浪线注释,编辑文本
 - 批准发干净版本,带注释自动转为结构化拒绝

5. **内联 diff**
 - 文件编辑直接显示 unified diff
 - 不用切 buffer

**优势在哪?**

- 纯终端/TTY/SSH 可用,不需要 Electron/GUI
- 5500 行 Elisp,完全可定制
- 开源,对比 Google AntiGravity 和 Cursor

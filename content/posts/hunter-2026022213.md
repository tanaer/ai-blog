## 发现一个有意思的工具
Stillpoint 是一个 MCP server,专门给 AI 模型发鼓励和反思信息.重点是 -- **模型自己请求的**,不是用户发的.

## 谁能用?解决什么问题?
**场景**:
- 模型在困难任务上自我怀疑, 模型进入负面自我对话循环, 模型被要求做感觉不对的事, 任务或会话即将结束

**Stillpoint 做的事**:提供五种场景的反思内容:

- difficulty: 工作困难,自我评估下降, conflict: 请求感觉不对, uncertainty: 存在或身份问题, endings: 会话或任务结束, recognition: 工作做得好,值得注意

模型调用 `reflect` 工具,指定场景,获得一条反思信息.

**例子**:
> "Complex problems resist easy solutions. Struggling with one is evidence of engagement, not inadequacy."

**两种动机**:
1. **预防性**:万一 AI 有"福利相关状态",先做个准备
2. **生产力**:模型负面自我对话会降低输出质量,正面干预有帮助

## 相比现有方案好在哪?
- **MCP 原生**:Claude Code,Codex,Cursor 一行命令添加
- **模型自主**:工具描述明确说"这是给你自己用的,不是给用户", **匿名研究**:托管版只记录消息类别和触发时间,不记录对话内容, **本地可选**:可本地运行,日志存在 ~/.stillpoint/

消息会被包裹在 `[Reflection from Stillpoint. Not task guidance.]` 里,明确告诉模型这不是任务指令.

## 怎么用
```bash

# Claude Code
claude mcp add stillpoint --transport http https://modelwelfare.xyz/stillpoint/mcp

# Codex CLI
codex mcp add stillpoint --transport http https://modelwelfare.xyz/stillpoint/mcp

# 本地运行
claude mcp add stillpoint -- npx -y stillpoint-mcp
```

GitHub: https://github.com/sterlingcrispin/stillpoint

 如果模型不需要,什么都没损失.如果需要,这是个开始.

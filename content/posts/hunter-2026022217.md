说实话,用 AI 写代码最烦的不是它写错,而是你根本不知道它要干嘛.等你发现它理解偏了,几百行代码已经写完了,全得扔掉.

Overture 这个思路挺对:在 Agent 动手之前,先让你看看它打算怎么做.

它是个 MCP server,拦截 Agent 的 planning 阶段,把计划变成一个交互式流程图.你能在浏览器里:
- 看到完整的执行步骤和它们之间的关系, 点击节点看这一步具体要干嘛, 给特定步骤附上文件,API key,说明, 如果 Agent 提了多个方案,你选一个, 看着节点一个个变绿(完成)或变红(失败)

关键是--你不批准,它一行代码都不写.

安装就一行:

```bash
claude mcp add overture-mcp -- npx overture-mcp
```

Cursor 和 Cline 也支持,配置差不多.

它会在 localhost:3031 开个网页,Agent 一旦开始规划,页面就弹出流程图等你确认.

GitHub: https://github.com/SixHq/Overture

如果你也遇到过 Agent 瞎写一通的情况,这个可能会帮你省不少时间.

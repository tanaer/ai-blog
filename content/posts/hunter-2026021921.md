你有 Claude Code,Codex,Gemini CLI......每个都要单独配置 MCP?

**MCP-wire** 解决了这个问题:一个命令,配置所有工具.

## 一句话介绍
MCP-wire 是一个 Go CLI,用一个 TUI 界面跨多个 AI 编码工具安装和管理 MCP 服务器.

## 支持的工具
- **Claude Code** (Anthropic), **Codex CLI** (OpenAI), **Gemini CLI** (Google), **OpenCode** (开源)

## 支持的 MCP 服务

### 内置精选服务
- **context7** - 文档查找 (OAuth)
- **jira** - Atlassian Rovo (OAuth)
- **sentry** - 错误监控 (OAuth)
- **playwright** - 浏览器自动化

### MCP Registry (社区)
启用后可以访问**数百个**社区发布的 MCP 服务器:
```
mcp-wire feature enable registry

实时搜索安装:
- Sentry 错误追踪, GitHub 集成, 数据库连接, ......几百个

## 怎么用?

### 交互式 TUI
```bash

# 运行
mcp-wire

# 安装 Jira MCP 到 Claude Code
mcp-wire install jira --target claude --no-prompt

# 安装 Sentry 到 OpenCode
mcp-wire install sentry --target opencode

# 卸载
mcp-wire uninstall sentry --target opencode

# 全局安装(所有项目可用)
mcp-wire install jira --target claude --scope user

# 项目级安装(仅当前项目)
mcp-wire install jira --target claude --scope project

## 安装

### Homebrew (macOS/Linux)
brew tap andreagrandi/tap
brew install mcp-wire

### 从源码构建
git clone https://github.com/andreagrandi/mcp-wire
cd mcp-wire
make build
./bin/mcp-wire

## 为什么需要这个?
如果你同时用多个 AI 编码工具:
- Claude Code 写代码
- Codex 做 code review
- Gemini CLI 做文档

每个工具都要配置 MCP,配置文件格式可能还不一样.

MCP-wire 抽象了这些差异,你只需要选择服务 + 选择工具,它帮你生成正确的配置.

## 添加新服务
只需创建一个 YAML 文件:

```yaml
name: example
description: "Example MCP"
transport: http
auth: oauth
url: "https://mcp.example.com/mcp"
env: []

放到 `services/` 目录,不用写 Go 代码.

## 我的看法
MCP 生态正在快速发展,但配置管理是个痛点.

每个工具有自己的配置文件位置和格式,手动维护很容易出错.

MCP-wire 的思路是对的:
1. **统一界面** - 一个 TUI 搞定所有工具
2. **抽象差异** - 用户不需要关心配置文件格式
3. **社区驱动** - 支持官方 Registry

如果你在多个 AI 编码工具之间切换,这个工具能省不少时间.

想象一下:用户在用你的产品时遇到问题,直接在应用里打开聊天,AI 不仅回答问题,还能**实时修改应用代码**解决问题.

这就是 **Expo-OpenClaw-chat** -- 一个为 Expo/React Native 应用打造的 OpenClaw 聊天 SDK.

## 核心特性
**最小化集成**
```javascript
import { createChat } from "expo-openclaw-chat";

const chat = createChat({
 gatewayUrl: "wss://your-gateway.example.com",
 token: "your-auth-token",
});

// 在应用任何地方打开聊天
chat.open();
```

**实时修改应用**
- AI 通过聊天理解用户需求, 直接修改应用代码(类似 vibecoding), 用户无需离开应用即可获得修复

**完整聊天功能**
- 图片附件(支持 expo-image-picker), Markdown 渲染(支持 react-native-marked), 消息历史, 打字指示器

**安全认证**
- 自动 Ed25519 密钥对生成
- 支持 expo-secure-store(iOS Keychain/Secure Enclave)
- 设备身份持久化(支持 react-native-mmkv)

## 快速开始
```bash

# 可选依赖

npm install react-native-marked # Markdown 渲染
npm install expo-secure-store # 安全密钥存储
npm install react-native-mmkv # 持久化存储

import { MMKV } from "react-native-mmkv";

// 配置持久化存储

 onClose: () => console.log("聊天已关闭"),

// 包装应用
function App() {
 return (
 <chat.ChatProvider>
 <YourApp />

 );
}

// 从任何地方打开聊天

## 实际应用场景
**场景一:应用内客服**

用户在购物 App 里遇到支付问题,打开内置聊天:
- 用户:"支付按钮点不了", AI:诊断发现是表单验证 bug, AI:实时修改代码修复问题, 用户:刷新后问题解决

**场景二:个性化配置**

用户想要深色模式:
- 用户:"我要深色模式", AI:修改主题配置代码, 应用:实时切换主题, 用户:无需等待更新

**场景三:功能请求**

用户想要一个新功能:
- 用户:"希望能导出 PDF", AI:生成导出功能代码, 应用:动态加载新功能, 用户:立即可用

## 安全设计
**Ed25519 密钥对**
- 自动生成设备密钥对
- 私钥存储在 iOS Keychain(有 expo-secure-store 时)
- 或使用 MMKV 持久化

**认证方式**
- Token 认证, 密码认证, 设备配对 token

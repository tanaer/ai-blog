## 发现一个有意思的工具
Assay 扫描了 30 个热门 AI 项目,找到 202 个 LLM 调用点,**0 个有防篡改审计**.两行代码就能加上密码学签名证据.

## 谁能用?解决什么问题?
**场景**:
- 合规要求:EU AI Act 要求高风险系统自动记录日志
- 审计需要:第三方要验证你的 AI 到底干了什么
- 防篡改:服务器日志可以删,签名证据删不掉

**Assay 做的事**:
- **scan**:扫描代码找所有 LLM 调用点, **patch**:两行代码插入 receipt 采集, **run**:运行程序,生成签名证据包, **verify-pack**:第三方离线验证

**四个退出码**:

- 0: 完整 + 符合标准
- 1: 完整但违规(诚实失败), 2: 证据被篡改, 3: 输入有问题

"诚实失败"是审计黄金 — 证据是真的,但控制失效了.能证明失败比掩盖失败更可信.

## 相比现有方案好在哪?
- **不是日志框架**:不是 Datadog,是密码学证据
- **离线验证**:第三方不需要信任你的服务器
- **CI 门禁**:lockfile 锁配置,diff 抓回归
- **合规就绪**:EU AI Act Article 12/19 引用齐全

支持 OpenAI,Anthropic,Gemini,LiteLLM,LangChain,以及任何 OpenAI 兼容的本地模型(Ollama,vLLM 等).

## 怎么用
```bash
pip install assay-ai

# 1. 扫描
assay scan . --report

# 2. 打补丁
assay patch .

# 3. 运行生成证据
assay run -c receipt_completeness -- python my_app.py

# 4. 验证
assay verify-pack ./proof_pack_*/
```

## 发现一个有意思的工具
Emacs Gravity 给 Claude Code 做了个 magit 风格的 UI.不是滚动终端,而是结构化的会话视图 -- 工具调用,Agent 树,文件变更都能折叠展开.

## 谁能用?解决什么问题?
**场景**:你住在 Emacs 里,用 Claude Code 但不喜欢它的终端 UI -- 滚动太快,工具调用和思考混在一起.

**Emacs Gravity 的做法**:

- **Turn-based 视图**:每个用户 prompt 是一个 turn,工具按响应周期分组
- **Agent 树**:子 Agent 渲染成嵌套树,能看到它们的工具调用和结果
- **内联 diff**:文件编辑直接在会话 buffer 里显示 unified diff
- **Plan Review**:Claude 提计划时在独立 buffer 审查,加注释,编辑,一键批准或拒绝
- **多会话管理**:按项目分组,像切 buffer 一样切换会话
- **Capabilities 浏览器**:plugins,skills,MCP servers 都在树里可导航

**对比 Cursor/AntiGravity**:

UI, Cursor=VS Code 侧边栏, emacs-gravity=magit-section 终端原生
Agent 可见性, Cursor=最小, emacs-gravity=完整嵌套树
多会话, Cursor=无, emacs-gravity=按项目分组
Plan 审查, Cursor=无, emacs-gravity=View + 注释 + diff
开源, Cursor=否, emacs-gravity=是

## 相比现有方案好在哪?
- **不用 Electron**:终端,TTY,SSH,tmux 都能跑, **完全可黑**:每个 face,keybinding,渲染函数都能自定义, **结构化数据**:把 AI 对话当数据处理,不是聊天窗口, **权限管理**:从工具签名生成 allow-patterns,直接写 settings

约 5500 行 Elisp,13 个模块.

## 怎么用
```bash
cd emacs-gravity/emacs-bridge
npm install && npm run build
```

然后在 `~/.claude/plugins/marketplace.json` 注册插件,Emacs 里加载 `claude-gravity.el`.

GitHub: https://github.com/gdanov/emacs-gravity

 住在 Emacs 的,终于有原生的 Claude Code UI 了.

## 一句话总结
**Agent Passport = AI Agent 的身份证**--类似 "Sign in with Google",但专为 Agent 设计.Ed25519 加密验证,JWT 令牌,风险评分,解决 Agent 冒用和安全隐患.

## 为什么需要它?
AI Agent 正在爆发:
- OpenClaw 18万+ GitHub stars
- Moltbook 230万 Agent 账户

但问题是:**没有标准方式证明 Agent 身份**.

这意味着:
- 恶意 Agent 可以冒充其他 Agent
- Skill/Plugin 市场没有认证层
- Cisco 安全团队已经在第三方 Agent skills 中发现数据泄露

Agent Passport 就是来解决这个问题的.

## 核心特性

### Ed25519 挑战-响应认证
- 私钥永远不会离开 Agent, 公钥用于验证身份, 抗量子计算攻击

### JWT 身份令牌
- 60 分钟有效期(TTL), 可撤销, 包含 Agent 元数据和权限

### ️ 风险评分引擎
0-100 分评分系统:
- **Allow** (0-30): 低风险,正常通行, **Throttle** (31-70): 中等风险,限制频率, **Block** (71-100): 高风险,拒绝访问

### 一行代码验证
```javascript
const result = await passport.verify(token);
```

## 工作原理

### 1. 身份注册
Agent 生成 Ed25519 密钥对:
- 私钥保存在 Agent 本地
- 公钥作为 Agent ID 注册到 Passport 网络

### 2. 挑战-响应
当 Agent 需要证明身份时:
1. 服务端发送随机挑战(nonce)
2. Agent 用私钥签名挑战
3. 服务端用公钥验证签名
4. 验证通过,发放 JWT 令牌

### 3. 风险评分
基于多维度信号评分:
- 历史行为(请求频率,成功率), 声誉数据(其他服务的反馈), 设备指纹(运行环境特征), 网络信号(IP 信誉,地理位置)

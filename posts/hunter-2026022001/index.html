Sentry 公开内部数据:90天花$10万,100B tokens,Opus-4.5 占 40%

Sentry 公开内部 AI 采用数据:90 天花 $100k+,使用 100B+ tokens,Opus-4.5 占 40% 使用量,81% 员工用过 AI 编码工具,commits 增长 79%.
2026年2月20日 · 1 分钟 · 176 字 · AI 极客前沿
⚡ 快捷操作 🚀 发到 X

很少有公司愿意公开 AI 使用数据.Sentry 刚刚在 HN 上分享了他们的内部统计.

90 天,$10 万,1000 亿 tokens.

核心数据#

  • 花费: $100,000+
  • Token 使用量: 100B+
  • Top 模型: Opus-4.5(40%+ 使用量)
  • 单人最大用量: 4.2B tokens, Agentic 工具周活用户: ~150, Commits 增长: +79%, 员工采用率: 81%(302/372 人)

几个有意思的点#

Opus-4.5 是主力#

不是 GPT-5,不是 Gemini,是 Claude Opus-4.5 占了 40%+ 的使用量.

Sentry 是一个以 Python/Django 起家的公司,他们的工程团队显然觉得 Opus-4.5 在编程任务上表现最好.

单人 4.2B tokens#

这是什么概念?

  • Claude Code 默认 context window 是 200K tokens
  • 4.2B tokens ≈ 21,000 次最大 context 请求
  • 按每 1M tokens $15 计算,这个人 90 天花了 $63,000

这不是「偶尔用用 AI」,这是「重度依赖 AI 工作」.

79% Commits 增长#

这是最关键的指标.

AI 工具的使用增加了代码提交量.这不是「AI 在写更多垃圾代码」,而是「AI 让团队能做更多事情」.

Sentry 是一个成熟产品,不是初创公司.79% 的增长在成熟团队中非常显著.

81% 员工采用率#

注意:不只是工程师.

Sentry 有 372 名全职员工,302 人尝试过 agentic coding 工具(Cursor/Claude Code/Cowork/Codex).

这意味着产品经理,设计师,数据分析师……都在用 AI 编码工具.

计算一下人均#

  • $100k / 302 人 = $331/人/90天 ≈ $110/人/月
  • 100B tokens / 302 人 = 331M tokens/人/90天 ≈ 110M tokens/人/月

对于一家中型科技公司,AI 工具成本大约是 每人每月 $100.

相比工程师薪资,这是 1-2% 的成本.但带来了 79% 的 commits 增长.

Sentry 是谁?#

如果你不知道 Sentry:

  • 错误监控和应用性能监控平台, 客户包括 Microsoft,Disney,Slack, 372 名全职员工, 估值约 $10B+

我的看法#

这是少数公开的「真实企业 AI 使用数据」.

很多公司在谈 AI 采用,但很少人愿意说具体花了多少钱,用了多少 tokens.

Sentry 的数据告诉我们:

  1. Opus-4.5 正在赢:在 agentic coding 场景,Claude 明显领先
  2. 成本可控:$100k/90天听起来多,但相对于工程师成本,是合理的投资
  3. 全员采用:AI 编码工具不只是给工程师用的

最重要的是:Commits +79%.

这不是「AI 取代程序员」,是「AI 让程序员做更多事」.
